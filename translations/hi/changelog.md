#### मूल अवधारणा संवर्धन (01-CoreConcepts/)
- **नया क्लाइंट प्रिमिटिव - Roots**: रूट क्लाइंट प्रिमिटिव पर व्यापक दस्तावेज़ जोड़ा गया, जिससे सर्वर फ़ाइल सिस्टम सीमाओं और पहुँच अनुमतियों को समझ सकें
- **टूल एनोटेशन**: बेहतर टूल निष्पादन निर्णय के लिए टूल व्यवहार एनोटेशन (`readOnlyHint`, `destructiveHint`) पर दस्तावेज़ जोड़ा गया
- **सैम्पलिंग में टूल कॉलिंग**: सैम्पलिंग दस्तावेज़ अपडेट किया गया ताकि `tools` और `toolChoice` पैरामीटर शामिल हों, जो मॉडल-चालित टूल कॉल को सैम्पलिंग अनुरोधों के दौरान सक्षम करते हैं
- **URL मोड उद्घाटन**: सर्वर-प्रेरित बाहरी वेब इंटरैक्शन के लिए URL-आधारित उद्घाटन पर दस्तावेज़ जोड़ा गया
- **कार्य (प्रायोगिक)**: ड्यूरेबल कार्यान्वयन रैपर और विलंबित परिणाम पुनःप्राप्ति के लिए प्रायोगिक टास्क फीचर का नया अनुभाग जोड़ा गया
- **आइकन समर्थन**: नोट किया गया कि टूल्स, संसाधन, संसाधन टेम्पलेट्स, और प्रॉम्प्ट्स अब अतिरिक्त मेटाडेटा के रूप में आइकन शामिल कर सकते हैं

### शुरुआत अनुभाग विस्तार – उन्नत सर्वर उपयोग और सरल प्रमाणीकरण

#### उन्नत सर्वर उपयोग (03-GettingStarted/10-advanced)
- **नया अध्याय जोड़ा गया**: उन्नत MCP सर्वर उपयोग के लिए एक व्यापक गाइड पेश किया गया, जिसमें नियमित और निम्न-स्तरीय सर्वर आर्किटेक्चर दोनों शामिल हैं।
  - **नियमित बनाम निम्न-स्तरीय सर्वर**: दोनों दृष्टिकोणों के लिए Python और TypeScript में विस्तृत तुलना और कोड उदाहरण।
  - **हैंडलर-आधारित डिज़ाइन**: स्केलेबल, लचीले सर्वर कार्यान्वयन के लिए हैंडलर-आधारित टूल/संसाधन/प्रॉम्प्ट प्रबंधन का विवरण।
  - **व्यावहारिक पैटर्न**: वास्तविक दुनिया के परिदृश्य जहां निम्न-स्तरीय सर्वर पैटर्न उन्नत सुविधाओं और आर्किटेक्चर के लिए लाभकारी हैं।

#### सरल प्रमाणीकरण (03-GettingStarted/11-simple-auth)
- **नया अध्याय जोड़ा गया**: MCP सर्वरों में सरल प्रमाणीकरण को लागू करने के लिए चरण-दर-चरण मार्गदर्शिका।
  - **ऑथ अवधारणाएँ**: प्रमाणीकरण और प्राधिकरण का स्पष्ट विवरण, और क्रेडेंशियल हैंडलिंग।
  - **बेसिक ऑथ कार्यान्वयन**: Python (Starlette) और TypeScript (Express) में मिडलवेयर-आधारित प्रमाणीकरण पैटर्न, कोड नमूनों के साथ।
  - **उन्नत सुरक्षा की ओर प्रगति**: सरल ऑथ से शुरू करके OAuth 2.1 और RBAC तक मार्गदर्शन, उन्नत सुरक्षा मॉड्यूल के संदर्भ के साथ।

ये जोड़ MCP सर्वर कार्यान्वयनों को अधिक मज़बूत, सुरक्षित, और लचीला बनाने के लिए व्यावहारिक, हाथों-हाथ मार्गदर्शन प्रदान करते हैं, बुनियादी अवधारणाओं और उन्नत उत्पादन पैटर्न के बीच पुल बनाते हैं।

    - प्रांप्ट इंजेक्शन एवं टूल पॉइज़निंग रोकथाम के साथ AI-विशिष्ट सुरक्षा नियंत्रण  
    - OAuth प्रॉक्सी सुरक्षा के साथ कन्फ्यूज्ड डेप्यू हमला रोकथाम  
    - सैंडबॉक्सिंग एवं पृथक्करण के साथ टूल निष्पादन सुरक्षा  
    - निर्भरता सत्यापन के साथ आपूर्ति श्रृंखला सुरक्षा नियंत्रण  
    - SIEM एकीकरण के साथ निगरानी एवं पहचान नियंत्रण  
    - स्वचालित क्षमताओं के साथ घटना प्रतिक्रिया एवं पुनर्प्राप्ति  
  - **कार्यान्वयन उदाहरण**: विस्तृत YAML विन्यास ब्लॉकों और कोड उदाहरण जोड़े गए  
  - **Microsoft समाधान एकीकरण**: Azure सुरक्षा सेवाएँ, GitHub Advanced Security, और एंटरप्राइज़ पहचान प्रबंधन का व्यापक कवरेज

  - **AI सुरक्षा एकीकरण**: Microsoft Prompt Shields और Azure Content Safety क्रियान्वयन, विस्तृत Python उदाहरणों के साथ

    - ऑडियंस सत्यापन और सुरक्षित टोकन प्रबंधन के साथ टोकन पासथ्रू रोकथाम  
    - क्रिप्टोग्राफिक बाइंडिंग और व्यवहार विश्लेषण के साथ सत्र अपहरण रोकथाम  
  - **एंटरप्राइज़ सुरक्षा एकीकरण**: Azure Application Insights निगरानी, खतरे का पता लगाने वाली पाइपलाइन, और आपूर्ति श्रृंखला सुरक्षा

#### मूल अवधारणाएँ (01-CoreConcepts/) - पूर्ण आधुनिकीकरण  
- **प्रोटोकॉल संस्करण अपडेट**: MCP विनिर्देशन 2025-06-18 को संदर्भित करते हुए तारीख आधारित संस्करण (YYYY-MM-DD प्रारूप)  
- **आर्किटेक्चर परिष्कार**: होस्ट्स, क्लाइंट्स, और सर्वर्स का वर्तमान MCP आर्किटेक्चर पैटर्न के अनुसार वर्णन बेहतर किया गया  
  - होस्ट अब स्पष्ट रूप से AI अनुप्रयोग हैं जो कई MCP क्लाइंट कनेक्शन समन्वयित करते हैं  
  - क्लाइंट्स को प्रोटोकॉल कनेक्टर के रूप में वर्णित किया गया जो एक-से-एक सर्वर संबंध बनाए रखते हैं  
  - सर्वर्स में स्थानीय एवं दूरस्थ तैनाती परिदृश्यों का विस्तार किया गया  
- **प्रिमिटिव पुनर्गठन**: सर्वर और क्लाइंट प्रिमिटिव का पूर्ण पुनर्लेखन  
  - सर्वर प्रिमिटिव: संसाधन (डेटा स्रोत), प्रॉम्प्ट (टेम्प्लेट), टूल्स (निष्पादनीय फ़ंक्शन) के विस्तृत विवरण एवं उदाहरण  
  - क्लाइंट प्रिमिटिव: सैंपलिंग (LLM पूर्णता), इलिसिटेशन (उपयोगकर्ता इनपुट), लॉगिंग (डिबगिंग/निगरानी)  
  - वर्तमान खोज (`*/list`), पुनर्प्राप्ति (`*/get`), और निष्पादन (`*/call`) विधि पैटर्न के साथ अद्यतन  
- **प्रोटोकॉल आर्किटेक्चर**: दो-स्तरीय वास्तुकला मॉडल प्रस्तुत किया गया  
  - डेटा स्तर: JSON-RPC 2.0 आधार, जीवनचक्र प्रबंधन और प्रिमिटिव  
  - परिवहन स्तर: STDIO (स्थानीय) और स्ट्रीमेबल HTTP के साथ SSE (दूरस्थ) परिवहन तंत्र  
- **सुरक्षा ढांचा**: स्पष्ट उपयोगकर्ता सहमति, डेटा गोपनीयता सुरक्षा, टूल निष्पादन सुरक्षा, और परिवहन स्तर सुरक्षा सहित व्यापक सुरक्षा सिद्धांत  
- **संचार पैटर्न**: प्रारंभिक करना, अन्वेषण, निष्पादन, और सूचना प्रवाह दिखाने के लिए प्रोटोकॉल संदेश अपडेटेड  
- **कोड उदाहरण**: वर्तमान MCP SDK पैटर्न को दर्शाने के लिए बहु-भाषा उदाहरण (.NET, Java, Python, JavaScript) ताज़ा किए गए  

#### सुरक्षा (02-Security/) - व्यापक सुरक्षा पुनर्लेखन  
- **मानक अनुरूपता**: MCP विनिर्देशन 2025-06-18 सुरक्षा आवश्यकताओं के साथ पूर्ण अनुरूपता  
- **प्रमाणीकरण विकास**: कस्टम OAuth सर्वर्स से बाहरी पहचान प्रदाता प्रतिनिधित्व (Microsoft Entra ID) तक का विकास दस्तावेजीकृत  
- **AI-विशिष्ट खतरे विश्लेषण**: आधुनिक AI हमले वेक्टर का संवर्धित कवरेज  
  - विस्तृत प्रॉम्प्ट इंजेक्शन हमला परिदृश्य वास्तविक उदाहरणों के साथ  
  - टूल पॉइज़निंग तंत्र और "रग पुल" हमला पैटर्न  
  - संदर्भ विंडो पॉइज़निंग और मॉडल भ्रम हमले  
- **Microsoft AI सुरक्षा समाधान**: Microsoft सुरक्षा इकोसिस्टम का व्यापक कवरेज  
  - AI Prompt Shields के साथ उन्नत पहचान, स्पॉटलाइटिंग और डिमिटर्स तकनीक  
  - Azure Content Safety एकीकरण पैटर्न  
  - GitHub Advanced Security आपूर्ति श्रृंखला सुरक्षा के लिए  
- **उन्नत खतरा कम करना**: विस्तृत सुरक्षा नियंत्रण  
  - MCP-विशिष्ट सत्र अपहरण हमले परिदृश्य और क्रिप्टोग्राफिक सत्र ID आवश्यकताएं  
  - स्पष्ट सहमति आवश्यकताओं के साथ MCP प्रॉक्सी में कन्फ्यूज्ड डेप्यू समस्या  
  - अनिवार्य सत्यापन नियंत्रण के साथ टोकन पासथ्रू कमजोरियाँ  
- **आपूर्ति श्रृंखला सुरक्षा**: आधार मॉडल, एम्बेडिंग सेवाएँ, संदर्भ प्रदाता, और तृतीय-पक्ष API सहित AI आपूर्ति श्रृंखला कवरेज का विस्तार  
- **फाउंडेशन सुरक्षा**: शून्य विश्वास वास्तुकला और Microsoft सुरक्षा इकोसिस्टम सहित एंटरप्राइज़ सुरक्षा पैटर्न में एकीकरण बेहतर किया गया  
- **संसाधन संगठन**: प्रकार (आधिकारिक दस्तावेज़, मानक, शोध, Microsoft समाधान, कार्यान्वयन गाइड) द्वारा वर्गीकृत व्यापक संसाधन लिंक
