# บันทึกการเปลี่ยนแปลง: หลักสูตร MCP สำหรับผู้เริ่มต้น

เอกสารนี้ทำหน้าที่เป็นบันทึกของการเปลี่ยนแปลงที่สำคัญทั้งหมดที่ทำกับหลักสูตร Model Context Protocol (MCP) สำหรับผู้เริ่มต้น การเปลี่ยนแปลงจะถูกบันทึกในลำดับย้อนกลับ (รายการล่าสุดก่อน)

#### เพิ่มเนื้อหาหลักสูตรใหม่

**โมดูล 05 - หัวข้อขั้นสูง**
- **บทเรียน 5.17: การถกเถียงหลายตัวแทนแบบขัดแย้งด้วย MCP** (`05-AdvancedTopics/mcp-adversarial-agents/README.md`): คู่มือเจาะลึกใหม่ ครอบคลุมรูปแบบการถกเถียงแบบขัดแย้งสำหรับระบบหลายตัวแทน
  - แผนภาพสถาปัตยกรรม Mermaid: ตัวแทนสองตัว → เซิร์ฟเวอร์ MCP ร่วม → บทถกเถียง → ผู้ตัดสิน → คำตัดสิน
  - เซิร์ฟเวอร์เครื่องมือ MCP ร่วม (`web_search` + `run_python`) ที่พัฒนาด้วย Python และ TypeScript
  - ระบบ prompt ตรงข้าม (FOR / AGAINST / ผู้ตัดสิน) พร้อมข้อกำหนดการใช้เครื่องมืออย่างชัดเจน
  - ตัวจัดการการถกเถียงใน Python, TypeScript และ C# ควบคุมรอบและเส้นทางโต้แย้ง
  - การเชื่อมต่อ MCP `ClientSession` สำหรับตัวจัดการไปยังการเรียกใช้งานเครื่องมือจริง
  - ตารางกรณีการใช้งาน (ตรวจจับการหลอกลวง, การวางโมเดลภัยคุกคาม, การทบทวนออกแบบ API, การตรวจสอบข้อเท็จจริง, การเลือกเทคโนโลยี)
  - ข้อควรระวังด้านความปลอดภัย: การรันแบบจำกัด, การตรวจสอบการเรียกเครื่องมือ, จำกัดอัตราการเรียก, บันทึกตรวจสอบ
  - แบบฝึกหัดที่มีโครงสร้างพร้อมสามสถานการณ์ปฏิบัติ (ทบทวนโค้ด, การตัดสินใจสถาปัตยกรรม, การควบคุมเนื้อหา)

## 28 มกราคม 2026

### การทบทวนความสอดคล้องของข้อกำหนด MCP 2025-11-25

#### การปรับปรุงแนวคิดหลัก (01-CoreConcepts/)
- **Primitive ฝั่งไคลเอนต์ใหม่ - Roots**: เพิ่มเอกสารครบถ้วนเกี่ยวกับ Primitive Roots ซึ่งทำให้เซิร์ฟเวอร์เข้าใจขอบเขตระบบแฟ้มและสิทธิ์การเข้าถึง
- **คำอธิบายพฤติกรรมเครื่องมือ**: เพิ่มเอกสารคำอธิบายพฤติกรรมเครื่องมือ (`readOnlyHint`, `destructiveHint`) เพื่อช่วยในการตัดสินใจการรันเครื่องมือ
- **การเรียกเครื่องมือใน Sampling**: อัปเดตเอกสาร Sampling เพื่อรวมพารามิเตอร์ `tools` และ `toolChoice` สำหรับการเรียกใช้เครื่องมือโดยโมเดลระหว่างคำขอ sampling
- **การเปิดเผยโหมด URL**: เพิ่มเอกสารเกี่ยวกับการเปิดเผยโหมด URL สำหรับการโต้ตอบเว็บภายนอกที่เซิร์ฟเวอร์เริ่มต้น
- **Tasks (ทดลอง)**: เพิ่มส่วนใหม่ที่เอกสารฟีเจอร์ทดลอง Tasks สำหรับ wrapper การดำเนินการที่ทนทานและการดึงผลล่าช้า
- **รองรับไอคอน**: แจ้งว่าเครื่องมือ ทรัพยากร เทมเพลตทรัพยากร และ prompt สามารถรวมไอคอนได้เป็นเมตาดาต้าเพิ่มเติม

#### การตรวจสอบความสอดคล้องข้อกำหนด
- **เวอร์ชันโปรโตคอล**: ยืนยันว่าสื่อสารอ้างอิงเอกสาร MCP Specification 2025-11-25 ล่าสุด
- **สถาปัตยกรรมที่สอดคล้องกัน**: ยืนยันความถูกต้องของเอกสารสถาปัตยกรรมสองชั้น (Data Layer + Transport Layer)
- **เอกสาร Primitive**: ตรวจสอบ Primitive ฝั่งเซิร์ฟเวอร์ (Resources, Prompts, Tools) และฝั่งไคลเอนต์ (Sampling, Elicitation, Logging, Roots)
- **กลไกการขนส่ง**: ยืนยันเอกสารขนส่ง STDIO และ HTTP แบบสตรีมที่ถูกต้อง
- **แนวทางความปลอดภัย**: ยืนยันว่าสอดคล้องกับแนวทางปฏิบัติความปลอดภัย MCP ปัจจุบัน

#### ฟีเจอร์สำคัญของ MCP 2025-11-25 ที่ได้รับการจัดทำเอกสาร
- **การค้นหาผ่าน OpenID Connect**: การค้นหาเซิร์ฟเวอร์ Authentication ผ่าน OIDC
- **เอกสารเมตาดาต้า Client ID OAuth**: กลไกการลงทะเบียนไคลเอนต์ที่แนะนำ
- **JSON Schema 2020-12**: ภาษาพูดมาตรฐานสำหรับการกำหนด schema ของ MCP
- **ระบบระดับ SDK**: กำหนดความต้องการอย่างเป็นทางการสำหรับฟีเจอร์ SDK และการบำรุงรักษา
- **โครงสร้างการกำกับดูแล**: จัดรูปแบบกลุ่มงานและกลุ่มความสนใจในระบบการกำกับดูแล MCP

## 18 ธันวาคม 2025
### การอัปเดตเอกสารความปลอดภัย - สเปค MCP 2025-11-25

#### แนวทางปฏิบัติที่ดีที่สุดด้านความปลอดภัย MCP (02-Security/mcp-best-practices.md) - อัปเดตเวอร์ชันสเปค
- **อัปเดตเวอร์ชันโปรโตคอล**: อัปเดตอ้างอิงสเปค MCP ล่าสุด 2025-11-25 (เผยแพร่วันที่ 25 พฤศจิกายน 2025)
  - อัปเดตอ้างอิงเวอร์ชันสเปคทั้งหมดจาก 2025-06-18 เป็น 2025-11-25
  - อัปเดตอ้างอิงวันที่ในเอกสารจาก 18 สิงหาคม 2025 เป็น 18 ธันวาคม 2025
  - ตรวจสอบ URL สเปคทั้งหมดให้ชี้ไปยังเอกสารปัจจุบัน
- **การตรวจสอบเนื้อหา**: ตรวจสอบแนวทางปฏิบัติที่ดีที่สุดด้านความปลอดภัยอย่างครอบคลุมตามมาตรฐานล่าสุด
  - **โซลูชันความปลอดภัย Microsoft**: ตรวจสอบคำศัพท์และลิงก์ปัจจุบันสำหรับ Prompt Shields (เดิมคือ "การตรวจจับความเสี่ยง Jailbreak"), Azure Content Safety, Microsoft Entra ID, และ Azure Key Vault
  - **ความปลอดภัย OAuth 2.1**: ยืนยันความสอดคล้องกับแนวทางปฏิบัติด้านความปลอดภัย OAuth ล่าสุด
  - **มาตรฐาน OWASP**: ตรวจสอบความอัปเดตของ OWASP Top 10 สำหรับ LLM
  - **บริการ Azure**: ตรวจสอบลิงก์เอกสาร Microsoft Azure และแนวทางปฏิบัติที่ดีที่สุดทั้งหมด
- **การสอดคล้องกับมาตรฐาน**: ยืนยันว่าเกณฑ์ความปลอดภัยที่อ้างอิงทั้งหมดเป็นปัจจุบัน
  - NIST AI Risk Management Framework
  - ISO 27001:2022
  - แนวทางปฏิบัติด้านความปลอดภัย OAuth 2.1
  - เฟรมเวิร์กด้านความปลอดภัยและการปฏิบัติตาม Azure
- **ทรัพยากรการใช้งาน**: ตรวจสอบลิงก์และทรัพยากรคู่มือการใช้งานทั้งหมด
  - รูปแบบการพิสูจน์ตัวตน Azure API Management
  - คู่มือการรวม Microsoft Entra ID
  - การจัดการความลับ Azure Key Vault
  - ท่อ DevSecOps และโซลูชันตรวจสอบ

### การประกันคุณภาพเอกสาร
- **การปฏิบัติตามสเปค**: ตรวจสอบให้แน่ใจว่าข้อกำหนดความปลอดภัย MCP ที่จำเป็นทั้งหมด (MUST/MUST NOT) สอดคล้องกับสเปคล่าสุด
- **ความทันสมัยของทรัพยากร**: ตรวจสอบลิงก์ภายนอกทั้งหมดที่เชื่อมไปยังเอกสาร Microsoft, มาตรฐานความปลอดภัย และคู่มือการใช้งาน
- **ครอบคลุมแนวทางปฏิบัติที่ดีที่สุด**: ยืนยันว่าครอบคลุมการพิสูจน์ตัวตน, การอนุญาต, ภัยคุกคามเฉพาะ AI, ความปลอดภัยห่วงโซ่อุปทาน, และรูปแบบองค์กรอย่างครบถ้วน

## 6 ตุลาคม 2025

### ขยายส่วน Getting Started – การใช้งานเซิร์ฟเวอร์ขั้นสูง & การพิสูจน์ตัวตนง่ายๆ

#### การใช้งานเซิร์ฟเวอร์ขั้นสูง (03-GettingStarted/10-advanced)
- **เพิ่มบทใหม่**: แนะนำคู่มือครบถ้วนสำหรับการใช้งาน MCP เซิร์ฟเวอร์ขั้นสูง ครอบคลุมทั้งสถาปัตยกรรมเซิร์ฟเวอร์ปกติและระดับต่ำ
  - **เซิร์ฟเวอร์ปกติเทียบกับระดับต่ำ**: เปรียบเทียบรายละเอียด พร้อมตัวอย่างโค้ด Python และ TypeScript ของทั้งสองรูปแบบ
  - **การออกแบบแบบใช้ Handler**: อธิบายการจัดการเครื่องมือ/ทรัพยากร/พรอมต์โดยใช้ handler เพื่อสนับสนุนการขยายและความยืดหยุ่นของเซิร์ฟเวอร์
  - **รูปแบบการใช้งานจริง**: สถานการณ์จริงที่รูปแบบเซิร์ฟเวอร์ระดับต่ำเป็นประโยชน์สำหรับฟีเจอร์และสถาปัตยกรรมขั้นสูง

#### การพิสูจน์ตัวตนง่ายๆ (03-GettingStarted/11-simple-auth)
- **เพิ่มบทใหม่**: คู่มือทีละขั้นตอนสำหรับการนำไปใช้การพิสูจน์ตัวตนง่ายๆ ใน MCP เซิร์ฟเวอร์
  - **แนวคิดของการพิสูจน์ตัวตน**: อธิบายความแตกต่างระหว่างการพิสูจน์ตัวตนและการอนุญาต พร้อมการจัดการข้อมูลรับรอง
  - **การนำเสนอการพิสูจน์ตัวตนพื้นฐาน**: รูปแบบการพิสูจน์ตัวตนแบบ middleware ใน Python (Starlette) และ TypeScript (Express) พร้อมตัวอย่างโค้ด
  - **ก้าวสู่ความปลอดภัยขั้นสูง**: คำแนะนำสำหรับเริ่มต้นด้วยการพิสูจน์ตัวตนง่ายๆ และพัฒนาต่อไปสู่ OAuth 2.1 และ RBAC พร้อมอ้างอิงโมดูลความปลอดภัยขั้นสูง

การเพิ่มเติมเหล่านี้มอบคำแนะนำเชิงปฏิบัติสำหรับการสร้าง MCP เซิร์ฟเวอร์ที่แข็งแกร่ง ปลอดภัย และยืดหยุ่น รองรับแนวคิดพื้นฐานสู่รูปแบบการใช้งานขั้นสูงในสภาพแวดล้อมจริง

    - การตอบสนองและการกู้คืนเหตุการณ์ด้วยความสามารถอัตโนมัติ  
  - **ตัวอย่างการนำไปใช้**: เพิ่มบล็อกการกำหนดค่า YAML และตัวอย่างโค้ดอย่างละเอียด  
  - **การผสานรวมโซลูชันของ Microsoft**: ครอบคลุมบริการความปลอดภัย Azure, GitHub Advanced Security, และการจัดการตัวตนระดับองค์กร  

#### หัวข้อความปลอดภัยขั้นสูง (05-AdvancedTopics/mcp-security/) - การใช้งานระดับผลิตภัณฑ์  
- **README.md**: การเขียนใหม่ทั้งหมดสำหรับการใช้งานความปลอดภัยระดับองค์กร  
  - **สอดคล้องกับข้อกำหนดปัจจุบัน**: ปรับปรุงเป็น MCP Specification 2025-06-18 พร้อมข้อกำหนดความปลอดภัยที่บังคับใช้  
  - **การตรวจสอบสิทธิ์ขั้นสูง**: การผสานรวม Microsoft Entra ID พร้อมตัวอย่าง .NET และ Java Spring Security ครอบคลุม  
  - **การผสานรวมความปลอดภัย AI**: การใช้งาน Microsoft Prompt Shields และ Azure Content Safety พร้อมตัวอย่าง Python รายละเอียด  
  - **การบรรเทาภัยคุกคามขั้นสูง**: ตัวอย่างการนำไปใช้ฉบับสมบูรณ์สำหรับ  
    - การป้องกันการโจมตี Confused Deputy ด้วย PKCE และการตรวจสอบความยินยอมผู้ใช้  
    - การป้องกันการผ่านโทเค็นด้วยการตรวจสอบผู้รับและการจัดการโทเค็นอย่างปลอดภัย  
    - การป้องกันการแฮ็กเซสชันด้วยการผูกพันด้านการเข้ารหัสและการวิเคราะห์พฤติกรรม  
  - **การผสานรวมความปลอดภัยในองค์กร**: การตรวจสอบ Azure Application Insights, ท่อการตรวจจับภัยคุกคาม, และความปลอดภัยซัพพลายเชน  
  - **รายการตรวจสอบการใช้งาน**: การแยกแยะชัดเจนระหว่างการควบคุมความปลอดภัยที่บังคับใช้กับคำแนะนำ พร้อมประโยชน์ของระบบนิเวศความปลอดภัยของ Microsoft  

### คุณภาพเอกสารและการสอดคล้องมาตรฐาน  
- **การอ้างอิงข้อกำหนด**: ปรับปรุงอ้างอิงทั้งหมดเป็น MCP Specification 2025-06-18 ปัจจุบัน  
- **ระบบนิเวศความปลอดภัยของ Microsoft**: ปรับปรุงคำแนะนำการผสานรวมในเอกสารความปลอดภัยทั้งหมด  
- **การใช้งานเชิงปฏิบัติ**: เพิ่มตัวอย่างโค้ดละเอียดใน .NET, Java และ Python พร้อมรูปแบบองค์กร  
- **การจัดการแหล่งข้อมูล**: การจัดหมวดหมู่ที่ครอบคลุมของเอกสารอย่างเป็นทางการ, มาตรฐานความปลอดภัย, และคู่มือการใช้งาน  
- **ตัวชี้วัดภาพ**: การทำเครื่องหมายชัดเจนของข้อกำหนดที่บังคับและแนวปฏิบัติแนะนำ  

#### แนวคิดหลัก (01-CoreConcepts/) - การปรับปรุงสมัยใหม่เต็มรูปแบบ  
- **การอัปเดตเวอร์ชันโปรโตคอล**: ปรับปรุงให้ใช้อ้างอิง MCP Specification 2025-06-18 ที่มีการระบุวันที่ (รูปแบบ YYYY-MM-DD)  
- **การปรับแต่งสถาปัตยกรรม**: ขยายคำอธิบายของโฮสต์, ไคลเอนต์ และเซิร์ฟเวอร์เพื่อสะท้อนแบบแผนสถาปัตยกรรม MCP ปัจจุบัน  
  - โฮสต์ถูกกำหนดอย่างชัดเจนว่าเป็นแอป AI ที่ประสานงานการเชื่อมต่อไคลเอนต์ MCP หลายตัว  
  - ไคลเอนต์อธิบายว่าเป็นตัวเชื่อมโปรโตคอลที่รักษาความสัมพันธ์ เซิร์ฟเวอร์หนึ่งต่อหนึ่ง  
  - เซิร์ฟเวอร์ปรับปรุงให้รองรับสถานการณ์การปรับใช้แบบท้องถิ่นและระยะไกล  
- **การปรับโครงสร้าง Primitive**: ปรับปรุงใหม่ทั้งหมดของ primitives เซิร์ฟเวอร์และไคลเอนต์  
  - primitives เซิร์ฟเวอร์: แหล่งข้อมูล (ฐานข้อมูล), พรอมต์ (แม่แบบ), เครื่องมือ (ฟังก์ชันที่รันได้) พร้อมคำอธิบายและตัวอย่างละเอียด  
  - primitives ไคลเอนต์: การสุ่มตัวอย่าง (ผลสรุป LLM), การสอบถาม (ข้อมูลผู้ใช้), การบันทึก (ดีบัก/ตรวจสอบ)  
  - ปรับปรุงด้วยรูปแบบวิธีค้นหา (`*/list`), ดึงข้อมูล (`*/get`) และเรียกใช้งาน (`*/call`) ปัจจุบัน  
- **สถาปัตยกรรมโปรโตคอล**: นำเสนอโมเดลสถาปัตยกรรมสองชั้น  
  - ชั้นข้อมูล: พื้นฐาน JSON-RPC 2.0 พร้อมการจัดการวงจรชีวิตและ primitives  
  - ชั้นขนส่ง: STDIO (โลคัล) และ HTTP ที่รองรับการสตรีมด้วย SSE (ระยะไกล)  
- **กรอบความปลอดภัย**: หลักการความปลอดภัยอย่างครอบคลุม รวมถึงความยินยอมผู้ใช้ที่ชัดเจน, การปกป้องความเป็นส่วนตัวข้อมูล, ความปลอดภัยการรันเครื่องมือ, และความปลอดภัยชั้นขนส่ง  
- **รูปแบบการสื่อสาร**: ปรับปรุงข้อความโปรโตคอลเพื่อแสดงการเริ่มต้น, ค้นหา, เรียกใช้ และลำดับการแจ้งเตือน  
- **ตัวอย่างโค้ด**: ปรับปรุงตัวอย่างหลายภาษา (.NET, Java, Python, JavaScript) ให้สะท้อนรูปแบบ SDK MCP ปัจจุบัน  

#### ความปลอดภัย (02-Security/) - ปรับปรุงความปลอดภัยอย่างครบถ้วน  
- **การสอดคล้องกับมาตรฐาน**: สอดคล้องสมบูรณ์กับข้อกำหนดความปลอดภัย MCP Specification 2025-06-18  
- **วิวัฒนาการการตรวจสอบสิทธิ์**: บันทึกวิวัฒนาการจากเซิร์ฟเวอร์ OAuth แบบกำหนดเองสู่การมอบหมายผู้ให้บริการตัวตนภายนอก (Microsoft Entra ID)  
- **การวิเคราะห์ภัยคุกคามเฉพาะ AI**: ขยายเนื้อหาการโจมตีของ AI สมัยใหม่  
  - สถานการณ์โจมตีแบบแทรกแซงพรอมต์อย่างละเอียดพร้อมตัวอย่างในโลกจริง  
  - กลไกการวางยาพิษเครื่องมือและรูปแบบการโจมตี "rug pull"  
  - การวางยาพิษหน้าต่างบริบทและการโจมตีสับสนโมเดล  
- **โซลูชันความปลอดภัย AI ของ Microsoft**: ครอบคลุมระบบนิเวศความปลอดภัย Microsoft อย่างครอบคลุม  
  - AI Prompt Shields ด้วยการตรวจจับขั้นสูง, เทคนิคการไฮไลต์และการใช้ตัวคั่น  
  - รูปแบบการผสานรวม Azure Content Safety  
  - GitHub Advanced Security สำหรับการป้องกันซัพพลายเชน  
- **การบรรเทาภัยคุกคามขั้นสูง**: ควบคุมความปลอดภัยอย่างละเอียดสำหรับ  
  - การแฮ็กเซสชัน พร้อมสถานการณ์โจมตีเฉพาะ MCP และข้อกำหนด session ID แบบเข้ารหัส  
  - ปัญหา Confused deputy ในสถานการณ์พร็อกซี MCP พร้อมข้อกำหนดความยินยอมชัดเจน  
  - ช่องโหว่ token passthrough พร้อมการควบคุมการตรวจสอบความถูกต้องบังคับ  
- **ความปลอดภัยซัพพลายเชน**: ขยายครอบคลุมซัพพลายเชน AI รวมโมเดลพื้นฐาน, บริการฝังข้อมูล, ผู้ให้บริบท และ API ของบุคคลที่สาม  
- **ความปลอดภัยพื้นฐาน**: ผสานรวมปรับปรุงกับรูปแบบความปลอดภัยองค์กร เช่น สถาปัตยกรรม zero trust และระบบนิเวศความปลอดภัย Microsoft  
- **การจัดการแหล่งข้อมูล**: จัดกลุ่มลิงก์ทรัพยากรตามประเภท (เอกสารทางการ, มาตรฐาน, งานวิจัย, โซลูชัน Microsoft, คู่มือการใช้งาน)  

### การปรับปรุงคุณภาพเอกสาร  
- **วัตถุประสงค์การเรียนรู้ที่มีโครงสร้าง**: เพิ่มวัตถุประสงค์การเรียนรู้ที่เจาะจงและปฏิบัติได้  
- **การอ้างอิงไขว้**: เพิ่มลิงก์ระหว่างหัวข้อความปลอดภัยและแนวคิดหลักที่เกี่ยวข้อง  
- **ข้อมูลปัจจุบัน**: ปรับปรุงวันที่อ้างอิงและลิงก์ข้อกำหนดให้เป็นมาตรฐานปัจจุบัน  
- **คำแนะนำการใช้งาน**: เพิ่มแนวทางการใช้งานที่เฉพาะเจาะจงและปฏิบัติได้ในทั้งสองส่วน
