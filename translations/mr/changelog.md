#### मुख्य संकल्पना सुधारणा (01-CoreConcepts/)
- **नवीन क्लायंट प्रिमिटिव - रूटीज**: रूटीज क्लायंट प्रिमिटिवचे सखोल दस्तऐवज जोडले, जे सर्व्हरना फाइलसिस्टीम मर्यादा आणि प्रवेश परवानग्या समजण्यास सक्षम करतो
- **टूल स्पष्टीकरणे**: टूल वर्तनावर स्पष्टीकरणे (`readOnlyHint`, `destructiveHint`) जोडले ज्यामुळे टूल अंमलबजावणी निर्णय चांगले होतात
- **नमुनाकरणामध्ये टूल कॉलिंग**: नमुनाकरण दस्तऐवजात `tools` आणि `toolChoice` पर्यायांसह मॉडेल-आधारित टूल कॉलिंगसाठी अद्यतन केले
- **URL मोड उद्भवणे**: सर्व्हर-प्रेरित बाह्य वेब संवादासाठी URL-आधारित उद्भवणाच्या दस्तऐवजांची भर टाकली
- **टास्क्स (प्रायोगिक)**: टिकाऊ अंमलबजावणी व्रॅपर आणि विलंबित निकाल प्राप्तीसाठी नवीन टास्क्स वैशिष्ट्याचा प्रायोगिक विभाग जोडला
- **आयकॉन समर्थन**: टूल्स, संसाधने, संसाधन टेम्पलेट्स, आणि प्रम्प्ट्स आता अतिरिक्त मेटाडेटा म्हणून आयकॉन समाविष्ट करू शकतात

### प्रारंभिक विभाग विस्तार – प्रगत सर्व्हर वापर & सोपा प्रमाणीकरण

#### प्रगत सर्व्हर वापर (03-GettingStarted/10-advanced)
- **नवीन अध्याय जोडा**: नियमित आणि कमी पातळीच्या सर्व्हर अभियांत्रिकींचा विस्ताराने मार्गदर्शक सादर केला.
  - **नियमित विरुद्ध कमी-पातळी सर्व्हर**: दोन्ही दृष्टिकोनांसाठी Python आणि TypeScript मधील तपशीलवार तुलना व कोड उदाहरणे.
  - **हँडलर-आधारित डिझाइन**: स्केलेबल, लवचीक सर्व्हर अमलबजावणींसाठी हँडलर-आधारित टूल/संसाधन/प्रॉम्प्ट व्यवस्थापनाचे स्पष्टीकरण.
  - **प्रायोगिक नमुने**: प्रगत वैशिष्ट्ये आणि अभियांत्रिकी साठी कमी-पातळी सर्व्हर नमुन्यांच्या वास्तविक उपयोग प्रकरणे.

#### सोपा प्रमाणीकरण (03-GettingStarted/11-simple-auth)
- **नवीन अध्याय जोडा**: MCP सर्व्हरमध्ये साध्या प्रमाणीकरणाची टप्प्याटप्प्याने अमलबजावणी मार्गदर्शक.
  - **ऑथ संकल्पना**: प्रमाणीकरण आणि अधिकृतता यामधील फरक व क्रेडेन्शियल व्यवस्थापन स्पष्ट.
  - **मूलभूत ऑथ अमलबजावणी**: Python (Starlette) व TypeScript (Express) मध्ये मिडलवेअर-आधारित ऑथ नमुने, कोडसह.
  - **प्रगत सुरक्षा कडे प्रगती**: सोप्या प्रमाणीकरणापासून OAuth 2.1 आणि RBAC कडे मार्गदर्शन, प्रगत सुरक्षा मोड्यूल संदर्भांसह.

हे जोडणी प्रायोगिक, हाताळणी मार्गदर्शक देते ज्यामुळे अधिक मजबूत, सुरक्षित, आणि लवचीक MCP सर्व्हर अमलबजावणी तयार करता येतात, मूलभूत संकल्पनांपासून प्रगत उत्पादन नमुन्यांपर्यंत पुल बांधत.

- **README.md**: एंटरप्राइझ सुरक्षा अंमलबजावणीसाठी संपूर्ण पुनर्लेखन  
  - **सध्याचा स्पेसिफिकेशन संरेखण**: MCP स्पेसिफिकेशन 2025-06-18 नुसार अपडेट केले

  - **AI सुरक्षा समाकलन**: Microsoft Prompt Shields आणि Azure Content Safetyच्या तपशीलवार Python उदाहरणांसहित अंमलबजावणी

  - **अंमलबजावणी तपासणी सूची**: स्पष्ट अनिवार्य विरुद्ध शिफारस केलेले सुरक्षा नियंत्रण आणि Microsoft सुरक्षा परिसंस्थेचे फायदे

#### मुख्य संकल्पना (01-CoreConcepts/) - संपूर्ण आधुनिकीकरण  
- **प्रोटोकॉल आवृत्ती अद्यतन**: MCP स्पेसिफिकेशन 2025-06-18 संदर्भित केलं, तारीख-आधारित आवृत्तीकरण (YYYY-MM-DD स्वरूप)  
- **आर्किटेक्चर सुधारणा**: Hosts, Clients, आणि Servers चे वर्तमान MCP आर्किटेक्चर नमुन्यांसह वर्णन वाढवले  
  - Hosts आता स्पष्टपणे AI अनुप्रयोग म्हणून परिभाषित जे बहु-क्लायंट कनेक्शन्स समन्वयित करतात  
  - Clients एक ते एक सर्व्हर नाते राखणारे प्रोटोकॉल कनेक्टर्स म्हणून वर्णन  
  - Servers स्थानिक विरुद्ध दूरस्थ तैनाती परिस्थितींसह सुधारित  
- **प्रिमिटिव्ह पुनर्रचना**: सर्व्हर आणि क्लायंट प्रिमिटिव्हची संपूर्ण पुनर्रचना  
  - सर्व्हर प्रिमिटिव्ह: संसाधने (डेटा स्रोत), प्रॉम्प्ट्स (टेम्प्लेट्स), टूल्स (कार्यक्षमता कार्य) तपशीलवार स्पष्टीकरणे व उदाहरणांसह  
  - क्लायंट प्रिमिटिव्ह: सॅम्पलिंग (LLM पूर्णता), एलिसिटेशन (वापरकर्ता इनपुट), लॉगिंग (डिबगिंग/निरीक्षण)  
  - सध्याच्या शोध (`*/list`), पुनर्प्राप्ति (`*/get`), आणि अंमलबजावणी (`*/call`) पद्धतींचा वापर  
- **प्रोटोकॉल आर्किटेक्चर**: दोन-स्तरीय आर्किटेक्चर मॉडेलच्या ओळखी  
  - डेटा लेयर: JSON-RPC 2.0 पाया जोडिला, जीवनचक्र व्यवस्थापन आणि प्रिमिटिव्हसह  
  - ट्रान्सपोर्ट लेयर: STDIO (स्थानिक) आणि Streamable HTTP SSE सह (दूरस्थ) ट्रान्सपोर्ट मेकॅनिझम  
- **सुरक्षा फ्रेमवर्क**: स्पष्ट वापरकर्ता संमती, डेटा गोपनीयता संरक्षण, टूल अंमलबजावणी सुरक्षितता, आणि ट्रान्सपोर्ट लेयर सुरक्षा यांसह व्यापक सुरक्षा तत्त्वे  
- **संप्रेषण नमुने**: प्रारंभ, शोध, अंमलबजावणी आणि नोटिफिकेशन प्रवाह दाखविणारे प्रोटोकॉल मेसेजेस अद्यतनित  
- **कोड उदाहरणे**: .NET, Java, Python, JavaScript मध्ये सध्याच्या MCP SDK नमुन्यांनुसार ताजे केले  

#### सुरक्षा (02-Security/) - व्यापक सुरक्षा पुनर्रचना  
- **मानके संरेखण**: MCP स्पेसिफिकेशन 2025-06-18 सुरक्षा आवश्यकता पूर्णपणे समाविष्ट  
- **प्रमाणीकरण उत्क्रांती**: सानुकूल OAuth सर्व्हरपासून बाह्य ओळख प्रदाता (Microsoft Entra ID) प्रतिनिधत्वाकडे documented उत्क्रांती  
- **AI-विशिष्ट धोका विश्लेषण**: आधुनिक AI हल्ल्यांचे विस्तृत कव्हरेज  
  - वास्तविक उदाहरणांसह तपशीलवार प्रॉम्प्ट इंजेक्शन हल्ले  
  - टूल विषबाधा यंत्रणा आणि "रग पुल" हल्ले  
  - संदर्भ विंडो विषबाधा आणि मॉडेल गोंधळ हल्ले  
- **Microsoft AI सुरक्षा सोल्यूशन्स**: Microsoft सुरक्षा परिसंस्थेचे व्यापक कव्हरेज  
  - प्रगत शोध, स्पॉटलाईटिंग, आणि डेलिमीटर तंत्रासह AI Prompt Shields  
  - Azure Content Safety समाकलन नमुने  
  - पुरवठा साखळी संरक्षणासाठी GitHub Advanced Security  
- **प्रगत धोका प्रतिबंधन**:  
  - MCP-विशिष्ट हल्ल्यांसह सत्र हायजॅकिंग आणि क्रिप्टोग्राफिक सत्र आयडी आवश्यकता  
  - MCP प्रॉक्सी परिस्थितीत Confused Deputy समस्या आणि स्पष्ट संमती आवश्यकता  
  - टोकन पासथ्रू धोके आणि अनिवार्य पडताळणी नियंत्रण  
- **पुरवठा साखळी सुरक्षा**: फाउंडेशन मॉडेल्स, एम्बेडिंग सेवा, संदर्भ प्रदाते, आणि तृतीय-पक्ष API यांचा समावेश  
- **फाउंडेशन सुरक्षा**: झिरो ट्रस्ट आर्किटेक्चर आणि Microsoft सुरक्षा परिसंस्था समाकलन वाढविले  
- **संसाधन संघटना**: प्रकारानुसार (अधिकृत दस्तऐवज, मानके, संशोधन, Microsoft सोल्यूशन्स, अंमलबजावणी मार्गदर्शक) वर्गीकृत संसाधने
