  - **حلول أمان مايكروسوفت**: تم التحقق من المصطلحات الحالية والروابط لـ Prompt Shields (المعروف سابقًا بـ "كشف مخاطر كسر الحماية")، Azure Content Safety، Microsoft Entra ID، وAzure Key Vault
  - **أمان OAuth 2.1**: تم التأكد من التوافق مع أحدث أفضل ممارسات أمان OAuth

  - إطار عمل إدارة مخاطر الذكاء الاصطناعي من NIST
  - ISO 27001:2022
  - أفضل ممارسات أمان OAuth 2.1
  - أطر عمل الأمان والامتثال لـ Azure
- **موارد التنفيذ**: تم التحقق من جميع روابط وأدلة التنفيذ
  - أنماط مصادقة إدارة واجهات برمجة التطبيقات Azure API Management

#### المفاهيم الأساسية (01-CoreConcepts/) - تحديث شامل  
- **تحديث إصدار البروتوكول**: تحديث للإشارة إلى مواصفة MCP الحالية 2025-06-18 بصيغة تاريخ (YYYY-MM-DD)  
- **تحسين البنية المعمارية**: تعزيز وصف المضيفين، والعملاء، والخوادم لتعكس أنماط هندسة MCP الحالية  
  - تعريف واضح للمضيفين كتطبيقات ذكاء اصطناعي تنسق الاتصالات مع عملاء MCP متعددين  
  - وصف العملاء كمُوصلات للبروتوكول تحافظ على علاقات أحد لواحد مع الخادم  
  - تحسين وصف الخوادم حسب سيناريوات النشر المحلية عن بعد  
- **إعادة هيكلة البنى الأولية**: مراجعة كاملة لبنى الخادم والعميل  
  - البنى الأولية للخادم: الموارد (مصادر البيانات)، الحثّات (القوالب)، الأدوات (الدوال القابلة للتنفيذ) مع شروحات وأمثلة مفصلة  
  - البنى الأولية للعميل: أخذ العينات (اكتمالات LLM)، الاستدراج (مدخلات المستخدم)، التسجيل (التصحيح/المراقبة)  
  - تحديث أنماط الاكتشاف (`*/list`)، الاسترداد (`*/get`)، والتنفيذ (`*/call`)  
- **بنية البروتوكول**: تقديم نموذج معماري ذو طبقتين  
  - طبقة البيانات: أساس JSON-RPC 2.0 مع إدارة دورة الحياة والبنى الأولية  
  - طبقة النقل: STDIO (محلي) وStreamable HTTP مع SSE (عن بعد) كآليات نقل  
- **إطار الأمان**: مبادئ أمان شاملة تشمل موافقة المستخدم الصريحة، حماية خصوصية البيانات، أمان تنفيذ الأدوات، وأمان طبقة النقل  
- **أنماط الاتصال**: تحديث رسائل البروتوكول لعرض تدفقات التهيئة، الاكتشاف، التنفيذ، والإشعارات  
- **أمثلة الكود**: تحديث أمثلة متعددة اللغات (.NET، Java، Python، JavaScript) لتعكس أنماط SDK الحالية  

#### الأمان (02-Security/) - تحديث أمني شامل  
- **التوافق مع المعايير**: مطابقة كاملة لمتطلبات أمان مواصفة MCP 2025-06-18  
- **تطور المصادقة**: توثيق التطور من خوادم OAuth المخصصة إلى التفويض عبر موفري هوية خارجيين (Microsoft Entra ID)  
- **تحليل تهديدات ذكاء اصطناعي متخصص**: تغطية موسعة لموجهات هجوم الذكاء الاصطناعي الحديثة  
  - سيناريوهات هجوم حقن الحثّات مع أمثلة واقعية  
  - آليات تسمم الأدوات ونماذج هجوم "سحب السجاد"  
  - تسمم نوافذ السياق وهجمات تشويش النماذج  
- **حلول أمان الذكاء الاصطناعي من مايكروسوفت**: تغطية كاملة لمنظومة أمان مايكروسوفت  
  - AI Prompt Shields مع تقنيات الكشف المتقدمة، التركيز، وفواصل الحث  
  - أنماط تكامل Azure Content Safety  
  - GitHub Advanced Security لحماية سلسلة التوريد  
- **التخفيف المتقدم للتهديدات**: ضوابط أمان مفصلة لـ  
  - اختطاف الجلسة مع سيناريوهات هجوم MCP ومتطلبات التشفير لهوية الجلسة  
  - مشاكل الوكيل المضلل في سيناريوهات وكيل MCP مع متطلبات الموافقة الصريحة  
  - نقاط ضعف تمرير الرموز مع ضوابط تحقق إجبارية  
- **أمان سلسلة التوريد**: توسيع تغطية سلسلة توريد الذكاء الاصطناعي بما في ذلك نماذج الأساس، خدمات التضمين، مزودو السياق وواجهات برمجة تطبيقات الطرف الثالث  
- **أمن الأساس**: تعزيز التكامل مع أنماط الأمان المؤسسية بما في ذلك بنية الثقة الصفرية ومنظومة أمان مايكروسوفت  
- **تنظيم المصادر**: تصنيف شامل للروابط المصادر حسب النوع (الوثائق الرسمية، المعايير، الأبحاث، حلول مايكروسوفت، أدلّة التنفيذ)
