#### मुख्य अवधारणा सुधार (01-CoreConcepts/)  
- **नया क्लाइन्ट प्रिमेटिव - Roots**: मूल क्लाइन्ट प्रिमेटिवको बारेमा व्यापक कागजात थपियो, जसले सर्भरहरूलाई फाइल सिस्टम सीमाहरू र पहुँच अनुमति बुझ्न सक्षम बनाउँछ  
- **उपकरण टिप्पणिहरू**: उपकरण व्यवहारिक टिप्पणिहरू (`readOnlyHint`, `destructiveHint`) को कागजात थपियो, जसले उपकरण सञ्चालन निर्णय सुधार गर्छ  
- **साम्प्लिङमा उपकरण कल**: साम्प्लिङ कागजातलाई अद्यावधिक गरी मोडेल चालक उपकरण अनुरोधहरूमा `tools` र `toolChoice` प्यारामिटरहरू समावेश गरियो  
- **URL मोड कॉल**: सर्भरद्वारा आरम्भ गरिएका बाह्य वेब अन्तरक्रियाहरूका लागि URL आधारित संचालनको कागजात थपियो  
- **टास्कहरू (प्रयोगात्मक)**: दीर्घकालीन कार्यान्वयन रुप, र ढिला परिणाम निष्कासनका लागि प्रयोगात्मक टास्क सुविधा दस्तावेज गरिएको  
- **आइकन समर्थन**: उपकरणहरू, स्रोतहरू, स्रोत टेम्प्लेटहरू, र संकेतहरूले अब आइकनहरू थप मेटाडेटाको रूपमा समावेश गर्न सक्ने उल्लेख गरिएको

## डिसेम्बर १८, २०२५
### सुरक्षा दस्तावेज अद्यावधिक - MCP विशिष्टता 2025-11-25

#### MCP सुरक्षा सर्वोत्तम अभ्यासहरू (02-Security/mcp-best-practices.md) - विशिष्टता संस्करण अद्यावधिक
- **प्रोटोकल संस्करण अद्यावधिक**: नवीनतम MCP विशिष्टता 2025-11-25 (नोभेम्बर २५, २०२५ मा जारी) लाई संदर्भ गर्न अद्यावधिक गरिएको
  - सबै विशिष्टता संस्करण सन्दर्भहरू 2025-06-18 बाट 2025-11-25 मा अद्यावधिक गरियो
  - दस्तावेज मिति सन्दर्भहरू अगष्ट १८, २०२५ बाट डिसेम्बर १८, २०२५ मा अद्यावधिक गरियो
  - सबै विशिष्टता URL हरू वर्तमान दस्तावेजतर्फ संकेत गरिरहेका छन् भनी प्रमाणित गरियो
- **सामग्री प्रमाणीकरण**: नवीनतम मापदण्डहरू विरुद्ध सुरक्षा सर्वोत्तम अभ्यासहरूको व्यापक प्रमाणीकरण
  - **माइक्रोसफ्ट सुरक्षा समाधानहरू**: Prompt Shields (अघिल्लो "Jailbreak risk detection") , Azure Content Safety, Microsoft Entra ID, र Azure Key Vault का लागि वर्तमान शब्दावली र लिङ्कहरूको प्रमाणिकरण
  - **OAuth 2.1 सुरक्षा**: नवीनतम OAuth सुरक्षा सर्वोत्तम अभ्यासहरूसँग मेल भएकामा पुष्टि
  - **OWASP मापदण्डहरू**: LLM का लागि OWASP शीर्ष १० सन्दर्भहरू हालसालैको छन् भनी प्रमाणित
  - **Azure सेवा**: सबै माइक्रोसफ्ट Azure दस्तावेज लिंकहरू र सर्वोत्तम अभ्यासहरू पुष्टि गरियो
- **मापदण्ड समन्वय**: सबै उल्लेखित सुरक्षा मापदण्डहरू वर्तमान रहेको सुनिश्चित
  - NIST AI जोखिम व्यवस्थापन फ्रेमवर्क
  - ISO 27001:2022
  - OAuth 2.1 सुरक्षा सर्वोत्तम अभ्यासहरू
  - Azure सुरक्षा र अनुपालन फ्रेमवर्कहरू
- **कार्यान्वयन स्रोतहरू**: सबै कार्यान्वयन मार्गदर्शक लिंकहरू र स्रोतहरू प्रमाणित
  - Azure API व्यवस्थापन प्रमाणीकरण ढाँचाहरू
  - माइक्रोसफ्ट Entra ID एकीकरण मार्गदर्शकहरू
  - Azure Key Vault रहस्य व्यवस्थापन
  - DevSecOps पाइपलाइन र अनुगमन समाधानहरू

### दस्तावेज गुणस्तर आश्वासन
- **विशिष्टता पालना**: सबै अनिवार्य MCP सुरक्षा आवश्यकताहरू (MUST/MUST NOT) नयाँतम विशिष्टता अनुसार सुनिश्चित
- **स्रोत वर्तमानता**: माइक्रोसफ्ट दस्तावेज, सुरक्षा मापदण्डहरू, र कार्यान्वयन मार्गदर्शकहरूका सबै बाह्य लिङ्कहरू प्रमाणित
- **सर्वोत्तम अभ्यासहरूको कभरज**: प्रमाणीकरण, प्राधिकरण, AI-विशिष्ट खतरा, आपूर्ति श्रृंखला सुरक्षा, र उद्यम ढाँचाहरूको व्यापक कभरज पुष्टि

## अक्टोबर ६, २०२५

### सुरुवात गर्ने खण्ड विस्तार – उन्नत सर्भर प्रयोग र सरल प्रमाणीकरण

#### उन्नत सर्भर प्रयोग (03-GettingStarted/10-advanced)
- **नयाँ अध्याय थपियो**: नियमित र कम-स्तर सर्भर वास्तुकलासम्मिलित उन्नत MCP सर्भर प्रयोगको व्यापक मार्गदर्शक प्रस्तुत
  - **नियमित बनाम कम-स्तर सर्भर**: दुबै दृष्टिकोणका लागि Python र TypeScript मा विस्तृत तुलना र कोड उदाहरणहरू
  - **ह्यान्डलर-आधारित डिजाइन**: दायरामा, लचिलो सर्भर कार्यान्वयनका लागि ह्यान्डलर-आधारित उपकरण/स्रोत/प्रॉम्प्ट व्यवस्थापनको व्याख्या
  - **व्यावहारिक ढाँचाहरू**: उन्नत सुविधाहरू र वास्तुकलाका लागि कम-स्तर सर्भर ढाँचाहरू लाभप्रद हुने वास्तविक परिदृश्यहरू

#### सरल प्रमाणीकरण (03-GettingStarted/11-simple-auth)
- **नयाँ अध्याय थपियो**: MCP सर्भरहरूमा सरल प्रमाणीकरण कार्यान्वयन गर्न चरण-दर-चरण मार्गदर्शक प्रदान
  - **प्रमाणीकरण अवधारणाहरू**: प्रमाणीकरण र प्राधिकरणको स्पष्ट व्याख्या, र क्रेडेन्सियल ह्यान्डलिङ
  - **आधारभूत प्रमाणीकरण कार्यान्वयन**: Python (Starlette) र TypeScript (Express) मा मिडलवेयर-आधारित प्रमाणीकरण ढाँचाहरू, कोड नमूनासहित
  - **उन्नत सुरक्षा तर्फ प्रगतिलाई मार्गदर्शन**: सरल प्रमाणीकरणबाट OAuth 2.1 र RBAC तर्फ सुरू गर्न निर्देशन, उन्नत सुरक्षा मोड्युलहरूको सन्दर्भ सहित

यी थपहरूले थप दृढ, सुरक्षित र लचिलो MCP सर्भर कार्यान्वयन निर्माणका लागि व्यवहारिक, हात-मा-हात सल्लाह प्रदान गर्छन्, मुलभूत अवधारणाहरूसँग उन्नत उत्पादन ढाँचाहरूलाई जोड्दै।

  - Clients प्रोटोकल कन्टेक्टरहरू जसले एक-देखि-एक सर्भर सम्बन्ध राख्छन्  
  - Servers स्थानीय र दूरस्थ प्रयोग परिदृश्यहरूसँग सुधार गरियो  
- **प्रिमिटिभ पुनर्संरचना**: सर्भर र क्लाइन्ट प्रिमिटिभहरूको पूर्ण पुनर्गठन  
  - सर्भर प्रिमिटिभहरू: संसाधनहरू (डाटा स्रोतहरू), प्रॉम्प्टहरू (ढाँचा), उपकरणहरू (कार्यन्वयन योग्य कार्यहरू) विस्तृत व्याख्या र उदाहरणहरूसहित  
  - क्लाइन्ट प्रिमिटिभहरू: नमुना (LLM पूरा गर्ने), स्पष्टिकरण (प्रयोगकर्ता इनपुट), लगिङ (डिबगिङ/निगरानी)  
  - हालको खोज (`*/list`), प्राप्ति (`*/get`), र कार्यान्वयन (`*/call`) विधि ढाँचासँग अद्यावधिक  
- **प्रोटोकल वास्तुकला**: दुई-स्तर वास्तुकला मोडेल  
  - डाटा स्तर: JSON-RPC 2.0 आधार, जीवनचक्र व्यवस्थापन र प्रिमिटिभहरूसहित  
  - ट्रान्सपोर्ट स्तर: STDIO (स्थानीय) र Streamable HTTP सँग SSE (दूरस्थ) ट्रान्सपोर्ट विधिहरू  
- **सुरक्षा फ्रेमवर्क**: स्पष्ट प्रयोगकर्ता सहमति, डाटा गोपनीयता संरक्षण, उपकरण निष्पादन सुरक्षा, र ट्रान्सपोर्ट स्तर सुरक्षा सहित व्यापक सुरक्षा सिद्धान्तहरू  
- **सञ्चार ढाँचाहरू**: प्रोटोकल सन्देशहरू प्रारम्भ, खोज, कार्यान्वयन, र सूचनाप्रवाहहरू प्रदर्शन गरी अद्यावधिक

  - GitHub Advanced Security को आपूर्ति श्रृंखला संरक्षण  
- **उन्नत खतरा कम गर्ने उपायहरू**: विस्तृत सुरक्षा नियन्त्रणहरू

  - स्पष्ट सहमति आवश्यकतासहित MCP प्रोक्सी परिदृश्यहरूमा भ्रमित डिप्युटी समस्याहरू  
  - अनिवार्य मान्यता नियन्त्रणसहित टोकन पासथ्रू कमजोरताहरू  
- **आपूर्ति श्रृंखला सुरक्षा**: आधारभूत मोडेलहरू, इम्बेडिङ सेवा, सन्दर्भ प्रदायकहरू, र तेस्रो-पक्ष API सहित AI आपूर्ति श्रृंखला कभरेज विस्तार

- SUPPORT.md सेटअप गरियो, सहायता प्राप्तिका लागि निर्देशनहरू सहित
