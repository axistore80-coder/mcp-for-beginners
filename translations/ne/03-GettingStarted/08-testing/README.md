## टेस्टिङ र डिबगिङ

तपाईंले आफ्नो MCP सर्भर परीक्षण गर्न सुरु गर्नु अघि, उपलब्ध उपकरणहरू र डिबगिङको लागि उत्तम अभ्यासहरू बुझ्नु महत्त्वपूर्ण छ। प्रभावकारी परीक्षणले तपाईंको सर्भरले अपेक्षित रूपमा काम गर्छ भन्ने सुनिश्चित गर्छ र तपाईंलाई छिटो समस्याहरू पत्ता लगाउन र समाधान गर्न मद्दत गर्दछ। तलको खण्डले तपाईंको MCP कार्यान्वयनलाई मान्य गर्ने सिफारिस गरिएका दृष्टिकोणहरूलाई वर्णन गर्दछ।

## अवलोकन

यो पाठले सही परीक्षण दृष्टिकोण कसरी छनौट गर्ने र सबैभन्दा प्रभावकारी परीक्षण उपकरण कुन हो भन्ने कुरा समेट्छ।

## सिकाइका उद्देश्यहरू

यो पाठको अन्त्यसम्म, तपाईं यी गर्न सक्षम हुनुहुनेछ:

- परीक्षणका विभिन्न दृष्टिकोणहरू वर्णन गर्न।
- तपाईंको कोडलाई प्रभावकारी रूपमा परीक्षण गर्नका लागि विभिन्न उपकरणहरू प्रयोग गर्न।

## MCP सर्भरहरूको परीक्षण

MCP ले तपाईंको सर्भरहरू परीक्षण र डिबग गर्न मद्दत गर्ने उपकरणहरू प्रदान गर्दछ:

- **MCP Inspector**: कमाण्ड लाइन उपकरण जुन CLI उपकरणको रूपमा र भिजुअल उपकरणको रूपमा दुबै चलाउन सकिन्छ।
- **म्यानुअल परीक्षण**: तपाईं curl जस्तो टुल प्रयोग गरेर वेब अनुरोधहरू रन गर्न सक्नुहुन्छ, तर कुनै पनि HTTP चलाउन सक्षम टुल चले हुन्छ।
- **युनिट परीक्षण**: तपाईंले आफ्नो मनपर्ने परीक्षण फ्रेमवर्क प्रयोग गरेर सर्भर र क्लाइन्ट दुबैका सुविधाहरू परीक्षण गर्न सक्नुहुन्छ।

### MCP Inspector को प्रयोग

हामीले अघिल्लो पाठहरूमा यस उपकरणको प्रयोग वर्णन गरिसकेका छौं तर यसलाई केही उच्च स्तरमा कुरा गरौं। यो Node.js मा बनेको उपकरण हो र तपाईंले `npx` एग्जिक्युटेबल कल गरेर यसलाई प्रयोग गर्न सक्नुहुन्छ जसले त्यो उपकरणलाई अस्थायी रूपमा डाउनलोड र इन्स्टल गर्नेछ र तपाईंको अनुरोध चलिसकेपछि सफा पनि गर्नेछ।

[MCP Inspector](https://github.com/modelcontextprotocol/inspector) तपाईंलाई मद्दत गर्दछ:

- **सर्भर क्षमताहरू पत्ता लगाउने**: उपलब्ध स्रोतहरू, उपकरणहरू, र प्रम्प्टहरू स्वचालित रूपमा पत्ता लगाउने
- **उपकरण कार्यान्वयन परीक्षण गर्ने**: विभिन्न प्यारामिटरहरू प्रयास गर्ने र प्रतिक्रियाहरू वास्तविक समयमा हेर्ने
- **सर्भर मेटाडेटा हेर्ने**: सर्भर जानकारी, स्कीमा, र कन्फिगरेसनहरू जाँच गर्ने

टुलको सामान्य सञ्चालन यसरी देखिन्छ:

```bash
npx @modelcontextprotocol/inspector node build/index.js
```
  
माथिको कमाण्डले एउटा MCP र यसको भिजुअल इन्टरफेस सुरु गर्दछ र तपाईंको ब्राउजरमा स्थानीय वेब इन्टरफेस लन्च गर्दछ। तपाईंले पंजीकृत MCP सर्भरहरूको ड्यासबोर्ड देख्न सक्नुहुनेछ, तिनीहरूका उपलब्ध उपकरणहरू, स्रोतहरू, र प्रम्प्टहरू। यो इन्टरफेसले तपाईंलाई अन्तरक्रियात्मक रूपमा उपकरण कार्यान्वयन परीक्षण गर्न, सर्भर मेटाडेटा जाँच गर्न, र वास्तविक समय प्रतिक्रियाहरू हेर्न अनुमति दिन्छ, जसले तपाईंको MCP सर्भर कार्यान्वयनहरू मान्य र डिबग गर्न सहज बनाउँछ।

यसले यस्तो देखिन सक्छ: ![Inspector](../../../../translated_images/ne/connect.141db0b2bd05f096.webp)

तपाईं यो उपकरण CLI मोडमा पनि चलाउन सक्नुहुन्छ, यस अवस्थामा तपाईंले `--cli` एट्रिब्युट थप्नुहुनेछ। यहाँ "CLI" मोडमा उपकरण चलाउनको लागि एउटा उदाहरण छ जुन सर्भरमा रहेका सबै उपकरणहरू सूचीबद्ध गर्छ:

```sh
npx @modelcontextprotocol/inspector --cli node build/index.js --method tools/list
```
  
### म्यानुअल परीक्षण

सर्भर क्षमताहरू परीक्षण गर्न इन्स्पेक्टर टूल चलाउने बाहेक, अर्को समान विधि भनेको HTTP प्रयोग गर्न सक्ने क्लाइन्ट चलाउनु हो जस्तै curl।

curl प्रयोग गरेर, तपाईं MCP सर्भरहरूलाई सिधै HTTP अनुरोधहरू मार्फत परीक्षण गर्न सक्नुहुन्छ:

```bash
# उदाहरण: परीक्षण सर्भर मेटाडाटा
curl http://localhost:3000/v1/metadata

# उदाहरण: उपकरण चलाउनुहोस्
curl -X POST http://localhost:3000/v1/tools/execute \
  -H "Content-Type: application/json" \
  -d '{"name": "calculator", "parameters": {"expression": "2+2"}}'
```
  
माथिको curl प्रयोगबाट देखिन्छ कि तपाईं उपकरणको नाम र त्यसका प्यारामिटरहरू समावेश गर्ने पेलोड प्रयोग गरेर POST अनुरोध पठाउनुहुन्छ। तपाईंलाई सबैभन्दा अनुकूल हुने तरिका प्रयोग गर्नुहोस्। CLI उपकरणहरू सामान्यतया छिटो प्रयोग गर्न सकिन्छ र स्क्रिप्ट गर्न सजिलो हुन्छ जसले CI/CD वातावरणमा उपयोगी हुन सक्छ।

### युनिट परीक्षण

तपाईंका उपकरण र स्रोतहरूको युनिट टेस्टहरू सिर्जना गरेर तपाईंले तिनीहरू अपेक्षित रूपमा काम गर्छन् भन्ने सुनिश्चित गर्नुहोस्। यहाँ केही उदाहरण परीक्षण कोड छ।

```python
import pytest

from mcp.server.fastmcp import FastMCP
from mcp.shared.memory import (
    create_connected_server_and_client_session as create_session,
)

# सम्पूर्ण मोड्युललाई एसिंक्रोनस परीक्षणहरूको लागि चिन्ह लगाउनुहोस्
pytestmark = pytest.mark.anyio


async def test_list_tools_cursor_parameter():
    """Test that the cursor parameter is accepted for list_tools.

    Note: FastMCP doesn't currently implement pagination, so this test
    only verifies that the cursor parameter is accepted by the client.
    """

 server = FastMCP("test")

    # दुईवटा परीक्षण उपकरणहरू सिर्जना गर्नुहोस्
    @server.tool(name="test_tool_1")
    async def test_tool_1() -> str:
        """First test tool"""
        return "Result 1"

    @server.tool(name="test_tool_2")
    async def test_tool_2() -> str:
        """Second test tool"""
        return "Result 2"

    async with create_session(server._mcp_server) as client_session:
        # कर्सर प्यारामिटर बिना परीक्षण गर्नुहोस् (छोडिएको)
        result1 = await client_session.list_tools()
        assert len(result1.tools) == 2

        # cursor=None सँग परीक्षण गर्नुहोस्
        result2 = await client_session.list_tools(cursor=None)
        assert len(result2.tools) == 2

        # स्ट्रिङको रूपमा कर्सरसँग परीक्षण गर्नुहोस्
        result3 = await client_session.list_tools(cursor="some_cursor_value")
        assert len(result3.tools) == 2

        # खाली स्ट्रिङ कर्सरसँग परीक्षण गर्नुहोस्
        result4 = await client_session.list_tools(cursor="")
        assert len(result4.tools) == 2
    
```
  
माथिको कोडले निम्न गर्दछ:

- pytest फ्रेमवर्क प्रयोग गर्दछ जसले तपाईंलाई परीक्षणहरू कार्यहरूका रूपमा सिर्जना गर्न र assert स्टेटमेन्टहरू प्रयोग गर्न अनुमति दिन्छ।
- दुई फरक उपकरणहरू सहितको MCP सर्भर सिर्जना गर्दछ।
- निश्चित अवस्थामा पूरा भएको छ कि छैन भनेर जाँच गर्न `assert` स्टेटमेन्ट प्रयोग गर्दछ।

[यहाँ पूरा फाइल हेर्नुहोस्](https://github.com/modelcontextprotocol/python-sdk/blob/main/tests/client/test_list_methods_cursor.py)

माथिको फाइलको आधारमा, तपाईंले आफ्नो सर्भरलाई परीक्षण गर्न सक्नुहुन्छ र क्षमताहरू ठीकसँग सिर्जना भएका छन् कि छैनन् भनेर सुनिश्चित गर्न सक्नुहुन्छ।

सबै प्रमुख SDK हरूमा यस्तै परीक्षण खण्डहरू हुन्छन् ताकि तपाईं आफ्नो रोजाइको रनटाइम अनुरूप मिलाउन सक्नुहुनेछ।
