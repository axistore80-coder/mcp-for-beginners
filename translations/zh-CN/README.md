# 🚀 面向初学者的模型上下文协议 (MCP) 课程

## **通过 C#、Java、JavaScript、Rust、Python 和 TypeScript 的动手示例学习 MCP**

## 🧠 模型上下文协议课程概述
欢迎踏上模型上下文协议（MCP）的学习之旅！如果你曾好奇 AI 应用程序如何与不同的工具和服务通信，你即将发现一种优雅的解决方案，正在改变开发者构建智能系统的方式。

把 MCP 想象成 AI 应用的通用翻译器——就像 USB 端口让你可以将任何设备连接到电脑一样，MCP 让 AI 模型以标准化的方式连接到任何工具或服务。无论你是在构建第一个聊天机器人还是在处理复杂的 AI 工作流，理解 MCP 都能赋予你创建更强大、更灵活应用的能力。

本课程以耐心和细致的方式设计，适合你的学习旅程。我们将从你已经理解的简单概念开始，通过你喜爱的编程语言进行动手实践，逐步提升你的技能。每一步都包含清晰的解释、实用的示例和充分的鼓励支持。

完成这段旅程后，你将有信心构建自己的 MCP 服务器，将它们集成到流行的 AI 平台中，并理解这项技术如何重塑 AI 开发的未来。让我们一起开启这段激动人心的冒险吧！

本课程基于 **MCP 规范 2025-11-25**（最新稳定版本）。MCP 规范使用基于日期的版本号（格式为 YYYY-MM-DD），以确保协议版本的清晰追踪。

随着你理解的加深，这些资源将变得更加有价值，但不需要你一开始就全部阅读。请从你最感兴趣的部分开始！
- 📘 [MCP 文档](https://modelcontextprotocol.io/) – 这是你按步骤教程和用户指南的首选资源。文档针对初学者编写，提供清晰的示例，你可以按自己的节奏跟随。
- 📜 [MCP 规范](https://modelcontextprotocol.io/specification/2025-11-25) – 把它当作你全面的参考手册。随着你推进课程，你会经常回来看具体细节和高级功能。
- 📜 [MCP 规范版本管理](https://modelcontextprotocol.io/specification/versioning) – 包含协议版本历史和 MCP 如何使用基于日期的版本号的信息。
- 🧑‍💻 [MCP GitHub 仓库](https://github.com/modelcontextprotocol) – 这里有多种编程语言的 SDK、工具和代码示例。就像是一个充满实用示例和即用组件的宝库。
- 🌐 [MCP 社区](https://github.com/orgs/modelcontextprotocol/discussions) – 加入其他学习者和经验丰富的开发者，参与 MCP 的讨论。这是一个支持性强的社区，欢迎提问和知识自由分享。
  
## 学习目标

完成本课程后，你会感到自信和兴奋，掌握以下能力：

• **理解 MCP 基础**：你会掌握什么是模型上下文协议，以及为什么它正在革新 AI 应用的协同工作原理，通过类比和示例让概念易于理解。

• **构建你的第一个 MCP 服务器**：你将使用自己喜欢的编程语言创建一个可工作的 MCP 服务器，从简单示例开始，一步步提升技能。

• **连接 AI 模型与真实工具**：你将学习如何架起 AI 模型与实际服务之间的桥梁，赋予应用强大的新功能。

• <strong>实施安全最佳实践</strong>：你将了解如何保持 MCP 实现的安全，保护你的应用和用户。

• <strong>自信部署</strong>：你将掌握如何将 MCP 项目从开发推向生产环境，采用切实可行的部署策略。

• **加入 MCP 社区**：你将成为一个不断壮大的开发者社区的一员，共同塑造 AI 应用开发的未来。

## 重要背景知识

在深入 MCP 细节之前，让我们确保你对一些基础概念感到熟悉。别担心如果你不是这方面的专家——我们会逐步解释你需要知道的所有内容！

### 理解协议（基础）

把协议想象成对话的规则。当你给朋友打电话时，你们都知道接电话时要说“你好”，轮流说话，结束时说“再见”。计算机程序也需要类似规则才能有效沟通。

MCP 是一种协议——一套双方同意的规则，帮助 AI 模型和应用程序与工具及服务进行有效“对话”。就像对话规则让人类交流更顺畅一样，MCP 让 AI 应用通信更可靠、更强大。

### 客户端-服务器关系（程序如何协作）

你每天都在使用客户端-服务器关系！当你使用网页浏览器（客户端）访问网站时，你就是连接到了提供页面内容的网页服务器。浏览器知道如何请求信息，服务器知道如何响应。

在 MCP 中，我们有类似的关系：AI 模型作为客户端发出请求或操作，而 MCP 服务器则提供这些功能。这就像有个帮手（服务器），AI 可以让它执行特定任务。

### 标准化为何重要（让事物互通）

想象如果每个汽车制造商使用不同形状的加油枪——你得为每辆车准备不同的适配器！标准化就是达成共识，采用共同的方式，让各部分无缝协作。

MCP 为 AI 应用提供了这种标准化。无需每个 AI 模型都为每个工具写定制代码，MCP 创建了一个通用的通信方式。这意味着开发者可以构建工具一次，让它们适配多种 AI 系统。

## 🧭 你的学习路径概览

你的 MCP 学习之旅被精心设计，逐步建立你的自信和技能。每个阶段引入新概念，同时巩固已有知识。

### 🌱 基础阶段：理解基础 (模块 0-2)

你的冒险从这里开始！我们将用熟悉的类比和简单示例介绍 MCP 概念。你会理解 MCP 是什么，它为何存在，以及它在更广泛 AI 开发中的角色。

• **模块 0 - MCP 介绍**：我们将从探讨 MCP 是什么以及它为何对现代 AI 应用如此重要开始。你将看到 MCP 的真实案例，理解它如何解决开发者常见的问题。

• **模块 1 - 核心概念详解**：这里你将学习 MCP 的基本构件。我们会用大量类比和视觉示例，确保概念自然易懂。

• **模块 2 - MCP 中的安全**：安全听起来可能令人望而生畏，但我们会展示 MCP 内建的安全特性，并教你最佳实践，保护你的应用从一开始就安全。

### 🔨 构建阶段：创建你的第一个实现 (模块 3)
现在真正的乐趣开始了！你将获得构建实际 MCP 服务器和客户端的动手经验。别担心——我们会从简单开始，指导你完成每一步。

本模块包含多个动手指南，让你可以用自己喜欢的编程语言练习。你将创建第一个服务器，构建连接它的客户端，甚至集成流行的开发工具如 VS Code。

每个指南都包括完整的代码示例、故障排除技巧，以及我们为何做出特定设计选择的解释。完成本阶段后，你将拥有可骄傲的 MCP 实现！

### 🚀 成长阶段：高级概念与现实应用（模块 4-5）

掌握基础后，你将准备探索更复杂的 MCP 功能。我们将讲解实用的实现策略、调试技巧以及多模态 AI 集成等高级主题。

你还将学习如何扩展 MCP 实现以支持生产使用，并与 Azure 等云平台集成。这些模块将让你能够构建应对现实需求的 MCP 解决方案。

### 🌟 精通阶段：社区与专业化（模块 6-11）

最后阶段聚焦于加入 MCP 社区并专注于你最感兴趣的领域。你将学习如何为开源 MCP 项目做贡献，实现高级认证模式，并构建集成数据库的完整解决方案。

第 11 模块尤为值得一提——它是一个包含 13 个实验室的完整动手机学习路径，教你构建集成 PostgreSQL 的生产级 MCP 服务器。它就像一个综合项目，将你所学汇聚一处！
