# ပြောင်းလဲမှုမှတ်တမ်းများ: MCP စတင်သင်ယူသူများအတွက် သင်ရိုးညွှန်းတမ်း

ဤစာတမ်းသည် Model Context Protocol (MCP) စတင်သင်ယူသူများအတွက် သင်ရိုးညွှန်းတမ်းတွင် ပြုလုပ်ထားသည့် အရေးကြီးသောပြောင်းလဲမှုများအားလုံးကို မှတ်သားထားသည့် မှတ်တမ်းဖြစ်သည်။ ပြောင်းလဲမှုများကို အချိန်ရက်စဥ် လျှောက်လွန်စဉ် အမြောက်အမြားအတိုင်း (နောက်ဆုံးပြောင်းလဲမှုများကို ဦးစွာ) မှတ်တမ်းတင်ထားသည်။

## ၂၀၂၆ ခုနှစ်၊ ဧပြီလ ၁၁ ရက်

### သင်ခန်းစာအသစ်၊ စာရွက်စာတမ်းပြင်ဆင်မှုပြုခြင်းနှင့် မူလလက်ခံထောက်ပံ့မှုများ အပ်ဒိတ်လုပ်ခြင်း

#### သင်ရိုးအသစ် ထည့်သွင်းခြင်း

**Module 05 - အဆင့်မြင့်ခေါင်းစဉ်များ**
- **Lesson 5.17: MCP နှင့်အတူ ဆန့်ကျင်ဘက် မူလတန်းကြီးအမြီးမြား အမြင်အာရုံ ပြဿနာဖြေရှင်းခြင်း** (`05-AdvancedTopics/mcp-adversarial-agents/README.md`): အမြီးမြားစနစ်များအတွက် ဆန့်ကျင်ဘက်ဆွေးနွေးမှု ပုံစံကို ကျယ်ပြန့်စွာ ဖော်ပြထားသော လမ်းညွှန်စာတမ်း အသစ်
  - Mermaid ဖွဲ့စည်းပုံ ကိုးကားရေးရုပ်ပုံ: လူနှစ်ဦး → MCP ဆာဗာမျှဝေ → ဆွေးနွေးမှု မှတ်တမ်း → တရားသူကြီး → ဆုံးဖြတ်ချက်
  - Python နှင့် TypeScript တွင် ဆမ်းနိုင်သော MCP ကိရိယာဆာဗာ (`web_search` + `run_python`)
  - ဆန့်ကျင်ဘက် စနစ် အမိန့်များ (FOR / AGAINST / တရားသူကြီး) - ကိရိယာ အသုံးပြုမှု လိုအပ်ချက် အတိအကျ ဖော်ပြမှု
  - Python၊ TypeScript နှင့် C# ဖြင့် ဆွေးနွေးမှု စီမံခန့်ခွဲသူ - လှည့်များစီစဉ်ခြင်းနှင့် အကြောင်းပြချက်များ လမ်းညွှန်ခြင်း
  - MCP `ClientSession` ကို စစ်ဆေးမှုဖန်တီးသူ (orchestrator) အတွက် တိုက်ရိုက် ကိရိယာခေါ်ယူမှုဖြင့် တွဲစပ်ခြင်း
  - အသုံးပြုမှု အတွင်း-အရေးကြီးအမှုစာရင်း (ဟယ်လူစင်အတုစစ်ဆေးမှု၊ အန္တရာယ် မော်ဒယ်နိုင်ရန်၊ API ဒီဇိုင်းပြန်လည်သုံးသပ်ခြင်း၊ အတိအကျ အတည်ပြုခြင်း၊ နည်းပညာ ရွေးချယ်မှု)
  - လုံခြုံရေးစဉ်းစားချက်များ- စက်ဝိုင်းလွတ်ကွင်းစိတ်ချရသော အကောင်အထည်ဖော်ခြင်း၊ ကိရိယာခေါ်ဆိုမှု ကိုစစ်ဆေးခြင်း၊ ခြေလှမ်းကန့်သတ်ခြင်း၊ စစ်ဆေးမှတ်တမ်းတင်ခြင်း
  - လေ့ကျင့်ခန်း တည်ဆောက်မှု - လက်တွေ့ ရှိသည့် သုံးဆင့်စီစဉ်ချက်များ (ကုဒ်ဆန်းစစ်ခြင်း၊ ဖွဲ့စည်းပုံ ဆုံးဖြတ်ချက်ချခြင်း၊ ပါဝင်မှု စစ်ဆေးမှု)

#### ကုဒ်နှင့် လုံခြုံရေး ပြင်ဆင်မှုများ

**Module 05 - ဆန့်ကျင်ဘက် အမြီးမြား (`mcp-adversarial-agents`)**
- **လုံခြုံရေး ပြင်ဆင်မှု — command injection**: TypeScript `run_python` ကိရိယာတွင် `execSync` shell interpolation ကို `execFile` + `promisify` ဖြင့် အစားထိုးခဲ့ပြီး command injection ရှိနိုင်မှု ရှင်းလင်း (LLM က အမိန့်များ literal argv အဖြစ် shell ပါဝင်မှုမရှိဘဲ ထည့်သွင်းပြီးဖြစ်သည်)
- **MCP tool loop wiring**: Python ဆွေးနွေးမှု စီမံခန့်ခွဲသူအား `AsyncAnthropic` client ဖြင့် အသစ်ပြောင်းလဲ (blocking sync `Anthropic` မှ ပြောင်းလဲ), agent တစ်ဦးချင်းဆီသို့ တိုက်ရိုက် `ClientSession` လွှဲပြောင်းပေးခြင်း, တစ်ယောက်ချင်း `session.list_tools()` ဖြင့် ကိရိယာအချက်အလက်များယူခြင်းနှင့် `session.call_tool()` ဖြင့် `tool_use` block များကို လည်ပတ်စေရာ loop တွင် ထုတ်ပြန်ခဲ့သည်။

## ၂၀၂၆ ခုနှစ်၊ ဇန်နဝါရီ ၂၈ ရက်

### MCP Specification 2025-11-25 အညီကျမှု ပြန်လည်သုံးသပ်ခြင်း

#### အဓိက အယူအဆ တိုးချဲ့မှု (01-CoreConcepts/)
- **Client Primitive အသစ် - Roots**: Filesystem အကန့်အသတ်များနှင့် ခွင့်ပြုချက်များကို ဆာဗာများနားလည်နိုင်ရန် Roots client primitive အကြောင်း အသေးစိတ် စာရွက်စာတမ်း ထည့်သွင်း
- **ကိရိယာ အကြောင်းပြချက်ဆိုင်ရာ မှတ်ချက်များ**: `readOnlyHint`, `destructiveHint` ကိရိယာ လုပ်ဆောင်မှု ဆုံးဖြတ်ချက်များအတွက် မှတ်ချက် ထည့်သွင်းမှုအကြောင်း စာရွက်စာတမ်း
- **Sampling အတွင်း ကိရိယာခေါ်ဆိုခြင်း**: Sampling စာရွက်စာတမ်းအား ကိရိယာများနှင့် `toolChoice` parameter များ ထည့်သွင်း၍ မော်ဒယ်မှ တောင်းဆိုချက်များတွင် ကိရိယာခေါ်ဆိုမှု
- **URL Mode ပုံစံ**: ဆာဗာမှ ပြင်ပဝဘ်လုပ်ဆောင်ချက်များကို စတင်စေသော URL အခြေခံ စနစ်အကြောင်း ရေးသား
- **လုပ်ငန်းများ (အတွေ့အကြုံဆိုင်ရာ)**: ခိုင်ခံ့သော အကောင်အထည်ဖော်မှု နှင့် deferred ရလဒ် ရယူမှု အတွက် Experimental Tasks အပိုင်း အသစ် ထည့်သွင်း
- **သင်္ကေတများ ထောက်ပံ့မှု**: ကိရိယာများ၊ အရင်းအမြစ်များ၊ အရင်းအမြစ် ရေးစနစ်များ နှင့် မေးခွန်းများတွင် သင်္ကေတများ ထည့်သွင်းနိုင်သည်ဆိုသည်ကို မှတ်သား

#### စာရွက်စာတမ်း အသစ်များ ထည့်သွင်းခြင်း
- **README.md**: MCP Specification 2025-11-25 ဗားရှင်း ကိုးကားချက်နှင့် ရက်စွဲအပေါ် အခြေခံ ဗားရှင်းရေးသားခြင်း ဖော်ပြ
- **study_guide.md**: Core Concepts အပိုင်းတွင် Tasks နှင့် ကိရိယာ အကြောင်းပြချက်များ ထည့်သွင်းထားသော သင်ရိုးဇယား ပြင်ဆင်; စာရွက်စာတမ်း ရက်စွဲ အသစ်ပြင်ဆင်

#### သတ်မှတ်ချက် အညီကျမှု အတည်ပြုခြင်း
- **Protocol ဗားရှင်း**: MCP Specification 2025-11-25 ကို အားလုံး လိုက်နာ
- **ဖွဲ့စည်းပုံ ကိုက်ညီမှု**: အဆင့်နှစ်ဆင့် ဖွဲ့စည်းပုံ (Data Layer + Transport Layer)
- **Primitive များအကြောင်း စာရွက်စာတမ်း**: ဆာဗာ primitives (Resources, Prompts, Tools) နှင့် client primitives (Sampling, Elicitation, Logging, Roots)
- **သယ်ယူပို့ဆောင်မှု များ**: STDIO နှင့် Streamable HTTP ဖော်ပြချက်များ သေချာမှု
- **လုံခြုံရေး လမ်းညွှန်ချက်များ**: MCP လုံခြုံရေး အကောင်းဆုံး လုပ်ထုံးလုပ်နည်းနှင့် ကိုက်ညီမှု

#### MCP 2025-11-25 ဦးတည်ချက်အဓိက အချက်များ
- **OpenID Connect Discovery**: Auth ဆာဗာ OIDC အခြေခံ ရှာဖွေမှု
- **OAuth Client ID Metadata စာရွက်စာတမ်းများ**: client မှတ်ပုံတင်ခြင်း လမ်းညွှန်မှု အကြံပြုချက်
- **JSON Schema 2020-12**: MCP schema သတ်မှတ်ချက်မ်ားအတွက် ပုံမှန် သုံးစွဲခြင်း
- **SDK Tiering စနစ်**: SDK လုပ်ဆောင်ချက်ထောက်ပံ့မှုနှင့် ပြုပြင် ထိန်းသိမ်းမှု လိုအပ်ချက်များ အဆင့်ကွဲပြား
- **အုပ်ချုပ်မှု ဖွဲ့စည်းပုံ**: MCP တာဝန်ရှိ Working Groups နှင့် သဘောတူကြောင်း ဂိုဏ်းများ ဖော်ပြ

  - ၂၀၂၅-၀၆-၁၈ မှ ၂၀၂၅-၁၁-၂၅ အထိ အသေးစိတ်ဖော်ပြချက်ဗားရှင်း ကိုးကားမှုအားလုံးကိုအပ်ဒိတ်လုပ်ခဲ့သည်
  - ၂၀၂၅ ခုနှစ်၊ ဩဂုတ်လ ၁၈ ရက်မှ ဒီဇင်ဘာ ၁၈ ရက် ၂၀၂၅ အထိ စာရွက်စာတမ်းရက်စွဲကိုအပ်ဒိတ်လုပ်ခဲ့သည်
  - အသေးစိတ်ဖော်ပြချက် URL များအားလုံးသည် လတ်တလောစာရွက်စာတမ်းများသို့ ဦးတည်ထားကြောင်းအတည်ပြုခဲ့သည်
- **အကြောင်းအရာ အတည်ပြုခြင်း**: လတ်တလောစံချိန်မီစံနှုန်းများနှင့်ကိုက်ညီမှုအတွက် လုံခြုံရေးအကောင်းဆုံးဆရာရေးများ၏ ကွပ်ကဲအတည်ပြုခြင်း
  - **Microsoft လုံခြုံရေးဖြေရှင်းချက်များ**: Prompt Shields ("Jailbreak risk detection" ဟုခေါ်သော)၊ Azure Content Safety၊ Microsoft Entra ID နှင့် Azure Key Vault အတွက် လက်ရှိအသုံးအနှုန်းနှင့်လင့်ကွက်များ အတည်ပြု
  - **OAuth 2.1 လုံခြုံရေး**: လတ်တလော OAuth လုံခြုံရေးအကောင်းဆုံးဆရာရေးနှင့်ကိုက်ညီမှု အတည်ပြုသည်
  - **OWASP စံချိန်**: LLMs အတွက် OWASP အထိဆုံး ၁၀ ကိုးကားချက်များသည် လက်ရှိဖြစ်မှုအတည်ပြု
  - **Azure ဝန်ဆောင်မှုများ**: Microsoft Azure စာရွက်စာတမ်းလင့်ချက်များနှင့်အကောင်းဆုံးဆရာရေးများအားလုံးကို အတည်ပြု
- **စံနှုန်းနှင့်ကိုက်ညီမှု**: ကိုးကားထားသောလုံခြုံရေးစံနှုန်းများအားလုံးသည် လက်ရှိဖြစ်ကြောင်း အတည်ပြု
  - NIST AI Risk Management Framework
  - ISO 27001:2022
  - OAuth 2.1 လုံခြုံရေးအကောင်းဆုံးဆရာရေးများ
  - Azure လုံခြုံရေးနှင့်လိုက်နာမှုစနစ်များ
- **အကောင်အထည်ဖော်ရန် အရင်းအမြစ်များ**: အကောင်အထည်ဖော်ခြင်းလမ်းညွှန်များနှင့် အရင်းအမြစ်များအားလုံးကို အတည်ပြု
  - Azure API Management အတည်ပြုမူပုံစံများ
  - Microsoft Entra ID ပေါင်းစည်းမှုလမ်းညွှန်များ
  - Azure Key Vault လျှို့ဝှက်ချက်စီမံခန့်ခွဲမှု
  - DevSecOps ပိုင်းလိုက်နှင့် မျိုးကြိုးစောင့်ကြည့်ဖြေရှင်းချက်များ

### စာရွက်စာတမ်း အရည်အသွေး အာမခံချက်
- **အသေးစိတ်ဖော်ပြချက်လိုက်နာမှု**: အကယ်၍ လိုအပ်သော MCP လုံခြုံရေးလိုအပ်ချက်များ (MUST/MUST NOT) ကို လတ်တလောအသေးစိတ်ဖော်ပြချက်နှင့် ကိုက်ညီမှုရှိစေရန် အာမခံ
- **အရင်းအမြစ်လက်ရှိမှု**: Microsoft စာရွက်စာတမ်းများ၊ လုံခြုံရေးစံနှုန်းများနှင့် အကောင်အထည်ဖော်ခြင်းလမ်းညွှန်များထံလင့်ကွက်များအားလုံးကို စစ်ဆေးထား
- **အကောင်းဆုံးဆရာရေးပုံမှန်ခွဲခြမ်း**: အတည်ပြုမှု၊ ခွင့်ပြုမှု၊ AI သက်ရောက်မှုများ၊ ပေးသွင်းသူချေးထည့်ရေး လုံခြုံရေးနှင့် စီးပွားရေးပုံစံများအားလုံးကို အပြည့်အဝ အတည်ပြုထား

## ၂၀၂၅ ခုနှစ်၊ အောက်တိုဘာ ၆ ရက်

### စတင်အသုံးပြုခြင်း အပိုင်းချဲ့ထွင်ခြင်း – အဆင့်မြင့် ဆာဗာအသုံးပြုမှုနှင့် အခြေခံအတည်ပြုခြင်း

#### အဆင့်မြင့် ဆာဗာအသုံးပြုမှု (03-GettingStarted/10-advanced)
- **ခေါင်းစဉ်အသစ် ထည့်သွင်းခြင်း**: ပုံမှန်နှင့် နိမ့်ဆုံးအဆင့် ဆာဗာသ архитектခရန်များနှစ်ခုစလုံး ပါဝင်သော MCP ဆာဗာအသုံးပြုမှု လမ်းညွှန်စာအုပ် အသုံးပြုသူအသေးစိတ်ဖော်ပြချက်
  - **ပုံမှန်နှင့် နိမ့်ဆုံးအဆင့်ဆာဗာရှေ့မှာ ကြားကွာခြားချက်**: Python နှင့် TypeScript ဖြင့် ကုဒ်နမူနာများအပါအဝင် အသေးစိတ်နှိုင်းယှဉ်ဖော်ပြချက်
  - **Handler-Based ဒီဇိုင်း**: သုံးစွဲသူအဆင်ပြေမှု၊ ကျယ်ပြန့်မှုနှင့် တစ်ဆင့်မြှင့် ဆာဗာမှာ handler-based လုပ်ငန်းစဉ်/အရင်းအမြစ်/ပြောဆိုမှုများကို ရှင်းလင်းအကြောင်းျပသောအပိုင်း
  - **လက်တွေ့ပုံစံများ**: နိမ့်ဆုံးအဆင့် ဆာဗာပုံစံများဖြင့် အဆင့်မြင့်အင်္ဂါရပ်များနှင့် architecture အတွက် အသုံးဝင်မှုများ
     
#### ရိုးရှင်းသော အတည်ပြုမှု (03-GettingStarted/11-simple-auth)
- **ခေါင်းစဉ်အသစ် ထည့်သွင်းခြင်း**: MCP ဆာဗာများတွင် ရိုးရှင်းသော အတည်ပြုမှု အကောင်အထည်ဖော်မှု အဆင့်ဆင့်လမ်းညွှန်
  - **အတည်ပြုမှုပညာရပ်များ**: အတည်ပြုမှုနှင့် ခွင့်ပြုမှုကြားကွာခြားချက်၊ ကိုယ်ရေးအချက်အလက် လက်ခံကူးယူမှု ရှင်းလင်းပြောကြားသည်
  - **အခြေခံ Auth လုပ်ထုံးလုပ်နည်း**: Python (Starlette) နှင့် TypeScript (Express) တွင် Middleware-based အတည်ပြုမှု ပုံစံများ၊ ကုဒ်နမူနာများပါဝင်သည်
  - **အဆင့်မြှင့် လုံခြုံရေးဆီသို့ တိုးတက်မှု**: ရိုးရှင်းသော auth သင့်မှ စတင်ပြီး OAuth 2.1 နှင့် RBAC သို့ တိုးတက်မှုအတွက် လမ်းညွှန်မှုများနှင့် အဆင့်မြင့်လုံခြုံရေး မော်ဂျူးများကို ကိုးကားပြသခြင်း

ဤဖြည့်စွက်ချက်များသည် အခြေခံသဘောတရားများနှင့် အဆင့်မြင့် ထုတ်လုပ်မှု ပုံမှန်များ အကြား ချိတ်ဆက်မှုဖြင့် စိတ်ချရပြီး လုံခြုံစိတ်ချရသော၊ ကျယ်ပြန့်သော MCP ဆာဗာ အကောင်အထည်ဖော်မှုများအတွက် လက်တွေ့လမ်းညွှန်ချက်များကို ပံ့ပိုးပေးပါသည်။

## ၂၀၂၅ ခုနှစ်၊ စက်တင်ဘာ ၂၉ ရက်

### MCP ဆာဗာ ဒေတာဘေ့စ် ပေါင်းစည်းမှု လေ့ကျင့်ခန်းများ - လက်တွေ့ လေ့လာသင်ယူရန် လမ်းကြောင်း

#### 11-MCPServerHandsOnLabs - ပြည့်စုံသော ဒေတာဘေ့စ် ပေါင်းစည်းမှု သင်တန်းအစီအစဉ်အသစ်
- **၁၃ လေ့ကျင့်ခန်း ပြည့်စုံမှု**: PostgreSQL ဒေတာဘေ့စ်ပေါင်းစည်းထားသည့် ထုတ်လုပ်မှုဆာဗာ MCP ဖန်တီးခြင်းအတွက် လေ့ကျင့်ခန်း အစီအစဉ်တစ်ခုတိုးချဲ့ထည့်သွင်းရန်
  - **လက်တွေ့ အကောင်အထည်ဖော်ခြင်း**: စီးပွားရေးအဆင့် pattern များဖြင့် Zava Retail အာနာလစ်တစ်စ်အသုံးပြုမှု
  - **ဖွဲ့စည်းမှုရှိ လေ့ကျင့်မှုတိုးတက်မှု**:
    - **လေ့ကျင့်ခန်း 00-03: အခြေခံများ** - မိတ်ဆက်ခြင်း၊ အခြေခံ architecture၊ လုံခြုံရေးနှင့် မလုံခြုံသော အရာများ၊ ပတ်ဝန်းကျင်ပြင်ဆင်မှု
    - **လေ့ကျင့်ခန်း 04-06: MCP ဆာဗာတည်ဆောက်ခြင်း** - ဒေတာဘေ့စ် ဒီဇိုင်းနှင့် စီမံခန့်ခွဲမှု၊ MCP ဆာဗာ အကောင်အထည်ဖော်ခြင်း၊ ကိရိယာ ဖွံ့ဖြိုးတိုးတက်မှု
    - **လေ့ကျင့်ခန်း 07-09: အဆင့်မြင့် အင်္ဂါရပ်များ** - Semantic Search ပေါင်းစည်းမှု၊ စမ်းသပ်မှုနှင့် ပြင်ဆင်မှုပြုခြင်း၊ VS Code ပေါင်းစည်းမှု
    - **လေ့ကျင့်ခန်း 10-12: ထုတ်လုပ်မှုနှင့် အကောင်းဆုံးလေ့လာမှုများ** - ထုတ်လုပ်မှု မိတ်ဆက်လမ်းညွှန်၊ စောင့်ကြည့်မှုနှင့် ကြည့်ရှုမှု၊ အကောင်းဆုံးဆရာရေးနှင့် တိုးတက်အောင်မြင်မှု
  - **စီးပွားရေးနည်းပညာများ**: FastMCP framework, PostgreSQL with pgvector, Azure OpenAI embeddings, Azure Container Apps, Application Insights
  - **အဆင့်မြင့် အင်္ဂါရပ်များ**: Row Level Security (RLS), semantic search, multi-tenant data access, vector embeddings, အချိန်နှင့်တပြေးညီ စောင့်ကြည့်မှု

#### စကားလုံးသတ်မှတ်ချက် ပြောင်းလဲမှု - မော်ဂျူးမှ လေ့ကျင့်ခန်း သို့ပြောင်းခြင်း
- **စာရွက်စာတမ်း ပြင်ဆင်အပ်ဒိတ် စနစ်တကျ**: 11-MCPServerHandsOnLabs တွင် README ဖိုင်အားလုံးတွင် "Module" ထက် "Lab" စကားလုံးသုံးစွဲမှု ဆန့်ကျင်ဖုံးအပ်ဒိတ်လုပ်ခြင်း
  - **အပိုင်းခေါင်းစဉ်များ**: "What This Module Covers" မှ "What This Lab Covers" သို့ အချိုးတည့် ပြောင်းလဲမှု ၁၃ လေ့ကျင့်ခန်းအားလုံးတွင်
  - **အကြောင်းအရာ ဖော်ပြချက်**: "This module provides..." ကို "This lab provides..." ဟု အပြောင်းအလဲပြုလုပ်မှု
  - **လေ့လာမှုရည်မှန်းချက်**: "By the end of this module..." ကို "By the end of this lab..." ဟု ပြောင်းလဲမှု
  - **သွားလာပြန်လည် ရှာဖွေရေး လင့်များ**: "Module XX:" ကို "Lab XX:" ချိတ်ဆက်မှု
  - **ပြီးဆုံးမှု စောင့်ကြည့်မှု**: "After completing this module..." ကို "After completing this lab..." ဟု အပ်ဒိတ်လုပ်ခြင်း

    - Webhook လက်ခံသူများနှင့် push-based သတင်းစကား ဆောင်ရွက်မှု
  - **Azure Event Hubs သယ်ယူပို့ဆောင်မှု**: မြန်နှုန်းမြင့် စတရီမင်း သယ်ယူပို့ဆောင်မှု အကောင်အထည်ဖော်မှု
    - နိမ့်အချိန် နောက်ကျမှု ရှိသည့် စတရီမင်းအင်္ဂါရပ်များ
    - Partitioning နည်းဗျူဟာများနှင့် Checkpoint စီမံခန့်ခွဲမှု
    - သတင်းစကား အစုလိုက်နှင့် စွမ်းဆောင်ရည်တိုးတက်စေရေး
  - **စီးပွားရေး ပေါင်းစည်းမှု ပုံမှန်တွေ**: ထုတ်လုပ်မှုသင့် architecture နမူနာများ
    - Azure Functions များစွာအကြား မျှဝေဖြန့်ချိမှု MCP တိုက်ရိုက်မဲ့
    - မျိုးစုံ သယ်ယူပို့ဆောင်မှု ပေါင်းစည်းမှုများ
    - သတင်းစကား အစွမ်းခံမှု၊ ယုံကြည်စိတ်ချမှုနှင့် အမှားကာကွယ်မှု စနစ်များ
  - **လုံခြုံရေး နှင့် စောင့်ကြည့်မှု**: Azure Key Vault ပေါင်းစည်းမှုနှင့် ကြည့်ရှုနည်းများ
    - Managed identity ဖြင့် အတည်ပြုမူနှင့် နိမ့်ဆုံး ခွင့်ပြုမှု များ
    - Application Insights ခရိုင်သတင်းရယူခြင်းနှင့် စွမ်းဆောင်ရည် ကြည့်ရှုမှု
    - Circuit breaker နှင့် အမှားစွဲနည်းများ
  - **စမ်းသပ်မှု မာယာနည်းပညာများ**: စိတ်ကြိုက် သယ်ယူပို့ဆောင်မှုများအတွက် စမ်းသပ်မှု နည်းဗျူဟာများ
    - Unit test များတွင် test doubles နှင့် mocking frameworks အသုံးပြုခြင်း
    - Azure Test Containers ဖြင့် ပေါင်းစည်းစမ်းသပ်မှု
    - စွမ်းဆောင်ရည်နှင့် တင်ဆောင်စာမျက်နှာ စမ်းသပ်မှု အထောက်အကူများ

#### ပတ်ဝန်းကျင် အင်ဂျင်နီယာရေး (05-AdvancedTopics/mcp-contextengineering/) - တိုးတက်လာသော AI ဝတ်မတ်
- **README.md**: အနာဂတ်မှ ထွက်ပေါ်လာသော ပတ်ဝန်းကျင် အင်ဂျင်နီယာရေး နယ်ပယ်အား စုံလင်တော်တော် သုံးသပ်ချက်
  - **အခြေခံ စည်းကမ်းများ**: ပတ်ဝန်းကျင် ပြန်လည်မျှဝေမှု အပြည့်အစုံ၊ လုပ်ဆောင်မှု ဆုံးဖြတ်ချက် သတိပြုမှုနှင့် ပတ်ဝန်းကျင် ပြတင်းပေါက် စီမံခန့်ခွဲမှု
  - **MCP ပရိုတိုကော ညှိနှိုင်းမှု**: MCP ဒီဇိုင်း သည် ပတ်ဝန်းကျင် အင်ဂျင်နီယာရေး စိန်ခေါ်မှုများကို မည်သို့ ဖြေရှင်းပေးသည်
    - ပတ်ဝန်းကျင် ပြတင်းပေါက် ကန့်သတ်ချက်များနှင့် စဉ်ဆက်မပြတ် ဒေတာဖော်ပြမှု နည်းဗျူဟာများ
    - သက်ဆိုင်မှု ဖော်ထုတ်ခြင်းနှင့် ကြာရှည်သည့် ပတ်ဝန်းကျင် ဆွဲထုတ်မှု
    - မျိုးစုံ ပတ်ဝန်းကျင် ကျင်းဆင်းမှုနှင့် လုံခြုံရေး စဉ်းစားချက်များ
  - **အကောင်အထည်ဖော်ခြင်း နည်းလမ်းများ**: တစ်ဦးတည်းသရုပ်ဆောင် vs. မျိုးစုံ agent architecture များ
    - ပတ်ဝန်းကျင် အပိုင်းခွဲခြင်း နှင့် ဦးစားပေး ချုပ်ကိုင်မှုနည်းလမ်းများ
    - စဉ်ဆက်မပြတ် ပတ်ဝန်းကျင်ဖွင့်ခြင်းနှင့် ဖိအားချမှု နည်းဗျူဟာများ
    - အလွှာအလိုက် ပတ်ဝန်းကျင် နည်းလမ်းများနှင့် ဆွဲထုတ်ခြင်း အဆင့်မြှင့်မှု
  - **တိုင်းတာမှု စနစ်**: ပတ်ဝန်းကျင် ထိရောက်မှုအတွက် တိုးတက်လာသော အတိုင်းအတာများ
    - ထွက်ရောက်မှု ညှိနှိုင်းမှု၊ စွမ်းဆောင်ရည်၊ အရည်အသွေးနှင့် အသုံးပြုသူ အတွေ့အကြုံ စဉ်းစားမှု
    - ပတ်ဝန်းကျင် ဖွံ့ဖြိုးတိုးတက်မှုအတွက် စမ်းသပ် ပြုလုပ်မှု နည်းလမ်းများ
    - အဆင်မပြေမှု လေ့လာမှုနှင့် တိုးတက်ကောင်းမွန်မှုနည်းလမ်းများ

### စာရွက်စာတမ်း ပြည့်စုံ အပ်ဒိတ် - MCP 2025-06-18 စံချိန်များ

#### MCP လုံခြုံရေးအကောင်းဆုံးဆရာရေးများ (02-Security/) - အပြည့်အဝ ခေတ်မီပြုပြင်မှု
- **MCP-SECURITY-BEST-PRACTICES-2025.md**: MCP Specification 2025-06-18 နှင့် ကိုက်ညီသော ပြန်လည်ရေးသားချက် ပြည့်စုံစွာပြုလုပ်済
  - **ပဋိညာဉ်လိုအပ်ချက်များ**: အတည်ပြုချက်မှရသော MUST/MUST NOT လိုအပ်ချက်များကို ထည့်သွင်းပြသခြင်း၊ တိကျရှင်းလင်းသော ကြည်လင်သော မြင်သာချက်များဖြင့်
  - **ဗဟိုလုံခြုံရေး လေ့ကျင့်မှု ၁၂ ခု**: ၁၅ စာရင်းမှ ပြုပြင်မွမ်းမံပြီး လုံခြုံရေး ဒိုမိန်းများအနက်သို့ ပြောင်းလဲပုံ
    - အရေးအသားလုံခြုံရေးနှင့် မှတ်ပုံတင်ခြင်း၊ ပြင်ပ တူညီမှုအဖွဲ့အစည်း ထည့်သွင်းခြင်း
    - အစည်းအဝေး စီမံခန့်ခွဲမှုနှင့် ကနေရာလုံခြုံရေး၊ စာလုံးစဉ်လုံခြုံရေး လိုအပ်ချက်များဖြင့်
    - AI-အထူး ရန်ကုန် ကာကွယ်မှု၊ Microsoft Prompt Shields ပေါင်းစပ်မှု
    - လက်လှမ်းအောက်မှီခြင်းနှင့် ခွင့်ပြုချက်များ၊ အနည်းဆုံး အာဏာအဆင့် 원칙 ဖြင့်
    - အကြောင်းအရာလုံခြုံမှုနှင့် စောင့်ကြည့်မှု၊ Azure Content Safety ပေါင်းစပ်မှု
    - ပစ္စည်းသိုလှောင်မှု လုံခြုံမှု၊ ပစ္စည်းစစ်ဆေးမှုအသေးစိတ်အဖုံး
    - OAuth လုံခြုံမှုနှင့် Confused Deputy ကာကွယ်မှု၊ PKCE ကွပ်တွဲမှု
    - ဖြစ်ရပ်တုံ့ပြန်မှုနှင့် ပြန်လည် ရန်ကုန်မှု၊ အလိုအလျောက် စွမ်းရည်
    - လိုက်နာမှုနှင့် အုပ်ချုပ်မှု၊ စည်းမျဉ်းစည်းကမ်းအညီ
    - အဆင့်မြင့် လုံခြုံမှုထိန်းချုပ်မှု၊ zero trust architecture
    - Microsoft လုံခြုံရေး စနစ်ပေါင်းစပ်မှု၊ ဖြေရှင်းချက်အပြည့်အစုံ
    - ဆက်လက် လုံခြုံရေး တိုးတက်မှု၊ လက်ခံနည်းလမ်းများ
  - **Microsoft လုံခြုံရေး ဖြေရှင်းချက်များ**: Prompt Shields, Azure Content Safety, Entra ID, GitHub Advanced Security ပေါင်းစပ်မှု လမ်းညွှန်ချက် မြှင့်တင်မှု
  - **အကောင်အထည်ဖော်ရန် အရင်းအမြစ်များ**: တရားဝင် MCP စာတမ်းများ၊ Microsoft လုံခြုံရေးဖြေရှင်းချက်များ၊ လုံခြုံရေးစံချိန်များနှင့် အကောင်အထည်ဖော်လမ်းညွှန်များ အလိုက် အဖွဲ့ခွဲထားခြင်း

#### အဆင့်မြင့် လုံခြုံမှု ထိန်းချုပ်မှုများ (02-Security/) - စီးပွားရေးအတွက် အကောင်အထည်ဖော်ခြင်း
- **MCP-SECURITY-CONTROLS-2025.md**: စီးပွားရေးအဆင့် လုံခြုံရေး ဘာသာရပ်ပြောင်းလဲမှု ပြုလုပ်ခြင်း
  - **လုံခြုံရေး ဒိုမိန်း ၉ ခု အပြည့်အစုံ**: အခြေခံထိန်းချုပ်မှုများမှ စီးပွားရေး မော်ဒယ်အသေးစိတ်
    - အဆင့်မြင့် မှတ်ပုံတင်ခြင်းနှင့် ခွင့်ပြုချက် ချမှတ်ခြင်း၊ Microsoft Entra ID ပေါင်းစပ်မှုဖြင့်
    - အရေးအသားလုံခြုံရေးနှင့် မလုပ်ဆောင်ခွင့် ထိန်းချုပ်မှု၊ ပြည့်စုံစစ်ဆေးမှုဖြင့်
    - အစည်းအဝေး လုံခြုံရေးထိန်းချုပ်မှုများ၊ တပ်မက်မှုကာကွယ်သော နည်းလမ်းများ
    - AI-အထူး လုံခြုံမှုထိန်းချုပ်မှုများ၊ prompt injection နှင့် tool poisoning ကာကွယ်မှု
    - Confused Deputy တိုက်ခိုက်မှု ကာကွယ်မှု၊ OAuth proxy လုံခြုံမှု
    - ကိရိယာ ခွဲထုတ် လုံခြုံမှု ၊ sandboxing နှင့် isolation
    - ပစ္စည်းသိုလှောင်မှု လုံခြုံရေး ထိန်းချုပ်မှုများ၊ အားထားမှုစစ်ဆေးခြင်း
    - စောင့်ကြည့်မှုနှင့် ဖော်ထုတ်မှု ထိန်းချုပ်မှု၊ SIEM ပေါင်းစပ်မှု
    - ဖြစ်ရပ်တုံ့ပြန်ရေးနှင့် ပြန်လည်ကယ်ဆယ်မှု၊ အလိုအလျောက် စွမ်းဆောင်ရည်များ
  - **အကောင်အထည်ဖော် ဥပမာများ**: YAML စနစ်တကျဖော်ပြမှုများနှင့် ကိရိယာ ကိုဒ် ဥပမာများ ထည့်သွင်းခြင်း
  - **Microsoft ဖြေရှင်းချက်များ ပေါင်းစပ်မှု**: Azure လုံခြုံရေး ဝန်ဆောင်မှုများ၊ GitHub Advanced Security နှင့် စီးပွားရေး ကိုက်ညီမှု

#### အဆင့်မြင့် ခေါင်းစဉ် လုံခြုံရေး (05-AdvancedTopics/mcp-security/) - ထုတ်လုပ်ရေးအဆင့် အကောင်အထည်ဖော်ခြင်း
- **README.md**: စီးပွားရေး လုံခြုံရေး အကောင်အထည်ဖော်မှုအတွက် ပြန်လည်ရေးသားမှု ပြည့်စုံ
  - **လက်ရှိ သတ်မှတ်ချက် နှိုင်းယှဉ်ခြင်း**: MCP Specification 2025-06-18 အညီ ဖြည့်စည်းချက်များ ရှိ
  - **အဆင့်မြင့် မှတ်ပုံတင်ခြင်း**: Microsoft Entra ID ပေါင်းစပ်မှု၊ .NET နှင့် Java Spring Security ဥပမာများအပြည့်အစုံ
  - **AI လုံခြုံရေး ပေါင်းစပ်မှု**: Microsoft Prompt Shields နှင့် Azure Content Safety အကောင်အထည်ဖော်မှု Python ဥပမာများ ဖြင့်
  - **အဆင့်မြင့် အန္တရာယ် လျှော့ချမှု**: လုပ်ဆောင်မှုအပြည့်အစုံအတွက် အကောင်အထည်ဖော်မှု ဥပမာများ
    - Confused Deputy တိုက်ခိုက်မှု ကာကွယ်မှု PKCE နှင့် အသုံးပြုသူ သဘောတူကန့်သတ်ချက်
    - အရေးအသား ပို့ခြင်း ကာကွယ်မှု၊ ကြားခံ သက်ဆိုင်မှု အတည်ပြုခြင်းနှင့် လုံခြုံသော token စီမံခန့်ခွဲမှု
    - အစည်းအဝေး တပ်မက်မှု ကာကွယ်မှု၊ စာလုံးစဉ်ကင်း ဂန္ထဝင် နှင့် သောကဆင်ခြင်းဆိုင်ရာ သုံးသပ်ချက်များဖြင့်
  - **စီးပွားရေး လုံခြုံရေး ပေါင်းစပ်မှု**: Azure Application Insights ကိုစောင့်ကြည့်မှု၊ အန္တရာယ်ရှာဖွေမှု လမ်းကြောင်းများနှင့် ပစ္စည်းသိုလှောင်မှု လုံခြုံမှု
  - **အကောင်အထည်ဖော် စာရင်း**: တိကျရှင်းလင်းသော လိုအပ်ချက်များနှင့် အကြံပြု လုံခြုံရေးထိန်းချုပ်မှုများ၊ Microsoft လုံခြုံရေး စနစ်၏ အကျိုးသက်ရောက်မှုများ

### စာရွက်စာတမ်း အရည်အသွေးနှင့် စံနှုန်းများ ကိုက်ညီမှု
- **သတ်မှတ်ချက် အညွှန်းများ**: လက်ရှိ MCP Specification 2025-06-18 နှင့် ကိုက်ညီသည့် အညွှန်းများအားလုံး ပြင်ဆင်ထားသည်
- **Microsoft လုံခြုံရေး စနစ်**: လုံခြုံရေး စာရွက်စာတမ်း အားလုံးတွင် ပေါင်းစပ် လမ်းညွှန်ချက် မြှင့်တင်
- **အကောင်အထည်ဖော်မှု အကျယ်အဝန်း**: .NET, Java နှင့် Python တွင် အသေးစိတ် ကုဒ် ဥပမာများနှင့် စီးပွားရေး ပုံစံများ ထည့်သွင်းထားသည်
- **အရင်းအမြစ် စီမံခန့်ခွဲမှု**: တရားဝင် စာရွက်စာတမ်းများ၊ လုံခြုံရေးစံချိန်များနှင့် အကောင်အထည်ဖော် လမ်းညွှန်များကို အုပ်စုခွဲထားသည်
- **မြင်သာသော အချက်ပြချက်များ**: လိုအပ်ချက်များနှင့် အကြံပြု လေ့ကျင့်မှုများ အတွက် တိကျရှင်းလင်းသော သင်္ကေတများ ဖော်ပြထားသည်

#### အခြေခံစိတ်အယူများ (01-CoreConcepts/) - ပြီးပြည့်စုံသော အဆန်းသစ်အပြောင်းအလဲ
- **ကြိုးပုံဗားရှင်း အသစ်ပြောင်းလဲမှု**: MCP Specification 2025-06-18 ကို ရည်ညွှန်းသော ယခုအချိန်အခြေခံ ရက်စွဲအပေါ် ထောက်ခံပြီး ပြောင်းလဲ
- **စီမံခန့်ခွဲမှု ဖွဲ့စည်းပုံ တိုးတက်မှု**: Hosts, Clients နှင့် Servers ကို ယခု MCP ဖွဲ့စည်းပုံ မော်ဒယ်များအောက် ဖြစ်ပေါ်ဖော်ပြခြင်း
  - Hosts ကို MCP client connection များစွာကို စီမံညှိနှိုင်းသည့် AI အပလီကေးရှင်းများအဖြစ် သေချာ ဖော်ပြထားသည်
  - Clients ကို protocol ချိတ်ဆက်ရပ်များ အဖြစ် ဖော်ပြထား၍ တစ်နေရာတစ်ခုစီ server နှင့်ဆက်သွယ်မှုကို ထိန်းသိမ်းခြင်း
  - Servers ကို ဒေသခံ/ဝေးလံသော တပ်ဆင်ခြင်း ရှိစေရန် တိုးတက်ပြုပြင်ထားသည်
- **ဒေတာ ပ<LM>တ်မျိုး ပုံစံ ပြောင်းလဲမှု**: server နှင့် client primitives များအား ပြန်လည်ပြုပြင်မှု ပြုလုပ်ပြီး အသေးစိတ် ရှင်းလင်းချက်နှင့် ဥပမာများကြုံ
  - Server Primitives: ရင်းမြစ်များ (ဒေတာ အရင်းအမြစ်များ), Prompt များ (စနစ်ပုံစံများ), Tools များ (ဖန်တီးနိုင်သော လုပ်ဆောင်ချက်များ)
  - Client Primitives: Sampling (LLM ပြီးမြောက်ခြင်းများ), Elicitation (အသုံးပြုသူ အကြောင်းကြားချက်), Logging (တိုးတက်ရေး/စောင့်ကြည့်ရေး)
  - ကုဒ် နည်းလမ်းအသစ်များ နှင့် discovery (`*/list`), retrieval (`*/get`), execution (`*/call`) များကို ဖော်ပြပုံ အသစ်နဲ့ တိုးတက်မှုများ
- **Protocol ဖွဲ့စည်းပုံ**: နှစ်အလွှာ ဖွဲ့စည်းမှု မော်ဒယ်အသစ်
  - ဒေတာ အလွှာ: JSON-RPC 2.0 အခြေခံဖြစ်ပြီး lifecycle စီမံခန့်ခွဲမှုနှင့် primitives များအပါအဝင်
  - သယ်ယူပို့ဆောင်မှု အလွှာ: STDIO (ဒေသခံ) နှင့် Streamable HTTP, SSE (ဝေးလံ) သယ်ယူပို့ဆောင်မှုနည်းလမ်းများ
- **လုံခြုံရေး စနစ်**: အသုံးပြုသူ သဘောတူချက် ပိုကောင်းစေရန်၊ ဒေတာ ကိုယ်ရေးကာကွယ်မှု၊ ကိရိယာ လုံခြုံရေးနှင့် သယ်ယူပို့ဆောင်မှု လုံခြုံရေးအပါအဝင် စုံလင့်လုံခြုံရေး စည်းကမ်းများ
- **ဆက်သွယ်မှု ပုံစံများ**: protocol ပတ်သက်သော မက်ဆေ့ချ်များစီမံ၊ ရှာဖွေ၊ လုပ်ဆောင်ခြင်းနှင့် အသိပေးပို့ဆောင်မှုများ ပြောင်းလဲမှုများ
- **ကုဒ် ဥပမာများ**: ဒေတာ အသစ်များနှင့် MCP SDK မော်ဒယ်များ အား .NET, Java, Python, JavaScript ဘာသာစကားများဖြင့် ပြန်လည်သွားဖြစ်

#### လုံခြုံရေး (02-Security/) - လုံခြုံရေး ပြောင်းလဲပြင်းထန်မှု  
- **စံနှုန်းများကိုက်ညီမှု**: MCP Specification 2025-06-18 လုံခြုံရေးလိုအပ်ချက်များနှင့် နူးညံ့စွာ ကိုက်ညီ
- **မှတ်ပုံတင်ခြင်း တိုးတက်မှု**: စိတ်ကြိုက် OAuth server များမှ ပြင်ပ တူညီမှုအဖွဲ့အစည်းတစ်ခုဖြစ် Microsoft Entra ID သို့ ခုခံခွင့်လွှတ်ပြောင်းလဲမှု
- **AI-အထူး အန္တရာယ် စစ်တမ်း**: ရှေ့ဆက်လှုပ်ရှားမှုမှုရပ် တိုက်ခိုက်မှုများ၊ တကယ့်ဘဝ နမူနာများဖြင့် ကြီးကြပ် တိုးတက်အားဖြင့် ပုံဖော်မှု
  - prompt injection တိုက်ခိုက်မှုနမူနာ အသေးစိတ်
  - tool poisoning နည်းလမ်းများနှင့် "rug pull" တိုက်ခိုက်မှု ရာဇဝင်ပုံစံများ
  - context window poisoning နှင့် မော်ဒယ် ရွံရှာမှုတိုက်ခိုက်မှုများ
- **Microsoft AI လုံခြုံရေး ဖြေရှင်းချက်များ**: Microsoft လုံခြုံရေး စနစ် ပေါင်းအုပ်မှု အပြည့်အစုံ
  - AI Prompt Shields ပါဝင်သော ရှာဖွေခြင်း၊ အလင်းပြတ်ခြင်းနှင့် delimiter နည်းလမ်းများ
  - Azure Content Safety ပေါင်းစည်းမှု ပုံစံများ
  - GitHub Advanced Security နဲ့ ပစ္စည်းသိုလှောင်မှု ကာကွယ်ရေး
- **အဆင့်မြင့် အန္တရာယ် လျှော့ချမှု**: အသေးစိတ် လုံခြုံရေး ထိန်းချုပ်မှုများ
  - MCP အထူး session hijacking တိုက်ခိုက်မှုနယ်ပယ်နှင့် စာလုံးစဉ်ထိုးနှိမ်နည်း
  - MCP proxy တွင် confused deputy ပြဿနာများအား သဘောတူညီချက် နှင့်အတူ ကာကွယ်ခြင်း
  - token passthrough အား ကန့်သတ်စစ်ဆေးမှုများနှင့် ပြဿနာများ
- **ပစ္စည်းသိုလှောင်မှု လုံခြုံမှု**: AI ပစ္စည်းသိုလှောင်မှု တိုးတက်မှုများ၊ foundation models, embeddings services, context providers နှင့် တတိယပါတီ API များ ပါဝင်ခြင်း
- **ပန်းတိုင် လုံခြုံမှု**: စီးပွားရေး လုံးဝ ပြင်ဆင်ခြင်းနှင့် Microsoft လုံခြုံရေး စနစ်များ ဆက်စပ်မှု
- **အရင်းအမြစ် စီမံခန့်ခွဲမှု**: အမျိုးအစားအလိုက် အဖွဲ့ခွဲမှုပြုလုပ်ထားခြင်း (တရားဝင် စာရွက်စာတမ်းများ၊ စံချိန်များ၊ သုတေသန၊ Microsoft ဖြေရှင်းချက်များ၊ အကောင်အထည်ဖော် လမ်းညွှန်များ)

### စာရွက်စာတမ်း အရည်အသွေး တိုးတက်မှုများ
- **ဖွဲ့စည်းတကျ သင်ခန်းစာ ရည်မှန်းချက်များ**: သီးသန့်နှင့် လုပ်ဆောင်နိုင်သော ရလဒ်များထည့်သွင်း ဖွဲ့စည်းမှု
- **တိုက်ဆိုင်မှု ချိတ်ဆက်မှုများ**: လုံခြုံရေးနှင့် အခြေခံစိတ်အယူ အကြောင်းအရာများအကြား ချိတ်ဆက်ထားခြင်း
- **လက်ရှိ အချက်အလက်များ**: ရက်စွဲနှင့် သတ်မှတ်ချက် အညွှန်းများအားလုံးကို လက်ရှိစံနှုန်းများနှင့် ပြောင်းလဲထားခြင်း
- **အကောင်အထည်ဖော် လမ်းညွှန်ချက်များ**: နှစ်ပိုင်းစီ အုပ်စုအတွင်း တိကျသော လုပ်ဆောင်နိုင်သည့် လမ်းညွှန်ချက်များ ထည့်သွင်းထားခြင်း

## ၂၀၂၅ ခုနှစ် ဇူလိုင်လ ၁၆ ရက်

### README နှင့် ဦးတည်ရာ တိုးတက်မှုများ
- README.md တွင် သင်တန်းညွှန်ကြားမှု တာဝန် အပြည့်အစုံ ပြန်လည်စီမံခြင်း
- `<details>` ကုဒ်အပိုင်းများအား ပိုမိုလွယ်ကူသော ဇယားပုံစံချပ်ဖြင့် အစားထိုးခြင်း
- "alternative_layouts" ဖိုလ်ဒါ အသစ် ဖန်တီးပြီး မတူညီသော ဖွဲ့စည်းပုံ ရွေးချယ်မှုများ ပြုလုပ်ခြင်း
- ကတ်ပုံစံ၊ တက်ဘ်ပုံစံနှင့် accordion ပုံစံ လှုပ်ရှားမှု ဥပမာများ ထည့်သွင်းခြင်း
- လက်ရှိဖိုင်များ အားလုံးအား Repository ဖွဲ့စည်းပုံ အစိတ်အပိုင်းတွင် ထည့်သွင်းပြင်ဆင်ခြင်း
- "How to Use This Curriculum" အပိုင်းကို တိကျသေချာသော အကြံပြုချက်များဖြင့် မြှင့်တင်ခြင်း
- MCP သတ်မှတ်ချက် လင့်များကို မှန်ကန်သည့် URL များသို့ အပ်ဒိတ်လုပ်ခြင်း
- သင်တန်း ဖွဲ့စည်းပုံ၌ Context Engineering အပိုင်း (5.14) ထည့်သွင်းခြင်း

### သင်ယူလမ်းညွှန် အပ်ဒိတ်များ
- လက်ရှိ Repository ဖွဲ့စည်းပုံနှင့် ကိုက်ညီစေရန် သင်ယူလမ်းညွှန် ပြန်လည်ရေးသားခြင်း
- MCP Clients နှင့် Tools များအတွက် အပိုင်းအသစ် ထည့်သွင်းခြင်း၊ လူကြိုက်များသော MCP Servers များအတွက် အပိုင်း သတ်မှတ်မှု
- Visual Curriculum Map တိကျစွာ ပြင်ဆင်ပြောင်းလဲခြင်း
- အဆင့်မြင့် ခေါင်းစဉ်များအကြောင်း ရှင်းလင်းချက်များ တိုးချဲ့ဖော်ပြခြင်း
- နမူနာများကို ပြန်လည်အသိပေး Case Studies အပိုင်း ပြင်ဆင်ခြင်း
- ဒီအပြည့်အစုံသော ပြိုင်ပွဲ မှတ်တမ်း ထည့်သွင်းခြင်း

### လူမှု ကောင်စီ ရည်လမ်းများ (06-CommunityContributions/)
- ပုံရိပ်ဖန်တီးမှုအတွက် MCP server များအကြောင်း နက်ရှိုင်းသော အချက်အလက် ထည့်သွင်းခြင်း
- VSCode တွင် Claude အသုံးပြုခြင်းအကောင်အထည်ဖော်မှု အကျဉ်းချုပ်
- Cline terminal client သုံးသပ်မှုနှင့် သတ်မှတ်ချက် များထည့်ခြင်း
- MCP client အပိုင်းကို လူကြိုက်များသော client အစုံအလင် ထည့်သွင်းပြင်ဆင်ခြင်း
- ပံ့ပိုးမှု ရေးထားမည့် ဥပမာများအား ပိုမိုတိကျသော ကုဒ်နမူနာများဖြင့် မြှင့်တင်

### အဆင့်မြင့် ခေါင်းစဉ် (05-AdvancedTopics/)
- အထူးသီးသန့် ခေါင်းစဉ် ဖိုလ်ဒါများ စည်းမျဉ်းတစ်မျိုးတည်း နှင့် စီမံအုပ်ချုပ်မှု
- Context Engineering အကြောင်းအရာနှင့် ဥပမာများ ထည့်သွင်း
