## சோதனை மற்றும் பிழைத்திருத்தம்

உங்கள் MCP சர்வரை சோதனை செய்ய ஆரம்பிப்பதற்கு முன், கிடைக்கும் கருவிகள் மற்றும் பிழைத்திருத்தத்திற்கான சிறந்த நடைமுறைகள் பற்றிய புரிதல் அவசியம். பயனுள்ள சோதனை உங்கள் சர்வர் எதிர்பார்ப்பிற்கேற்ப செயல்படுவதை உறுதிப்படுத்துகிறது மற்றும் திடீர் பிழைகளை விரைவாக கண்டறிந்து சரிசெய்ய உதவுகிறது. கீழ்க்காணும் பகுதி உங்கள் MCP செயலாக்கத்தை சரிபார்க்க பரிந்துரைக்கப்பட்ட அணுகுமுறைகளைக் குறிப்பிடுகிறது.

## ஒட்டுமொத்தம்

இந்த பாடம் சரியான சோதனை அணுகுமுறையை தேர்வு செய்வது மற்றும் மிக பொதுவான சோதனை கருவியைப் பயன்படுத்துவது குறித்து விளக்குகிறது.

## கற்றல் குறிக்கோள்கள்

இந்த பாடம் முடிவடைவதற்குப் பிறகு, நீங்கள் திறமையாக முடியும்:

- பல்வேறு சோதனை அணுகுமுறைகளை விவரிக்க.
- வெவ்வேறு கருவிகளை பயன்படுத்தி உங்கள் குறியீட்டை பயனுள்ளதாக சோதிக்க.


## MCP சர்வர்களை சோதனை செய்தல்

MCP உங்கள் சர்வர்களை சோதிப்பதற்கும் பிழைத்திருத்துவதற்கும் உதவும் கருவிகளை வழங்குகிறது:

- **MCP இன்ஸ்பெக்டர்**: இது CLI கருவி மற்றும் காட்சி கருவியாக இரண்டும் செயல்படும் கட்டளைக் கோல் கருவி.
- **கையேடு சோதனை**: நீங்கள் curl போன்ற கருவியை பயன்படுத்தி வலை கோரிக்கைகளை இயக்கலாம், ஆனால் HTTP இயங்கக்கூடிய எந்த கருவியும் பயன்படாது.
- **அலகு சோதனை**: உங்கள் தேர்ந்தெடுத்த சோதனை கட்டமைப்பை பயன்படுத்தி சர்வர் மற்றும் கிளையண்டின் அம்சங்களை சோதிக்க முடியும்.

### MCP இன்ஸ்பெக்டர் பயன்படுத்துதல்

இந்த கருவியின் பயன்பாடு நாங்கள் முந்தைய பாடங்களில் விவரித்துள்ளோம், இப்போது தளர்வான அளவில் சிறிது பேசுவோம். இது Node.js இல் கட்டமைக்கப்பட்ட கருவியாகும் மற்றும் நீங்கள் `npx` இயக்கக்கூடிய செயற்கூறு மூலம் பயன்படுத்தலாம், இது கருவியை தற்காலிகமாக பதிவிறக்கி நிறுவி உங்கள் கோரிக்கையை இயக்கியதும் தானாக சுத்தம் செய்யும்.

[MCP இன்ஸ்பெக்டர்](https://github.com/modelcontextprotocol/inspector) உதவுகிறது:

- **சர்வர் திறன்களை கண்டுபிடிக்க**: கிடைக்கக்கூடிய வளங்கள், கருவிகள் மற்றும் இடுக்கைகள் தானாக கண்டுபிடிக்கப்படும்
- **கருவி செயல்பாட்டை சோதிக்க**: வெவ்வேறு அளவுருக்கள் முயற்சி செய்து நேரடி பதில்களை காண
- **சர்வர் மெட்டா டேட்டாவை பார்க்க**: சர்வர் தகவல், திட்ட அட்டவணைகள் மற்றும் அமைப்புகளை ஆய்வு செய்ய

கருவி ஒரு வழக்கமான இயக்கம் கீழ்வருமாறு காணப்படுகிறது:

```bash
npx @modelcontextprotocol/inspector node build/index.js
```

மேலே உள்ள கட்டளை MCP மற்றும் அதன் காட்சி முகவரியை இயக்கி உங்கள் உலாவியில் உள்ளூர் வலை முகவரி ஆரம்பிக்கும். இதன் மேற்பரப்பில் பதிவு செய்யப்பட்ட MCP சர்வர்கள், அவற்றின் கிடைக்கும் கருவிகள், வளங்கள் மற்றும் இடுக்கைகள் காணப்படுகின்றன. இந்த இடைமுகம் உங்களுக்கு கருவி செயல்பாட்டை செயல்படுத்தி சோதிக்க, சர்வர் மெட்டா டேட்டாவை ஆய்வு செய்ய மற்றும் நேரடி பதில்களைப் பார்வையிட உதவுகிறது, இதனால் உங்கள் MCP சர்வர் செயலாக்கங்களை எளிதாக சரிபார்க்கவும் பிழைத்திருத்தவும் முடியும்.

இது இப்படி இருக்கும்: ![Inspector](../../../../translated_images/ta/connect.141db0b2bd05f096.webp)

மேலும், நீங்கள் இந்த கருவியை CLI முறையில் இயக்கலாம், அப்போது `--cli` பண்பை சேர்க்க வேண்டும். இது "CLI" முறையில் கருவியை இயக்கும் உதாரணமாகும், இது சர்வரில் உள்ள அனைத்து கருவிகளையும் பட்டியலிடும்:

```sh
npx @modelcontextprotocol/inspector --cli node build/index.js --method tools/list
```

### கையேடு சோதனை

இன்ஸ்பெக்டர் கருவியை பயன்படுத்தி சர்வர் திறன்களை சோதிப்பதற்கு வெளியே, HTTP பயன்பாட்டை செய்யக்கூடிய கிளையண்டைப் பயன்படுத்துவது இன்னொரு ஒத்த அணுகுமுறை.

curl மூலம், நீங்கள் MCP சர்வர்களை நேரடியாக HTTP கோரிக்கைகளால் சோதிக்கலாம்:

```bash
# உதாரணம்: சோதனை சர்வர் மெடாடேட்டா
curl http://localhost:3000/v1/metadata

# உதாரணம்: ஒரு கருவியை செயல்படுத்துக
curl -X POST http://localhost:3000/v1/tools/execute \
  -H "Content-Type: application/json" \
  -d '{"name": "calculator", "parameters": {"expression": "2+2"}}'
```

மேலே curl பயன்பாட்டைப் பார்த்தபோது, கருவியின் பெயர் மற்றும் அளவுருக்களுடன் கூடிய பேலோட்டை பயன்படுத்தி கருவியை இயக்க POST கோரிக்கை செய்யப்படுகிறது. உங்களுக்கு ஏற்ற அணுகுமுறையை பயன்படுத்துங்கள். பொதுவாக CLI கருவிகள் பயன்படுத்த விரைவாகவும் மற்றும் ஸ்கிரிப்ட் செய்யக்கூடியதாகவும் இருக்கும், இது CI/CD சூழலில் பயனுள்ளதாக இருக்கலாம்.

### அலகு சோதனை

உங்கள் கருவிகள் மற்றும் வளங்களுக்கு அலகு சோதனைகள் உருவாக்கி அவை எதிர்பார்ப்புப்போல் செயல்படுவதை உறுதிப்படுத்துங்கள். சில உதாரண சோதனை குறியீடு இதோ.

```python
import pytest

from mcp.server.fastmcp import FastMCP
from mcp.shared.memory import (
    create_connected_server_and_client_session as create_session,
)

# அசிங்க் டெஸ்டുകൾக்காக முழு தொகுதியையும் குறியிடவும்
pytestmark = pytest.mark.anyio


async def test_list_tools_cursor_parameter():
    """Test that the cursor parameter is accepted for list_tools.

    Note: FastMCP doesn't currently implement pagination, so this test
    only verifies that the cursor parameter is accepted by the client.
    """

 server = FastMCP("test")

    # சில டெஸ்ட் கருவிகளை உருவாக்கவும்
    @server.tool(name="test_tool_1")
    async def test_tool_1() -> str:
        """First test tool"""
        return "Result 1"

    @server.tool(name="test_tool_2")
    async def test_tool_2() -> str:
        """Second test tool"""
        return "Result 2"

    async with create_session(server._mcp_server) as client_session:
        # குர்சர் அளவுருவை இல்லாமல் சோதிக்கவும் (விட்டுவைக்கப்பட்டது)
        result1 = await client_session.list_tools()
        assert len(result1.tools) == 2

        # குர்சர்=None உடன் சோதிக்கவும்
        result2 = await client_session.list_tools(cursor=None)
        assert len(result2.tools) == 2

        # குர்சர் ஸ்டிரிங் ஆக இருக்கும் போது சோதிக்கவும்
        result3 = await client_session.list_tools(cursor="some_cursor_value")
        assert len(result3.tools) == 2

        # காலி ஸ்டிரிங் குர்சர் உடன் சோதிக்கவும்
        result4 = await client_session.list_tools(cursor="")
        assert len(result4.tools) == 2
    
```

மேலுள்ள குறியீடு:

- pytest கட்டமைப்பைப் பயன்படுத்துகிறது, இது நீங்கள் செயல்பாடுகள் வடிவில் சோதனைகள் உருவாக்கி, assert வாக்கியங்களை பயன்படுத்த அனுமதிக்கும்.
- இரண்டு வெவ்வேறு கருவிகளுடன் MCP சர்வரை உருவாக்குகிறது.
- குறிப்பிட்ட நிபந்தனைகள் பூர்த்தி செய்யப்பட்டுள்ளதா என்பதை `assert` மூலம் சோதிக்கிறது.

[முழு கோப்பைப் பார்க்க](https://github.com/modelcontextprotocol/python-sdk/blob/main/tests/client/test_list_methods_cursor.py)

மேலுள்ள கோப்பின் அடிப்படையில், உங்கள் சொந்த சர்வரை சோதித்து திறன்கள் சரியாக உருவாக்கப்பட்டுள்ளதை உறுதிப்படுத்துங்கள்.

அனைத்து முக்கிய SDKகளுக்கும் இதேபோல் சோதனை பிரிவுகள் உள்ளன, ஆகவே உங்கள் தேர்ந்தெடுத்த இயக்க சூழலுக்கு ஏற்ப மாற்றக்கூடும்.
