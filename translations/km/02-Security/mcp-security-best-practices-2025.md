## ១. **សុវត្ថិភាព Token និង ការផ្ទៀងផ្ទាត់អត្តសញ្ញាណ**

**ការគ្រប់គ្រង ការផ្ទៀងផ្ទាត់ និង អនុញ្ញាត៖**
   - **ការត្រួតពិនិត្យការអនុញ្ញាតយ៉ាងតឹងរឹង**៖ ប្រារព្ធរាវ៉ាន់ឯកតាសំរាប់ផ្លូវចូលឡូរិចម៉ាស៊ីនបម្រើ MCP ដើម្បីធានាថា មានតែអ្នកប្រើប្រាស់ និងអតិថិជនដែលគោលបំណងដែលអាចចូលបានតែប៉ុណ្ណោះ
   - **ការរួមបញ្ចូល អ្នកផ្គត់ផ្គង់អត្តសញ្ញាណខាងក្រៅ**៖ ប្រើអ្នកផ្គត់ផ្គង់អត្តសញ្ញាណដែលបានស្គាល់ ដូចជា Microsoft Entra ID ជំនួសការអនុវត្តផ្ទាល់

## ២. **ការគ្រប់គ្រងសម័យ និង សុវត្ថិភាពការរុករក**

**អនុវត្តន៍សម័យសុវត្ថិភាព:**
   - **លេខសម្គាល់សម័យដែលមានគន្លងគ្រាន់តែជាសុវត្ថិភាព**៖ ប្រើលេខសម្គាល់សម័យ ដែលមិនអាចទាយទុកបាន ហើយបង្កើតដោយម៉ាសុីនចាប់លេខកំណត់ក្នុងវិធីសាស្ត្រដែលមានសុវត្ថិភាព
   - **ចងចេញជាមួយអ្នកប្រើប្រាស់**៖ ចងលេខសម្គាល់សម័យជាមួយអត្តសញ្ញាណអ្នកប្រើប្រាស់ ដោយប្រើរបៀបដូចជា `<user_id>:<session_id>` ដើម្បីទប់ស្កាត់ការបំពានសម័យពីអ្នកប្រើផ្សេងគ្នា
   - **គ្រប់គ្រងវដ្តជីវិតសម័យ**៖ អនុវត្តការផុតកំណត់ ការប្ដូរ និងការបដិសេធល្អ ដើម្បីកំណត់រយៈពេលប្រេងស៊ីងងាយរងគ្រោះ
   - **ការបង្ខិតបង្ខំនូវ HTTPS/TLS**៖ តម្រូវការប្រើ HTTPS សម្រាប់ការទំនាក់ទំនងទាំងអស់ ដើម្បីទប់ស្កាត់ការចាប់លេខសម្គាល់សម័យ

**សុវត្ថិភាពស្រទាប់ដឹកជញ្ចូន:**
   -កំណត់ TLS 1.3 នៅកន្លែងដែលអាចធ្វើបាន ជាមួយការគ្រប់គ្រងវិញ្ញាបនបត្រដូចគ្នា
   - អនុវត្តការដាប់វិញ្ញាបនបត្រច្បាប់សម្រាប់ការតភ្ជាប់សំខាន់ៗ
   - ប្ដូរវិញ្ញាបនបត្រជាទៀងទាត់ និងពិនិត្យសុពលភាព

## ៤. **ការត្រួតពិនិត្យចូល និងសិទ្ធិ**

**គ្រឹះសិទ្ធិសាបសូន្យ (Principle of Least Privilege):**
   - ផ្ដល់សិទ្ធិបំផុតតិចតួចដល់ម៉ាស៊ីន​បម្រើ MCP ដែលត្រូវការ សម្រាប់មុខងារដែលគ្រោងដំណើរការ
   - អនុវត្ត RBAC ជាមួយសិទ្ធិស្ត្រីមុខយ៉ាងម៉ត់ចត់
   - ត្រួតពិនិត្យសិទ្ធិជាប្រចាំ និងត្រួតពិនិត្យប្រើប្រាស់សិទ្ធិដែលអាចរីកឡើង

**ការគ្រប់គ្រងសិទ្ធិសកម្មភាព Runtime:**
   - ចាត់ចម្លងកំណត់កម្រិតធនធាន ដើម្បីទប់ស្កាត់ការវាយប្រហារបរិភោគធនធាន
   - ប្រើការ隔離 container សម្រាប់បរិយាកាសអនុវត្តឧបករណ៍
   - អនុវត្តការចូលប្រើប្រាស់ឱ្យបានទាន់ពេលវេលាសម្រាប់មុខងារគ្រប់គ្រង

## ៧. **សុវត្ថិភាព OAuth និងការពារអ្នកតំណាងច្របូកច្របល់**

   - **ការផ្ទៀងផ្ទាត់ Redirect URI**៖ អនុវត្តការផ្ទៀងផ្ទាត់តឹងរឹង លើ Redirect URI និង Client Identifiers

**សុវត្ថិភាព Proxy:**
   - ទប់ស្កាត់ការល្បួងល្មើសអនុញ្ញាតតាម Static Client ID
   - អនុវត្តដំណើរការយល់ព្រមសមរម្យសម្រាប់ការចូលប្រើ API ចុងក្រោយ
   - ត្រួតពិនិត្យការលួចកូដអនុញ្ញាត និងការចូលប្រើ API មិនបានអនុញ្ញាត

   - **ត្រួតពិនិត្យការ​ធ្វើការ​នៃកម្មវិធី**៖ ត្រួតពិនិត្យរកការឍង់ចាប់ប្រតិបត្តិការដែលអាចបង្ហាញថាអស់សិទ្ធិវាយប្រហារ
   - **គោលការណ៍សុវត្ថិភាពដាក់បែបផ្លាស់ប្តូរ**៖ អនុវត្តគោលការណ៍សុវត្ថិភាពដែលផ្លាស់ប្តូរបានតាមបរិបទហានិភ័យបច្ចុប្បន្ន

   - **ការចូលរួមសហគមន៍សុវត្ថិភាព**៖ ចូលរួមសកម្មក្នុងសហគមន៍សុវត្ថិភាព MCP និងកម្មវិធីបង្ហាញចំណុចខ្សោយ

**សុវត្ថិភាពបត់បែន:**
   - **សុវត្ថិភាពប aprendizado ម៉ាស៊ីន**៖ ប្រើការរកឃើញអសកម្មភាពលើមូលដ្ឋានម៉ាស៊ីនសម្រាប់រកមើលរបៀបវាយប្រហារថ្មី
   - **វិភាគសុវត្ថិភាពបន្តិចបន្តួច**៖ អនុវត្តម៉ូឌែលទាយទ្រាប់សម្រាប់គោលបំណងរំពេចហានិភ័យ
   - **ស្វ័យប្រវត្តិកម្មសុវត្ថិភាព**៖ បច្ចុប្បន្នភាពគោលការណ៍សុវត្ថិភាពដោយស្វ័យប្រវត្តិតាមព័ត៌មានហានិភ័យ និងកែប្រែបរិវេណ
