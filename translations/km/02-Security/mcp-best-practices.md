# MCP Security Best Practices 2025

មគ្គុទេសក៍សម្ភាសន៍នេះបង្ហាញពីការអនុវត្តន៍លទ្ធផលល្អបំផុតនៃសុវត្ថិភាពសម្រាប់ការអនុវត្តប្រព័ន្ធ Model Context Protocol (MCP) ដោយផ្អែកលើ **MCP Specification 2025-11-25** និងស្តង់ដារផ្លូវការបច្ចុប្បន្ន។ អនុវត្តន៍ទាំងនេះគ្របដណ្តប់ពីបញ្ហាសុវត្ថិភាពពីបុរាណ និងគ្រឿងអាក់ខ្លាចមានលក្ខណៈពិសេសដល់ AI ដែលពាក់ព័ន្ធនឹងការចេញផ្សាយ MCP។

## តម្រូវការសុវត្ថិភាពសំខាន់ៗ

### ការគ្រប់គ្រងសុវត្ថិភាពដែលត្រូវធ្វើ (តម្រូវការ MUST)

1. **ការបញ្ជាក់ Token**: ម៉ាស៊ីនមេ MCP **មិនត្រូវ** ទទួលយក token ដែលមិនបានចេញអោយសម្រាប់ម៉ាស៊ីនមេ MCP នោះទេ
2. **ការបញ្ជាក់អាថ៌កំបាំង**: ម៉ាស៊ីនមេ MCP ដែលអនុវត្តអាថ៌កំបាំង **ត្រូវ** បញ្ជាក់សំណើទាំងអស់និង **មិនត្រូវ** ប្រើសម័យសំឡេងសម្រាប់ការផ្ទៀងផ្ទាត់
3. **ការយល់ព្រមរបស់អ្នកប្រើ**: ម៉ាស៊ីនភីរឹក MCP ដែលប្រើ ID לקוחម៉ាស៊ីនថេរ **ត្រូវ** ទទួលយកការយល់ព្រមច្បាស់លាស់ពីអ្នកប្រើក្នុងការចុះបញ្ជីអតិថិជនរាងរ៉ៃណាមិចនីក
4. **លេខសម្គាល់សម័យសុវត្ថិភាព**: ម៉ាស៊ីនមេ MCP **ត្រូវ** ប្រើលេខសម្គាល់សម័យដែលមានសុវត្ថិភាពគ្រប់គ្រាន់បង្កើតដោយម៉ាស៊ីនបង្កើតលេខចៃដន្យមានសុវត្ថិភាព

## អនុវត្តន៍សុវត្ថិភាពស្នូល

### 1. ការបញ្ជាក់ និងបំលែងទិន្នន័យចូល
- **បញ្ជាក់ទិន្នន័យចូលពេញលេញ**៖ បញ្ជាក់ និងបំលែងទិន្នន័យចូលទាំងអស់ដើម្បីទប់ស្កាត់ការវាយប្រហារជាតិក literally, បញ្ហាទីបំភ្លឺគំនិតហើយនិងក្រុមហ៊ុន injection ប្រយោជន៍
- **ការអនុវត្ត schema ប៉ារ៉ាម៉ែត្រ**៖ អនុវត្តការបញ្ជាក់ schema JSON ដែលតឹងរ៉ឹងសម្រាប់ប៉ារ៉ាម៉ែត្រឧបករណ៍ទាំងអស់និង API ចូល
- **ការត្រង់ប្លែកមាតិកា**៖ ប្រើ Microsoft Prompt Shields និង Azure Content Safety ដើម្បីត្រង់មាតិកាដែលមានគ្រោះថ្នាក់ទៅទីបញ្ចូលនិងចម្លើយ
- **ការបំលែងលទ្ធផលចេញ**៖ បញ្ជាក់ និងបំលែងលទ្ធផលម៉ូដែលទាំងអស់មុនបង្ហាញទៅអ្នកប្រើឬប្រព័ន្ធក្រោម

### 2. ការផ្ទៀងផ្ទាត់ និងការអនុញ្ញាតល្អបំផុត  
- **អ្នកផ្តល់អត្តសញ្ញាណខាងក្រៅ**៖ ផ្ដល់ការផ្ទៀងផ្ទាត់ទៅអ្នកផ្តល់អត្តសញ្ញាណដែលបានបង្កើត (Microsoft Entra ID, អ្នកផ្ដល់ OAuth 2.1) ជំនួសការអនុវត្តផ្ទាល់ខ្លួន
- **ការអនុញ្ញាតជាមធ្យម**៖ អនុវត្តអាថ៌កំបាំងលម្អិតលើឧបករណ៍ជាក់លាក់ដោយយោងតាមគោលការណ៍របៀបតិចបំផុត
- **ការគ្រប់គ្រងរយៈពេលពណ៌តខ្លីនៃ Token**៖ ប្រើ token ដែលមានរយៈពេលខ្លីជាមួយការបង្វិលសុវត្ថិភាពនិងការបញ្ជាក់ទស្សនកិច្ចបានត្រឹមត្រូវ
- **ការផ្ទៀងផ្ទាត់ពីរ-factor**៖ ទាមទារការផ្ទៀងផ្ទាត់ពីរ-factor សម្រាប់ការចូលគ្រប់គ្រងនិងប្រតិបត្តិការសំខាន់ៗ

### 3. ប្រព័ន្ធផ្ទេរសារសុវត្ថិភាព
- **សុវត្ថិភាពក្នុងផ្នែកផ្ទេរសារប្រើ HTTPS/TLS 1.3** ជាមួយការបញ្ជាក់សម្រង់ឯកសារបានត្រឹមត្រូវ
- **ការអ៊ិនគ្រីបបញ្ចប់ដល់បញ្ចប់**៖ អនុវត្តស្រទាប់អ៊ិនគ្រីបបន្ថែមសម្រាប់ទិន្នន័យមានសុវត្ថិភាពខ្ពស់កម្រិតទាំងឡាយក្នុងដំណើរផ្ទេរ និងផ្ទុក
- **ការគ្រប់គ្រងសញ្ញាប័ត្រ**៖ រក្សាបានប្រសិទ្ធភាពជាមួយប្រព័ន្ធផ្លាស់ប្ដូរបានស្វ័យប្រវត្តិកម្ម
- **ការបញ្ជាក់កំណែប្រតិបត្តិការ**៖ ប្រើកំណែ MCP ពិចារណាចុងក្រោយ (2025-11-25) ជាមួយការចរចាកំណែបានត្រឹមត្រូវ។

### 4. ការគ្រប់គ្រងកម្រិតបណ្ដោះអាសន្ន និងការពារធនធាន
- **កម្រិតបណ្ដោះអាសន្នជាច្រើនស្រទាប់**៖ អនុវត្តការគ្រប់គ្រងកម្រិតចាប់ពីអ្នកប្រើ សម័យសំឡេង ឧបករណ៍ និងធនធាន ដើម្បីទប់ស្កាត់ការប្រើប្រាស់លើសកំណត់
- **កម្រិតបណ្ដោះអាសន្នឆ្លើយតប**៖ ប្រើការសិក្សាម៉ាស៊ីនសម្រាប់កម្រិតបណ្ដោះអាសន្នដែលឆ្លើយតបលើបញ្ហារូបមន្តនិងសញ្ញាគំរាម
- **ការគ្រប់គ្រងបរិមាណធនធាន**៖ កំណត់ដែនកំណត់សមរម្យសម្រាប់ធនធានគណនា ការប្រើប្រាស់អង្គចងចាំ និងពេលវេលាចាត់ចែង
- **ការពារប្រឆាំងចម្លាក់ទៅ DDoS**៖ ដាក់បដិសណ្ឋារកិច្ចប្រព័ន្ធការពារDDoS និងការវិភាគចរាចរណ៍ទាំងមូល

### 5. ការចុះបញ្ជី និងត្រួតពិនិត្យដំណើរការ
- **កំណត់ហេតុ Audit មានរចនាសម្ព័ន្ធ**៖ អនុវត្តកំណត់ហេតុលម្អិតដែលអាចស្វែងរកបានសម្រាប់ដំណើរការ MCP ឧបករណ៍ និងព្រឹត្តិការណ៍សុវត្ថិភាពទាំងអស់
- **ត្រួតពិនិត្យសុវត្ថិភាពពេលវេលាពិតប្រាកដ**៖ ដាក់ប្រព័ន្ធ SIEM ជាមួយសមត្ថភាព AI សម្រាប់រកឃើញអំពើអាក្រក់ក្នុងការបំពេញការងារ MCP
- **កំណត់ហេតុគោលការណ៍ឯកជនភាព**៖ ចុះបញ្ជីព្រឹត្តិការណ៍សុវត្ថិភាពដោយគោរពតាមតម្រូវការមានឯកជនភាពនិងច្បាប់គោលការណ៍
- **ការតភ្ជាប់ទៅកាន់ការឆ្លើយតបគ្រោះហន្តរាយ**៖ ភ្ជាប់ប្រព័ន្ធកំណត់ហេតុទៅនឹងហ្វ្លូវវើកប្រតិបត្តិការ​ដើម្បីឆ្លើយតបគ្រោះហន្តរាយស្វ័យប្រវត្តិ

### 6. អនុវត្តផ្ទុកដាតាសុវត្ថិភាព
- **ម៉ូឌុលសុវត្ថិភាពរឹង (HSM)**៖ ប្រើតំបន់ផ្ទុកកូនសោ HSM (Azure Key Vault, AWS CloudHSM) សម្រាប់ប្រតិបត្តិការអ៊ិនគ្រីបសំខាន់ៗ
- **ការគ្រប់គ្រងកូនសោអ៊ិនគ្រីប**៖ អនុវត្តការបង្វិល ការបំបែក និងគ្រប់គ្រងចូលប្រើដែលត្រឹមត្រូវសម្រាប់កូនសោអ៊ិនគ្រីប
- **ការគ្រប់គ្រងសម្ងាត់**៖ រក្សាទុកកូនសោ API, token និងសញ្ញាបត្រទាំងអស់នៅក្នុងប្រព័ន្ធគ្រប់គ្រងសម្ងាត់ជាពិសេស
- **ការបែងចែកទិន្នន័យ**៖ ចាត់ថ្នាក់ទិន្នន័យដោយផ្អែកលើកម្រិតអារម្មណ៍ហើយអនុវត្តវិធានការពារយ៉ាងសមរម្យ

### 7. ការគ្រប់គ្រង Token ជាក់លាក់
- **ការទប់ស្កាត់ការចេញបញ្ជាលtoken**៖ ធ្វើយឲ្យច្បាស់គ្រប់គ្រងលំនាំវេចខ្ចប់ token ដែលល្មើសច្បាប់គ្រប់គ្រងសុវត្ថិភាព
- **ការបញ្ជាក់ទស្សនកិច្ច**៖ សូមត្រួតពិនិត្យ claims ទស្សនកិច្ច token ជានិច្ចឲ្យត្រូវនឹងអត្តសញ្ញាណម៉ាស៊ីនមេ MCP
- **អាថ៌កំបាំងលើ claims**៖ អនុវត្តការអនុញ្ញាតលម្អិតដែលផ្អែកលើ claims និងលក្ខណៈអ្នកប្រើ
- **ការ Binding Token**៖ តភ្ជាប់ token ចូលទៅនឹងសម័យអ្នកប្រើ ឬឧបករណ៍ដែលសមរម្យ

### 8. ការគ្រប់គ្រងសម័យសុវត្ថិភាព
- **លេខសម្គាល់សម័យអ៊ិនគ្រីប**៖ បង្កើតលេខសម្គាល់សម័យដោយប្រើម៉ាស៊ីនបង្កើតលេខចៃដន្យមានសុវត្ថិភាពគ្រប់គ្រាន់ (មិនអាចទាយមុន)
- **ការតភ្ជាប់បុគ្គល**៖ តភ្ជាប់លេខសម្គាល់សម័យទៅជាមួយព័ត៌មានបុគ្គលដោយប្រើទ្រង់ទ្រាយមានសុវត្ថិភាពដូចជា `<user_id>:<session_id>`
- **ការគ្រប់គ្រងអាយុកាលសម័យ**៖ អនុវត្តកំណត់អស់សព្វវ័យ ការបង្វិល និងការលុបសម័យបន្ថែម
- **ក្បាលសុវត្ថិភាពសម័យ**៖ ប្រើក្បាលសុវត្ថិភាព HTTP សមរម្យសម្រាប់ការពារសម័យ

### 9. ការគ្រប់គ្រងសុវត្ថិភាពពិសេស AI
- **ការការពារចាក់បញ្ចូលបញ្ហា Prompt Injection**៖ ដាក់ប្រើ Microsoft Prompt Shields ជាមួយការបំភ្លឺ spotlighting, delimiters និង datamarking
- **ការការពារបំពុលឧបករណ៍ Tool Poisoning**៖ ត្រួតពិនិត្យព័ត៌មាន metadata ឧបករណ៍ តាមដានការផ្លាស់ប្តូររាងរ៉ៃណាមិច ហើយបញ្ជាក់សេចក្តីពិតរបស់ឧបករណ៍
- **ការបញ្ជាក់លទ្ធផលម៉ូដែល**៖ ស្កេនលទ្ធផលម៉ូដែលសម្រាប់ការលេចធ្លាយទិន្នន័យ ខ្លឹមសារបំពានឬការបំពានគោលការណ៍សុវត្ថិភាព
- **ការពារវិនាទំព័របរិបទ Context Window**៖ អនុវត្តការគ្រប់គ្រងដើម្បីទប់ស្កាត់ការបំពុលនិងការវាយប្រហារវិញវិល Context Window

### 10. សុវត្ថិភាពការបំពេញការងារឧបករណ៍
- **បំពេញការងារឧបករណ៍ក្នុងធុង sandbox**៖ ប្រតិបត្តិការឧបករណ៍ក្នុងបរិយាកាស isolated មានកំណត់ធនធាន
- **ការបំបែកអាទិភាព Privilege Separation**៖ បំពេញកិច្ចការជាមួយអាទិភាពតិចបំផុត ត្រួតពិនិត្យគណនីសេវា
- **បំបែកបណ្តាញ Network Isolation**៖ អនុវត្តការបំបែកបណ្តាញសម្រាប់បរិយាកាសបំពេញការងារ
- **ត្រួតពិនិត្យការបំពេញការងារ Execution Monitoring**៖ តាមដានឧបករណ៍សម្រាប់អាកប្បកិរិយា ភាពប្រើធនធាន និងការបំពានសុវត្ថិភាព

### 11. ការផ្ទៀងផ្ទាត់សុវត្ថិភាពបន្តជាបន្តបន្ទាប់
- **ការធ្វើតេស្តសុវត្ថិភាពស្វ័យប្រវត្តិ**៖ ផ្សំចូលការធ្វើតេស្តសុវត្ថិភាពទៅក្នុងបំពង់ CI/CD ជាមួយឧបករណ៍ដូចជា GitHub Advanced Security
- **ការគ្រប់គ្រងអេហ្ស៊ីប៊ីលធី Vulnerability**៖ ស្កេនអាស្រ័យអំពើទាំងអស់ជាទៀងទាត់ រួមទាំងម៉ូដែល AI និងសេវាកម្មខាងក្រៅ
- **ការធ្វើតេស្តប៉ុនប៉ងចូល Penetration Testing**៖ បំពេញការវាយតំរៀបសុវត្ថិភាពជាទៀងទាត់សម្រាប់អនុវត្ត MCP
- **ការត្រួតពិនិត្យកូដសុវត្ថិភាព**៖ អនុវត្តការត្រួតពិនិត្យសុវត្ថិភាពចាំបាច់សម្រាប់ការផ្លាស់ប្តូរកូដទាក់ទង MCP ទាំងអស់

### 12. សុវត្ថិភាពខ្សែផ្គត់ផ្គង់ក្នុង AI
- **ការត្រួតពិនិត្យផ្នែក**៖ ត្រួតពិនិត្យប្រភព ប្រកបដោយភាពស្រាល និងសុវត្ថិភាពនៃផ្នែក AI ទាំងអស់ (ម៉ូដែល, ការបង្ហាញ, API)
- **ការគ្រប់គ្រងអាស្រ័យ**៖ រក្សាបញ្ជីនៅទាន់សម័យនៃកម្មវិធី និងអាស្រ័យ AI ទាំងអស់ជាមួយការតាមដានអេហ្ស៊ីប៊ីលធី
- **ឃ្លាំងទុកមានទំនុកចិត្ត**៖ ប្រើប្រភពដែលបានបញ្ជាក់ និងទំនុកចិត្តសម្រាប់ម៉ូដែល AI បណ្ណាល័យ និងឧបករណ៍
- **ការត្រួតពិនិត្យខ្សែផ្គត់ផ្គង់ Supply Chain**៖ ត្រួតពិនិត្យជាប្រចាំពីការលក់ចោលនៅក្នុងអ្នកផ្គត់ផ្គង់សេវា AI និងឃ្លាំងម៉ូដែល

## លំនាំសុវត្ថិភាពលំដាប់ខ្ពស់

### ស្ថាបត្យកម្ម Zero Trust សម្រាប់ MCP
- **មិនធ្វើជឿលើនរណាទេ តែត្រូវផ្ទៀងផ្ទាត់ជានិច្ច**៖ អនុវត្តការផ្ទៀងផ្ទាត់ជាបន្តបន្ទាប់សម្រាប់អ្នកចូលរួម MCP ទាំងអស់
- **កាត់បន្ថយបណ្តាញ Micro-segmentation**៖ បំបែកផ្នែក MCP ជាមួយការគ្រប់គ្រងបណ្តាញ និងអត្តសញ្ញាណយ៉ាងម៉ឺងម៉ាត់
- **ការចូលប្រើដោយលក្ខខណ្ឌ Conditional Access**៖ ចាត់វិធានការចូលប្រើដោយផ្អែកលើហានិភ័យដែលផ្លាស់ប្តូរតាមបរិបទ និងអាកប្បកិរិយា
- **ការវាយតម្លៃហានិភ័យជាបន្តបន្ទាប់ Continuous Risk Assessment**៖ វាយតម្លៃសន្តិសុខប្រភេទថ្នាក់លើស្ដីពីសញ្ញាគំរាមបច្ចុប្បន្ន

### ការអនុវត្ត AI រក្សាឯកជនភាព
- **កាត់បន្ថយទិន្នន័យ Data Minimization**៖ បង្ហាញទិន្នន័យតិចតួចដែលចាំបាច់សម្រាប់ដំណើរការ MCP រាល់ដំណើរ
- **ភាពឯកជនភាពផ្សេងៗ Differential Privacy**៖ អនុវត្តវិធីសាស្រ្តរក្សាទុកភាពឯកជនភាពសម្រាប់ដំណើរការទិន្នន័យសំខាន់
- **អ៊ិនគ្រីបហូមូម៉ូភិក Homomorphic Encryption**៖ ប្រើវិធីសាស្រ្តអ៊ិនគ្រីបដំណើរការលើទិន្នន័យអ៊ិនគ្រីប
- **ការសិក្សាក្នុងរបៀបផ្សព្វផ្សាយ Federated Learning**៖ អនុវត្តរបៀបសិក្សាបញ្ចាយដែលគាំទ្រការរក្សាទិន្នន័យនៅកន្លែងដើមនិងភាពឯកជនភាព

### ប្រតិបត្តិការឆ្លើយតបគ្រប់ហេតុការណ៍សម្រាប់ប្រព័ន្ធ AI
- **នីតិវិធីប្រតិបត្តិការឆ្លើយតបគ្រប់ហេតុការណ៍ AI**៖ រៀបចំនីតិវិធីឆ្លើយតបសំរាប់គំរាមមេរោគ AI និង MCP
- **ការឆ្លើយតបដោយស្វ័យប្រវត្តិ**៖ អនុវត្តការចាត់ចែង និងជៀសរញៅដោយស្វ័យប្រវត្តិសម្រាប់ហេតុការណ៍សុវត្ថិភាព AI ជាទូទៅ  
- **សមត្ថភាពជម្រះភេរវកម្ម**៖ រក្សាការរៀបចំសម្រាប់ពិនិត្យផ្នែកជម្រះភេរវកម្មពេលបំផ្លាញប្រព័ន្ធ AI និងការលេចធ្លាយទិន្នន័យ
- **នីតិវិធីស្ដារឡើងវិញ**៖ បង្កើតនីតិវិធីសម្រាប់ស្ដារឡើងវិញពីការបំពុលម៉ូដែល AI, ការវាយប្រហារ prompt injection និងការបំពានសេវា

## 🔄 ការកែលម្អជាបន្តបន្ទាប់

### សូមបន្តតាមដានស្តង់ដារកំពុងអភិវឌ្ឍ
- **ការអាប់ដេតលំអិតពី MCP**៖ តាមដានការផ្លាស់ប្តូរលំអិតពី MCP ស្រូបយកកែប្រែ និងអត្ថបទអំពីសុវត្ថិភាព
- **ព័ត៌មានព័ត៌មានអំពីគំរាមកំហែង**៖ សាវកទំព័រដើម្បីទទួលបានព័ត៌មានអំពីគំរាមកំហែង AI និងមូលដ្ឋានទិន្នន័យចំណុចងាយរងគ្រោះ
- **ការចូលរួមសហគមន៍**៖ ចូលរួមក្នុងការពិភាក្សានិងក្រុមការងារសុវត្ថិភាព MCP
- **ការវាយតម្លៃទៀងទាត់**៖ បញ្ជាប្រចាំត្រីមាសស្តីពីសភាពសុវត្ថិភាព និងបន្ថែមកិច្ចប្រតិបត្តិការដោយសមស្រប

### ការរួមចំណែកក្នុងសុវត្ថិភាព MCP
- **ការស្រាវជ្រាវសុវត្ថិភាព**៖ រួមចំណែកក្នុងការស្រាវជ្រាវសុវត្ថិភាព MCP និងកម្មវិធីបង្ហាញចំណុចងាយរងគ្រោះ
- **ចែករំលែកវិធីអនុវត្តល្អបំផុត**៖ ចែករំលែកការអនុវត្តសុវត្ថិភាព និងការបង្រៀនដែលបានសិក្សាជាមួយសហគមន៍
- **ការអភិវឌ្ឍន៍ស្តង់ដារ**៖ ចូលរួមក្នុងការអភិវឌ្ឍលំអិត MCP និងការបង្កើតស្តង់ដារសុវត្ថិភាព
- **ការអភិវឌ្ឍឧបករណ៍**៖ អភិវឌ្ឍន៍ និងចែករំលែកឧបករណ៍សុវត្ថិភាព និងបណ្ណាល័យសម្រាប់ប្រព័ន្ធបរិស្ថាន MCP
