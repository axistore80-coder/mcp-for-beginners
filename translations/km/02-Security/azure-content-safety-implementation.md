# អនុវត្តភាពសុវត្ថិភាពខ្លឹមសាររបស់ Azure ជាមួយ MCP

> **ហានិភ័យ OWASP MCP ដែលបានដោះស្រាយ**: [MCP06 - ការចាក់បញ្ចូលសំណើបែប Contextual Payloads](https://microsoft.github.io/mcp-azure-security-guide/mcp/mcp06-prompt-injection/)

ដើម្បីបង្កើតកម្លាំងសុវត្ថិភាព MCP ទល់នឹងការចាក់បញ្ចូលសំណើបែប prompt injection, ការប៉ះពាល់ឧបករណ៍, និងហានិភ័យពិសេស AI ផ្សេងទៀត គឺសូមណែនាំឱ្យបញ្ចូល Azure Content Safety។ មេរៀនអនុវត្តនេះស្របទៅនឹង [កម្មវិធី MCP Security Summit Workshop (Sherpa)](https://azure-samples.github.io/sherpa/) ខែប 3៖ សុវត្ថិភាព I/O។

## ការបញ្ចូលជាមួយម៉ាស៊ីនបម្រើ MCP

ដើម្បីបញ្ចូល Azure Content Safety ជាមួយម៉ាស៊ីនបម្រើ MCP របស់អ្នក សូមបន្ថែមតម្រៀបធារាសុវត្ថិភាពខ្លឹមសារជាមធ្យមភាពក្នុងបំពង់ដំណើរការសំណើររបស់អ្នក៖

1. ចាប់ផ្តើមតម្រៀបនៅពេលម៉ាស៊ីនបម្រើចាប់ផ្តើម
2. បញ្ជាក់សុវត្ថិភាពលើសំណើឧបករណ៍ទាំងអស់មុនដំណើរការ
3. ពិនិត្យនូវចម្លើយទាំងអស់មុនផ្ទេរវាមកអ្នកប្រើ
4. កត់ត្រានិងដាក់ជូនដំណឹងបញ្ហាសុវត្ថិភាព
5. អនុវត្តការដោះស្រាយកំហុសសមរម្យនៅពេលមានការត្រួតពិនិត្យសុវត្ថិភាពខ្លឹមសារបរាជ័យ

នេះផ្តល់ការការពារដែលរឹងមាំចំពោះ៖
- ការបោកបញ្ចូលសំណើ prompt injection
- ការប៉ះពាល់ឧបករណ៍
- ការរង្វាស់ទិន្នន័យតាមការបញ្ចូលជាចោរយល់
- ការបង្កើតខ្លឹមសារឥតបានសុខ

## អនុវត្តន៍ល្អបំផុតសម្រាប់ការបញ្ចូល Azure Content Safety

1. **បញ្ជីបិទផ្ទាល់ខ្លួន**: បង្កើតបញ្ជីបិទផ្ទាល់ខ្លួនជាពិសេសសម្រាប់ម៉ូទ័រចាក់ច្រក MCP
2. **កម្រិតកង្វះខាត**: សម្រួលកម្រិតកង្វះខាតផ្អែកលើករណីប្រើប្រាស់ច្បាស់លាស់ និងការទទួលទានហានិភ័យរបស់អ្នក
3. **ការត្រួតពិនិត្យទូលំទូលាយ**: អនុវត្តការពិនិត្យសុវត្ថិភាពខ្លឹមសារទៅលើការបញ្ចូល និងចេញទាំងអស់
4. **ប្រសិទ្ធភាពការប្រតិបត្តិការ**: ផ្ដល់ការគិតគូរសម្រាប់ការផ្ទុកទុកនូវការពិនិត្យសុវត្ថិភាពខ្លឹមសារដែលត្រូវធ្វើឡើងជាពន្ធ័
5. **វិធីសាស្រ្តបំផ្លាញបញ្ហា**: កំណត់អាកប្បកិរិយាក្នុងករណីសេវាកម្មសុវត្ថិភាពខ្លឹមសារមិនអាចប្រើបាន
6. **ចែករំលែកមតិយោបល់រាល់អ្នកប្រើប្រាស់**: ផ្តល់មតិយោបល់ច្បាស់លាស់ទៅអ្នកប្រើនៅពេលដែលខ្លឹមសារត្រូវបានបិទដោយសារភាពសុវត្ថិភាព
7. **ការកែលម្អរបៀបបន្ត**: ធ្វើបច្ចុប្បន្នភាពបញ្ជីបិទ និងម៉ូទ័រជាថ្មីជាបន្តបន្ទាប់មកពីមានគ្រោះថ្នាក់កើតមកថ្មីៗ
