# ឧទាហរណ៍ MCP Client ពេញលេញ

ថតនេះមានឧទាហរណ៍ MCP client ពេញលេញ និងដំណើរការបានជាភាសាកម្មវិធីផ្សេងៗគ្នា។ ពីរបាយការណ៍ client នីមួយៗបង្ហាញពីមុខងារពេញលេញដែលបានពិពណ៌នានៅក្នុងមេរៀន README.md សំខាន់។

## Client ដែលមានស្រាប់

### 1. Java Client (`client_example_java.java`)

- **ការដឹកជញ្ជូន**: SSE (Server-Sent Events) តាម HTTP
- **ម៉ាស៊ីនបម្រើគោលដៅ**: `http://localhost:8080`
- **មុខងារ**:
  - ការតភ្ជាប់ និង ping
  - ច្រកបញ្ជីឧបករណ៍
  - ប្រតិបត្ដិការគណនា (បូក, លុប, គុណ, ដាក់, ជំនួយ)
  - ដោះស្រាយកើតខុស និងបង្កើតលទ្ធផល

**ដើម្បីរត់:**

```bash
# ប្រាកដថាសេវាកម្ម MCP របស់អ្នកកំពុងដំណើរការនៅលើ localhost:8080
javac client_example_java.java
java client_example_java
```

### 2. C# Client (`client_example_csharp.cs`)

- **ការដឹកជញ្ជូន**: Stdio (ការបញ្ចូល/បញ្ចេញស្តង់ដារ)
- **ម៉ាស៊ីនបម្រើគោលដៅ**: ម៉ាស៊ីនបម្រើ .NET MCP នៅក្នុងតំបន់តាមការរត់ dotnet
- **មុខងារ**:
  - ការចាប់ផ្តើមម៉ាស៊ីនបម្រើដោយស្វ័យប្រវត្តិតាមការដឹកជញ្ជូន stdio
  - ច្រកបញ្ជីឧបករណ៍ និងធនធាន
  - ប្រតិបត្ដិការគណនា
  - វិភាគលទ្ធផល JSON
  - ដោះស្រាយកំហុសបានយ៉ាងពេញលេញ

**ដើម្បីរត់:**

```bash
dotnet run
```

### 3. TypeScript Client (`client_example_typescript.ts`)

- **ការដឹកជញ្ជូន**: Stdio (ការបញ្ចូល/បញ្ចេញស្តង់ដារ)
- **ម៉ាស៊ីនបម្រើគោលដៅ**: ម៉ាស៊ីនបម្រើ Node.js MCP នៅក្នុងតំបន់
- **មុខងារ**:
  - គាំទ្រពេញលេញនូវពិធីការម៉ាស៊ីនបម្រើ MCP
  - ប្រតិបត្ដិការឧបករណ៍, ធនធាន និងការបញ្ចេញស្នើរ
  - ប្រតិបត្ដិការ គណនា
  - ការអានធនធាន និងអនុវត្តស្នើរ
  - ដោះស្រាយកំហុសបានយ៉ាងរឹងមាំ

**ដើម្បីរត់:**

```bash
# រួច compile TypeScript ជំនួស (បើចាំបាច់)
npm run build

# បន្ទាប់មករត់ client
npm run client
# ឬ
node client_example_typescript.js
```

### 4. Python Client (`client_example_python.py`)

- **ការដឹកជញ្ជូន**: Stdio (ការបញ្ចូល/បញ្ចេញស្តង់ដារ)  
- **ម៉ាស៊ីនបម្រើគោលដៅ**: ម៉ាស៊ីនបម្រើ Python MCP នៅក្នុងតំបន់
- **មុខងារ**:
  - លំនាំ async/await សម្រាប់ប្រតិបត្ដិការផ្សេងៗ
  - ស្វែងរកឧបករណ៍ និងធនធាន
  - សាកល្បងប្រតិបត្ដិការគណនា
  - អានមាតិកាធនធាន
  - រៀបចំជាក្លាស

**ដើម្បីរត់:**

```bash
python client_example_python.py
```

## មុខងាររួមគ្នារវាង Clients ទាំងអស់

ការអនុវត្ត client នីមួយៗបង្ហាញពី៖

1. **ការគ្រប់គ្រងការតភ្ជាប់**
   - ការបង្កើតការតភ្ជាប់ទៅម៉ាស៊ីនបម្រើ MCP
   - ដោះស្រាយកំហុសការតភ្ជាប់
   - ការសំអាតដោយត្រឹមត្រូវ និងការគ្រប់គ្រងធនធាន

2. **ការស្វែងរកម៉ាស៊ីនបម្រើ**
   - បញ្ជីឧបករណ៍ដែលមាន
   - បញ្ជីធនធាន (នៅពេលគាំទ្រ)
   - បញ្ជីស្នើរ (នៅពេលគាំទ្រ)

3. **ការហៅប្រើឧបករណ៍**
   - ប្រតិបត្ដិការគណនាគ្រឹះ (បូក, លុប, គុណ, ដាក់)
   - ពាក្យបញ្ជាជំនួយសម្រាប់ព័ត៌មានម៉ាស៊ីនបម្រើ
   - ការផ្ទេរពីរឯកសារប្រយោជន៏និងការដោះស្រាយលទ្ធផលបានត្រឹមត្រូវ

4. **ការដោះស្រាយកំហុស**
   - កំហុសការតភ្ជាប់
   - កំហុសកើតឡើងក្នុងការប្រតិបត្ដិការឧបករណ៍
   - ការផ្លាស់ប្តូរដោយគ្មានកំហុស និងមតិយោបល់ទៅអ្នកប្រើប្រាស់

5. **ការតំរូវផល**
   - ដកយកមាតិកា​អត្ថបទពីការឆ្លើយតប
   - រៀបចំលទ្ធផលសម្រាប់អាន
   - ដោះស្រាយទ្រង់ទ្រាយការឆ្លើយតបផ្សេងៗ

## លក្ខខណ្ឌមុន

មុនការរត់ client ទាំងនេះ សូមប្រាកដថា៖

1. **ម៉ាស៊ីនបម្រើ MCP ស្របតាមគោលនីយាទៀត** (ពី `../01-first-server/`)
2. **បានដំឡើងអាសយដ្ឋានទាមទារសម្រាប់ភាសារបស់អ្នក**
3. **មានការតភ្ជាប់បណ្តាញត្រឹមត្រូវ** (សម្រាប់ការដឹកជញ្ជូនផ្អែកលើ HTTP)

## ខុសគ្នាសំខាន់រវាងការអនុវត្ត

| ភាសា      | ការដឹកជញ្ជូន | ការចាប់ផ្តើមម៉ាស៊ីនបម្រើ | គំរូ Async | បណ្ណាល័យសំខាន់ៗ        |
|------------|---------------|----------------------------|------------|-------------------------|
| Java       | SSE/HTTP      | ចុងក្រោយក្រៅ              | ស៊ីញ      | WebFlux, MCP SDK        |
| C#         | Stdio         | ស្វ័យប្រវត្តិ               | Async/Await| .NET MCP SDK            |
| TypeScript | Stdio         | ស្វ័យប្រវត្តិ               | Async/Await| Node MCP SDK            |
| Python     | Stdio         | ស្វ័យប្រវត្តិ               | AsyncIO    | Python MCP SDK          |
| Rust       | Stdio         | ស្វ័យប្រវត្តិ               | Async/Await| Rust MCP SDK, Tokio     |

## ជំហុងបន្ទាប់

បន្ទាប់ពីស្ទូចនិងមើលឧទាហរណ៍ client ទាំងនេះ៖

1. **កែប្រែ client** ដើម្បីបន្ថែមមុខងារឬប្រតិបត្ដិការ
2. **បង្កើតម៉ាស៊ីនបម្រើរបស់អ្នកផ្ទាល់** ហើយសាកល្បងជាមួយ client ទាំងនេះ
3. **សាកល្បងការដឹកជញ្ជូនផ្សេងៗ** (SSE ទល់នឹង Stdio)
4. **កសាងកម្មវិធីស្មុគស្មាញជាងនេះ** ដែលបញ្ចូលមុខងារ MCP

## ដោះស្រាយបញ្ហា

### បញ្ហាពេញនិយម

1. **ការតភ្ជាប់ត្រូវបានបដិសេធ**: សូមប្រាកដថាម៉ាស៊ីនបម្រើ MCP បានដំណើរការលើព្រិល/ផ្លូវដែលរំពឹងទុក
2. **មូឌុលមិនមាន**: ដំឡើង MCP SDK តាមភាសារបស់អ្នក
3. **ការអនុញ្ញាតអស់សុពលភាព**: ពិនិត្យការអនុញ្ញាតឯកសារសម្រាប់ការដឹកជញ្ជូន stdio
4. **ឧបករណ៍មិនមាន**: ផ្ទៀងផ្ទាត់ថាម៉ាស៊ីនបម្រើបានអនុវត្តឧបករណ៍ដែលរំពឹងទុក

### ជំនួយសម្គាល់កំហុស

1. **បើកកំណត់ហេតុឆ្នោតក្នុង MCP SDK របស់អ្នក**
2. **ពិនិត្យកំណត់ហេតុម៉ាស៊ីនបម្រើ** សម្រាប់សារ ខុស
3. **ផ្ទៀងផ្ទាត់ឈ្មោះឧបករណ៍ និងហត្ថលេខា** ត្រូវគ្នារវាង client និងម៉ាស៊ីនបម្រើ
4. **សាកល្បងជាមួយ MCP Inspector** ជាលើកដំបូងដើម្បីផ្ទៀងផ្ទាត់មុខងារម៉ាស៊ីនបម្រើ
