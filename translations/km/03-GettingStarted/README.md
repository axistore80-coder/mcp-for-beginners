## ការចាប់ផ្តើម  

[![Build Your First MCP Server](../../../translated_images/km/04.0ea920069efd979a.webp)](https://youtu.be/sNDZO9N4m9Y)

_(ចុចរូបភាពខាងលើដើម្បីមើលវីដេអូមេរៀននេះ)_

ផ្នែកនេះមានមេរៀនច្រើន៖

- **1 ម៉ាស៊ីនមេដំបូងរបស់អ្នក** នៅក្នុងមេរៀនដំបូងនេះ អ្នកនឹងរៀនពីរបៀបបង្កើតម៉ាស៊ីនមេដំបូងរបស់អ្នក និងពិនិត្យវាមួយជាមួយឧបករណ៍ប៉ាន់ប្រមាណ ដែលជាវិធីដ៏មានតម្លៃសម្រាប់សាកល្បង និងកែលម្អម៉ាស៊ីនមេរបស់អ្នក [ទៅមេរៀន](01-first-server/README.md) 

- **2 អតិថិជន** នៅក្នុងមេរៀននេះ អ្នកនឹងរៀនពីរបៀបសរសេរអតិថិជនដែលអាចភ្ជាប់ទៅម៉ាស៊ីនមេរបស់អ្នក [ទៅមេរៀន](02-client/README.md)

- **3 អតិថិជនជាមួយ LLM** វិធីសាស្រ្តល្អជាងនេះ សម្រាប់សរសេរអតិថិជនគឺ ដោយបន្ថែម LLM ចូលទៅ ដូច្នេះវាអាច "ចរចា" ជាមួយម៉ាស៊ីនមេរបស់អ្នកថាតើត្រូវធ្វើអ្វី [ទៅមេរៀន](03-llm-client/README.md)

- **4 ការប្រើម៉ូដ GitHub Copilot Agent សម្រាប់ម៉ាស៊ីនមេក្នុង Visual Studio Code** នៅទីនេះ យើងកំពុងមើលការរត់ម៉ាស៊ីនមេ MCP របស់យើងនៅក្នុង Visual Studio Code [ទៅមេរៀន](04-vscode/README.md)

- **5 ម៉ាស៊ីនមេផ្ទុក stdio Transport** stdio transport គឺជាមាត្រដ្ឋានដែលបានផ្ដល់អនុសាសន៍សម្រាប់ការទំនាក់ទំនងរវាងម៉ាស៊ីនមេ MCP និងអតិថិជនក្នុងស្រុក ដែលផ្តល់ការទំនាក់ទំនងមានសុវត្ថិភាពមូលដ្ឋានពី subprocess និងមានការបំបែកដំណើរការ [ទៅមេរៀន](05-stdio-server/README.md)

- **6 ការចាក់បញ្ជូនទិន្នន័យ HTTP ជាមួយ MCP (Streamable HTTP)** រៀនអំពីការចាក់បញ្ជូនទិន្នន័យ HTTP សព្វថ្ងៃ (វិធីសាស្រ្តដែលបានផ្ដល់អនុសាសន៍សម្រាប់ម៉ាស៊ីនមេ MCP ចម្រុះតាមរយៈ [MCP Specification 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/basic/transports/#streamable-http)) ការជូនដំណឹងនៃការរីកចម្រើន និងរបៀបអនុវត្តម៉ាស៊ីនមេ និងអតិថិជន MCP ដែលអាចតម្រៀបបាន និងដំណើរការជាពេលវេលាពិតប្រាកដដោយប្រើ Streamable HTTP [ទៅមេរៀន](06-http-streaming/README.md)

- **7 ការប្រើប្រាស់ឧបករណ៍ AI Toolkit សម្រាប់ VSCode** ដើម្បីប្រើ និងសាកល្បងអតិថិជន និងម៉ាស៊ីនមេ MCP របស់អ្នក [ទៅមេរៀន](07-aitk/README.md)

- **8 ការ​សាកល្បង** នៅទីនេះ យើងនឹងផ្តោតជាពិសេសចំពោះរបៀបដែល​យើងអាចសាកល្បងម៉ាស៊ីនមេ និងអតិថិជនរបស់យើងដោយវិធីចម្រុះៗ [ទៅមេរៀន](08-testing/README.md)

- **9 ការដាក់កម្មវិធី** ជំពូកនេះ​នឹងមើលទិដ្ឋភាពចម្ងាយនៃការដាក់​កម្មវិធី MCP របស់អ្នក [ទៅមេរៀន](09-deployment/README.md)

- **10 ការប្រើប្រាស់ម៉ាស៊ីនមេកម្រិតខ្ពស់** ជំពូកនេះគ្របដណ្តប់ការប្រើប្រាស់ម៉ាស៊ីនមេកម្រិតខ្ពស់ [ទៅមេរៀន](./10-advanced/README.md)

- **11 សុវត្ថិភាព** ជំពូកនេះពិពណ៌នា របៀបបន្ថែមសុវត្ថិភាពមូលដ្ឋាន ពី Basic Auth ដល់ការប្រើ JWT និង RBAC។ អ្នកត្រូវបានលើកទឹកចិត្តឲ្យចាប់ផ្តើមពីទីនេះ ហើយបន្ទាប់មកមើលប្រធានបទកម្រិតខ្ពស់នៅជំពូក 5 និងអនុវត្តការកែលម្អសុវត្ថិភាពបន្ថែមតាមអនុសាសន៍នៅជំពូក 2 [ទៅមេរៀន](./11-simple-auth/README.md)

- **12 អ្នកផ្តល់សេវា MCP** កំណត់តម្លៃ និងប្រើអតិថិជនម៉ាស៊ីនមេ MCP ដែលពេញនិយម រួមមាន Claude Desktop, Cursor, Cline និង Windsurf។ រៀនអំពីប្រភេទនៃការដឹកជញ្ជូន និងការជួសជុលបញ្ហា [ទៅមេរៀន](./12-mcp-hosts/README.md)

- **13 MCP Inspector** ដំឡើង និងសាកល្បងម៉ាស៊ីនមេ MCP របស់អ្នកដោយប្រើឧបករណ៍ MCP Inspector។ រៀនជួសជុលឧបករណ៍ ស្រទាប់ផែនការនិងសារ​នៃប្រភពព័ត៌មាន [ទៅមេរៀន](./13-mcp-inspector/README.md)

- **14 ការប្រមូល​អត្ថបទ** បង្កើតម៉ាស៊ីនមេ MCP ដែលសហការជាមួយអតិថិជន MCP សម្រាប់ភារកិច្ចដែលទាក់ទង LLM [ទៅមេរៀន](./14-sampling/README.md)

- **15 កម្មវិធី MCP** បង្កើតម៉ាស៊ីនមេ MCP ដែលផ្តល់ចម្លើយជាមួយសេចក្តីណែនាំ UI ផងដែរ [ទៅមេរៀន](./15-mcp-apps/README.md)

Model Context Protocol (MCP) គឺជាពិធីសម្ព័ន្ធបើកដែលកំណត់វិធីសាស្រ្តស្តង់ដារដើម្បីឲ្យកម្មវិធីផ្តល់បរិបទទៅកាន់ LLMs។ គិតថា MCP គឺដូចជា ច្រក USB-C សម្រាប់កម្មវិធី AI - វាបង្កើតវិធីស្ដង់ដារ ដើម្បីភ្ជាប់ម៉ូដែល AI ទៅនឹងប្រភពទិន្នន័យ និងឧបករណ៍ផ្សេងៗ។

## គោលបំណងរៀន

បន្ទាប់ពីបញ្ចប់មេរៀននេះ អ្នកនឹងអាច៖

- ដំឡើងបរិយាកាសអភិវឌ្ឍសម្រាប់ MCP ក្នុង C#, Java, Python, TypeScript និង JavaScript
- សង់ និងដាក់បញ្ចូលម៉ាស៊ីនមេ MCP មូលដ្ឋានជាមួយមុខងារផ្ទាល់ខ្លួន (ធនធាន, បញ្ជូនប្រភេទ, និងឧបករណ៍)
- បង្កើតកម្មវិធីដើម្បីភ្ជាប់ទៅម៉ាស៊ីនមេ MCP
- សាកល្បង និងកែលម្អការអនុវត្ត MCP
- យល់ដឹងអំពីបញ្ហាផ្ទុកកំណត់ និងដំណោះស្រាយរបស់ពួកគេ
- ភ្ជាប់អនុវត្ត MCP របស់អ្នកទៅជាមួយសេវាកម្ម LLM ម៉ាស៊ីនមេពេញនិយម

## ការតំឡើងបរិយាកាស MCP របស់អ្នក

មុននឹងចាប់ផ្តើមធ្វើការជាមួយ MCP វា​សំខាន់ក្នុងការរៀបចំ​បរិយាកាសអភិវឌ្ឍ និងយល់ដឹងអំពីដំណើរការដំបូង។ ផ្នែកនេះនឹងណែនាំអ្នកឆ្ពោះទៅរកជំហានដំបូង ដើម្បីធ្វើឲ្យការចាប់ផ្តើម MCP របស់អ្នករលូន។

### តម្រូវការមុន

មុននឹងចូលរួមអភិវឌ្ឍ MCP សូមប្រាកដថាអ្នកមាន៖

- **បរិយាកាសអភិវឌ្ឍ** សម្រាប់ភាសាដែលអ្នកជ្រើស (C#, Java, Python, TypeScript ឬ JavaScript)
- **IDE/Editor**: Visual Studio, Visual Studio Code, IntelliJ, Eclipse, PyCharm ឬក៏កម្មវិធីកូដសម្រាប់កំណែទំនើបណាមួយ
- **អ្នកគ្រប់គ្រងវគ្គបម្រើ**: NuGet, Maven/Gradle, pip ឬ npm/yarn
- **កូនសោ API** សម្រាប់សេវាកម្ម AI អ្វីៗដែលអ្នកមានបំណងប្រើក្នុងកម្មវិធីម៉ាស៊ីនមេរបស់អ្នក

### SDKs ផ្លូវការច្រើនភាសា

នៅក្នុងជំពូកខាងមុខ អ្នកនឹងឃើញដំណោះស្រាយត្រូវបានកសាងដោយប្រើ Python, TypeScript, Java និង .NET។ នេះគឺជាព័ត៌មានអំពី SDKs ផ្លូវការដែលគាំទ្រជាផ្លូវការ។

MCP ផ្ដល់ SDK ផ្លូវការសម្រាប់ភាសាច្រើន ដូចដែលបានស្របទៅតាម [MCP Specification 2025-11-25](https://spec.modelcontextprotocol.io/specification/2025-11-25/):
- [SDK C#](https://github.com/modelcontextprotocol/csharp-sdk) - បានថែរក្សារជាមួយ Microsoft
- [SDK Java](https://github.com/modelcontextprotocol/java-sdk) - បានថែរក្សារជាមួយ Spring AI
- [SDK TypeScript](https://github.com/modelcontextprotocol/typescript-sdk) - កម្មវិធី TypeScript ផ្លូវការ
- [SDK Python](https://github.com/modelcontextprotocol/python-sdk) - កម្មវិធី Python ផ្លូវការ (FastMCP)
- [SDK Kotlin](https://github.com/modelcontextprotocol/kotlin-sdk) - កម្មវិធី Kotlin ផ្លូវការ
- [SDK Swift](https://github.com/modelcontextprotocol/swift-sdk) - បានថែរក្សារជាមួយ Loopwork AI
- [SDK Rust](https://github.com/modelcontextprotocol/rust-sdk) - កម្មវិធី Rust ផ្លូវការ
- [SDK Go](https://github.com/modelcontextprotocol/go-sdk) - កម្មវិធី Go ផ្លូវការ

## ចំណុចសំខាន់ៗ

- ការតំឡើងបរិយាកាសអភិវឌ្ឍ MCP គឺងាយស្រួលដោយមាន SDK ច្បាស់លាស់តាមភាសា
- ការសាងសង់ម៉ាស៊ីនមេ MCP រួមមានការបង្កើត និងចុះបញ្ជីឧបករណ៍ជាមួយសេកម៉ា​ច្បាស់លាស់
- អតិថិជន MCP ភ្ជាប់ទៅម៉ាស៊ីនមេ និងម៉ូដែល ដើម្បីប្រើប្រាស់សមត្ថភាពបន្ថែម
- ការសាកល្បង និងកែលម្អសំខាន់សម្រាប់សុវត្ថិភាព និងភាពជឿជាក់នៃអនុវត្ត MCP
