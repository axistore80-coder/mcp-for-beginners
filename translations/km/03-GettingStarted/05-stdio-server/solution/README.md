# សមាសភាគម៉ាស៊ីនមេ MCP stdio

> **⚠️ សំខាន់**: ដំណោះស្រាយទាំងនេះត្រូវបានអាប់ដេតដើម្បីប្រើ **ការដឹកជញ្ជូន stdio** ដូចដែលបានណែនាំដោយលក្ខខណ្ឌ MCP ថ្ងៃទី 2025-06-18។ ការដឹកជញ្ជូន SSE (Server-Sent Events) ដើមបានលុបចោល។

នេះជាដំណោះស្រាយពេញលេញសម្រាប់ការបង្កើតម៉ាស៊ីនមេ MCP ដោយប្រើការដឹកជញ្ជូន stdio ក្នុងបរិស្ថានរាល់មួយ៖

- [TypeScript](../../../../../03-GettingStarted/05-stdio-server/solution/typescript) - ការអនុវត្តម៉ាស៊ីនមេ stdio ពេញលេញ
- [Python](../../../../../03-GettingStarted/05-stdio-server/solution/python) - ម៉ាស៊ីនមេ stdio Python ជាមួយ asyncio
- [.NET](../../../../../03-GettingStarted/05-stdio-server/solution/dotnet) - ម៉ាស៊ីនមេ stdio .NET ជាមួយការច injections។

ដំណោះស្រាយនីមួយៗបង្ហាញពី៖
- ការកំណត់ដឹកជញ្ជូន stdio
- ការកំណត់ឧបករណ៍ម៉ាស៊ីនមេ
- ការព្យាបាលសារជា JSON-RPC ត្រឹមត្រូវ
- ការរួមបញ្ចូលជាមួយអតិថិជន MCP ដូចជា Claude

ដំណោះស្រាយទាំងអស់អនុវត្តតាមលក្ខខណ្ឌ MCP បច្ចុប្បន្ន ហើយប្រើការដឹកជញ្ជូន stdio ដែលបានណែនាំសម្រាប់ការសម្របសម្រួលដំណើរការ និងសុវត្ថិភាពឲ្យល្អបំផុត។
