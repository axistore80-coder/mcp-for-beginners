# MCP stdio Server - ដំណោះស្រាយ Python

> **⚠️ សំខាន់**: ដំណោះស្រាយនេះត្រូវបានអាប់ដេតទៅប្រើ **stdio transport** ដូចបានផ្ដល់អនុសាសន៍ដោយ MCP Specification 2025-06-18។ នេះបែបផែន SSE ដើមត្រូវបានលុបចោល។

## សង្ខេប

ដំណោះស្រាយ Python នេះបង្ហាញពីរបៀបបង្កើតម៉ាស៊ែរកម្ម MCP ដោយប្រើ stdio transport បច្ចុប្បន្ន។ stdio transport សាមញ្ញជាង មានសុវត្ថិភាពខ្ពស់ជាង ហើយផ្តល់ការសម្រួលល្អជាងវិធីសាស្ត្រ SSE ដែលត្រូវបានលុបចោល។

## តម្រូវការមុន

- Python 3.8 ឬចុងក្រោយជាង
- អ្នកត្រូវបានផ្ដល់អនុសាសន៍ឱ្យដំឡើង `uv` សម្រាប់គ្រប់គ្រងកញ្ចប់, មើល [សេចក្តីណែនាំ](https://docs.astral.sh/uv/#highlights)

## សេចក្តីណែនាំដំឡើង

### ជំហាន 1: បង្កើតបរិស្ថាន virtual

```bash
python -m venv venv
```

### ជំហាន 2: ចាប់ផ្តើមបរិស្ថាន virtual

**Windows:**
```bash
venv\Scripts\activate
```

**macOS/Linux:**
```bash
source venv/bin/activate
```

### ជំហាន 3: ដំឡើងអាស្រ័យភាព

```bash
pip install mcp
```

## ការរត់ម៉ាស៊័រ

ម៉ាស៊ើរបែប stdio រត់ខុសពីម៉ាស៊ែរ SSE ចាស់។ ជំនួសការចាប់ផ្តើមម៉ាស៊ែរ គឺប្រើការប្រាស្រ័យទាក់ទងតាមរយៈ stdin/stdout៖

```bash
python server.py
```

**សំខាន់**: ម៉ាស៊ែរ នឹងបង្ហាញថាអត់ឆ្លើយ - នេះគឺធម្មតា! វាកំពុងរងចាំសារប្រភេទ JSON-RPC មកពី stdin។

## ការប្រឡងម៉ាស៊ែរ

### វិធីទី 1: ប្រើ MCP Inspector (ផ្ដល់អនុសាសន៍)

```bash
npx @modelcontextprotocol/inspector python server.py
```

វានឹង៖
1. បញ្ចេញម៉ាស៊ែរ ជា subprocess របស់អ្នក
2. បើកចំណុចប្រទាក់បណ្ដាញសម្រាប់ការប្រឡង
3. អនុញ្ញាតឱ្យអ្នកប្រើឧបករណ៍ម៉ាស៊ែរ ទាំងអស់ យ៉ាងមានប្រសិទ្ធភាព

### វិធីទី 2: ការប្រឡង JSON-RPC ត្រង់

អ្នកអាចប្រឡងដោយផ្ញើសារប្រភេទ JSON-RPC ត្រង់បាន៖

1. ចាប់ផ្តើមម៉ាស៊ែរ៖ `python server.py`
2. ផ្ញើសារប្រភេទ JSON-RPC (ឧទាហរណ៍)៖

```json
{"jsonrpc": "2.0", "id": 1, "method": "tools/list"}
```

3. ម៉ាស៊ែរ នឹងឆ្លើយតបជាមួយឧបករណ៍ដែលមានខ្លួន

### ឧបករណ៍ដែលមាន

ម៉ាស៊ែរ ផ្តល់ឧបករណ៍ដូចខាងក្រោម៖

- **add(a, b)**: បូកលេខពីរចូលគ្នា
- **multiply(a, b)**:គុណលេខពីរចូលគ្នា  
- **get_greeting(name)**: បង្កើតសារស្វាគមន៍ផ្ទាល់ខ្លួន
- **get_server_info()**: ទទួលបានព័ត៌មានអំពីម៉ាស៊ែរ

### ការប្រឡងជាមួយ Claude Desktop

ដើម្បីប្រើម៉ាស៊ែរ​នេះជាមួយ Claude Desktop សូមបន្ថែមកំណត់រចនាសម្ព័ន្ធនេះទៅក្នុង `claude_desktop_config.json` របស់អ្នក៖

```json
{
  "mcpServers": {
    "example-stdio-server": {
      "command": "python",
      "args": ["path/to/server.py"]
    }
  }
}
```

## មុខងារផ្សេងគ្នា​ពី SSE

**stdio transport (បច្ចុប្បន្ន):**
- ✅ ការតំឡើងងាយស្រួល - មិនចាំបាច់មានម៉ាស៊ែរបណ្ដាញ
- ✅ មានសុវត្ថិភាពល្អជាង - គ្មានចំណុចចូល HTTP
- ✅ ប្រើការប្រាស្រ័យទាក់ទងដោយ subprocess
- ✅ JSON-RPC ជាលើ stdin/stdout
- ✅ មានសមត្ថភាពល្អជាង

**SSE transport (បានលុបចោល):**
- ❌ តម្រូវឱ្យមានម៉ាស៊ែរក្នុង HTTP
- ❌ តម្រូវឱ្យមាន framework បណ្ដាញ (Starlette/FastAPI)
- ❌ ការចែកចាយថ្នាក់ និងការគ្រប់គ្រងសម័យស្មុគស្មាញជាង
- ❌ មានការពិចារណាសុវត្ថិភាពបន្ថែម
- ❌ ពេលនេះត្រូវបានលុបចោលនៅ MCP 2025-06-18

## ក្បាស់នីតិវិធីដោះស្រាយបញ្ហា

- ប្រើ `stderr` សម្រាប់ការចុះបញ្ជី (មិនប្រើ `stdout` ទេ)
- ប្រឡងជាមួយ Inspector សម្រាប់ការដោះស្រាយបញ្ហាដោយមើល
- ធានាថាសារប្រភេទ JSON ទាំងអស់មាន "newline-delimited"
- ពិនិត្យឲ្យម៉ាស៊ែរ ចាប់ផ្តើមដោយគ្មានកំហុស

ដំណោះស្រាយនេះអនុវត្តតាមការបញ្ជាក់ MCP បច្ចុប្បន្ន និងបង្ហាញពីវិធីល្អបំផុតក្នុងការអនុវត្ត stdio transport។
