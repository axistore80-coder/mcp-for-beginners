# MCP stdio Server - ដំណោះស្រាយ TypeScript

> **⚠️ សំខាន់**: ដំណោះស្រាយនេះត្រូវបានធ្វើបច្ចុប្បន្នភាព ដើម្បីប្រើ **stdio transport** ដូចដែលបានណែនាំដោយ MCP Specification 2025-06-18។ ការបញ្ជូន SSE ដើមត្រូវបានលុបចោល។

## សង្ខេប

ដំណោះស្រាយ TypeScript នេះបង្ហាញពីវិធីសាស្រ្តក្នុងការបង្កើតម៉ាស៊ីនមេ MCP ដោយប្រើ stdio transport បច្ចុប្បន្ន។ stdio transport មានភាពសាមញ្ញជាង មុនសុវត្ថិភាព និងផ្តល់ប្រសិទ្ធភាពកាន់តែប្រសើរជាងវីធី SSE ដែលបានលុបចោល។

## ទាមទារ​មុន

- Node.js 18+ ឬបន្ទាប់
- npm ឬអ្នកគ្រប់គ្រងកញ្ចប់ yarn

## សេចក្ដីណែនាំដំឡើង

### ជំហានទី 1៖ ដំឡើងការពឹងផ្អែក

```bash
npm install
```

### ជំហានទី 2៖ សាងសង់គម្រោង

```bash
npm run build
```

## រត់ម៉ាស៊ីនមេ

ម៉ាស៊ីនមេ stdio រត់ខុសពីម៉ាស៊ីនមេ SSE ពីមុន។ ជំនួសការចាប់ផ្តើមម៉ាស៊ីនមេវេប វាទំនាក់ទំនងតាមរយៈ stdin/stdout៖

```bash
npm start
```

**សំខាន់**៖ ម៉ាស៊ីនមេនឹងមើលទៅដូចជាចន្ទុក - នេះគឺធម្មតា! វាកំពុងរង់ចាំសារ JSON-RPC មកពី stdin។

## សាកល្បងម៉ាស៊ីនមេ

### វិធីទី 1៖ ប្រើ MCP Inspector (ណែនាំ)

```bash
npm run inspector
```

នេះនឹង៖
1. បើកម៉ាស៊ីនមេរបស់អ្នកជាផ្នែករង
2. បើកផ្ទាំងបង្ហាញវេបសម្រាប់សាកល្បង
3. អនុញ្ញាតឲ្យអ្នកសាកល្បងឧបករណ៍ម៉ាស៊ីនមេទាំងអស់បានយ៉ាងអន្តរក्रिय

### វិធីទី 2៖ សាកល្បងផ្ទាល់ ដោយបញ្ចូលពាក្យបញ្ជា

អ្នកក៏អាចសាកល្បងដោយបើក Inspector ផ្ទាល់បានផងដែរ៖

```bash
npx @modelcontextprotocol/inspector node build/index.js
```

### ឧបករណ៍ដែលមាន

ម៉ាស៊ីនមេផ្តល់ឧបករណ៍ដូចខាងក្រោម៖

- **add(a, b)**: បូកលេខពីរជាមួយគ្នា
- **multiply(a, b)**: គុណលេខពីរជាមួយគ្នា  
- **get_greeting(name)**: បង្កើតសារស្វាគមន៍ផ្ទាល់ខ្លួន
- **get_server_info()**: ទទួលបានព័ត៌មានអំពីម៉ាស៊ីនមេ

### សាកល្បងជាមួយ Claude Desktop

ដើម្បីប្រើម៉ាស៊ីនមេនេះជាមួយ Claude Desktop សូមបន្ថែមការកំណត់នេះទៅក្នុង `claude_desktop_config.json` របស់អ្នក៖

```json
{
  "mcpServers": {
    "example-stdio-server": {
      "command": "node",
      "args": ["path/to/build/index.js"]
    }
  }
}
```

## ភាពខុសគ្នាសំខាន់ពី SSE

**stdio transport (បច្ចុប្បន្ន):**
- ✅ ការតំឡើងសាមញ្ញ - មិនចាំបាច់ម៉ាស៊ីនមេ HTTP
- ✅ សុវត្ថិភាពកាន់តែប្រសើរ - មិនមានចំណុចចូល HTTP ទេ
- ✅ ការទំនាក់ទំនងជាមួយផ្នែករង
- ✅ JSON-RPC លើ stdin/stdout
- ✅ ប្រសិទ្ធភាពល្អជាង

**SSE transport (បានលុបចោល):**
- ❌ តម្រូវការតំឡើងម៉ាស៊ីនមេ Express
- ❌ តម្រូវការផ្លូវឆ្ពោះ និងគ្រប់គ្រងសម័យសំខាន់
- ❌ មានការពឹងផ្អែកច្រើន (Express, ការប្រតិបត្តិ HTTP)
- ❌ ការពិចារណាសុវត្ថិភាពបន្ថែម
- ❌ ឥឡូវនេះបានលុបចោលនៅក្នុង MCP 2025-06-18

## គន្លឹះអភិវឌ្ឍន៍

- ប្រើ `console.error()` សម្រាប់កំណត់ហេតុ (កុំប្រើ `console.log()` ព្រោះវាសរសេរទៅ stdout)
- សាងសង់ជាមួយ `npm run build` មុនសាកល្បង
- សាកល្បងជាមួយ Inspector សម្រាប់ការបកស្រាយត្រួតពិនិត្យ
- ធ្វើការផ្ទៀងផ្ទាត់ឲ្យសារទាំងអស់ JSON មានទ្រង់ទ្រាយត្រឹមត្រូវ
- ម៉ាស៊ីនមេទំនងខ្លួនរៀបចំការបិទ gracefully នៅពេល SIGINT/SIGTERM

ដំណោះស្រាយនេះគោរពតាមបញ្ជាក់ MCP បច្ចុប្បន្ន ហើយបង្ហាញពីការអនុវត្តការដ៏ល្អបំផុតសម្រាប់ stdio transport ដោយប្រើ TypeScript។
