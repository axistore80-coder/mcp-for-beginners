## ការធ្វើតេស្ត និងកំណត់សុំពិធីយោបល់

មុននឹងចាប់ផ្ដើមធ្វើតេស្តម៉ាស៊ីនមេ MCP របស់អ្នក វាជាការសំខាន់ក្នុងការយល់ដឹងអំពីឧបករណ៍មាន ស្រាប់ និងអនុវត្តន៍ល្អបំផុតសម្រាប់កំណត់សុំពិធីយោបល់។ ការធ្វើតេស្តប្រសើរជាធម្មតានឹងធានាថា ម៉ាស៊ីនមេរបស់អ្នកមានប្រតិបត្តិការដូចដែលបានរំពឹងទុក ហើយជួយឲ្យអ្នកអាចស្វែងរក និងដោះស្រាយបញ្ហាទ迅រហ័ស។ ផ្នែកខាងក្រោមនេះបង្ហាញពីវិធីសាស្ត្រដែលបានផ្ដល់អនុសាសន៍សម្រាប់បញ្ជាក់មើលការអនុវត្ត MCP របស់អ្នក។

## សង្ខេប

មេរៀននេះគ្របដណ្ដប់ពីរបៀបជ្រើសរើសវិធីសាស្ត្រធ្វើតេស្តត្រឹមត្រូវ និងឧបករណ៍ធ្វើតេស្តមានប្រសិទ្ធភាពខ្ពស់បំផុត។

## វិជ្ជាជីវៈរៀនសូត្រ

នៅចុងបញ្ចប់មេរៀននេះ អ្នកនឹងអាច៖

- សរសេរពិពណ៌នាវិធីសាស្ត្រផ្សេងៗសម្រាប់ធ្វើតេស្ត។
- ប្រើប្រាស់ឧបករណ៍ផ្សេងៗដើម្បីធ្វើតេស្តកូដរបស់អ្នកបានប្រសិទ្ធភាព។

## ការធ្វើតេស្តម៉ាស៊ីនមេ MCP

MCP ផ្ដល់ឧបករណ៍ជួយអ្នកធ្វើតេស្ត និងកំណត់សុំពិធីយោបល់លើម៉ាស៊ីនមេរបស់អ្នក៖

- **MCP Inspector**៖ ជា​ឧបករណ៍រាយបញ្ជារចុចបញ្ជារដែលអាចដំណើរការបានទាំងនៅជារបៀប CLI និងជារបៀបមើលឃើញក្រាហ្វិក។
- **ការធ្វើតេស្តដោយដៃ**៖ អ្នកអាចប្រើឧបករណ៍ដូចជា curl ដើម្បីដំណើរការសំណើតាមបណ្តាញ តែឧបករណ៍ណាមួយដែលអាចដំណើរការពេល HTTPបានក៏អាចប្រើបានដែរ។
- **ការធ្វើតេស្តឯកតា**៖ អ្នកអាចប្រើ rámára framework ផ្សេងៗដែលអ្នកចូលចិត្ត ដើម្បីធ្វើតេស្តលើមុខងារពីម៉ាស៊ីនមេ និងម្ចាស់ភ្ញាក់ផ្អើល។

### ការប្រើប្រាស់ MCP Inspector

យើងបានពិពណ៌នាអំពីការប្រើប្រាស់ឧបករណ៍នេះនៅមេរៀនមុនៗ ប៉ុន្តែត្រឡប់មកពិភាក្សាបន្ថែមតិចនៅលើកម្រិត យេីងបានបង្កើតវាលើ Node.js ហើយអ្នកអាចប្រើវាបានដោយហៅកម្មវិធី `npx` ដែលនឹងទាញយក និងដំឡើងឧបករណ៍នេះផ្ទាល់ខ្លួនក្នុងរយៈពេលបណ្តោះអាសន្ន ហើយត្រូវសម្អាតខ្លួនវាដោយស្វ័យប្រវត្តិនៅពេលចប់ដំណើរការសំណើរបស់អ្នក។

[MCP Inspector](https://github.com/modelcontextprotocol/inspector) ជួយអ្នក៖

- **រកឃើញសមត្ថភាពម៉ាស៊ីនមេ**៖ អាចស្វែងរកធនធាន មានស្រាប់ ឧបករណ៍ និងសំណើ
- **ធ្វើតេស្តការប្រតិបត្តិឧបករណ៍**៖ សាកល្បងប៉ារ៉ាម៉ែត្រ និងមើលការឆ្លើយតបនៅពេលវេលាពិត
- **មើលព័ត៌មាន​ម៉ាស៊ីន​មេ**៖ ពិនិត្យព័ត៌មានម៉ាស៊ីនមេ ស្កីមា និងការកំណត់រចនាសម្ព័ន្ធ

ការប្រតិបត្តិឧបករណ៍ជារឿយៗមើលដូចជា៖

```bash
npx @modelcontextprotocol/inspector node build/index.js
```

បញ្ជាខាងលើចាប់ផ្តើម MCP និងច្រើនរូបភាពអន្តរជាតិវិជ្ជាជីវកម្ម ហើយដំណើរការលើចំណុចអ៊ីនធឺណិតក្នុងកម្មវិធីរុករករបស់អ្នក។ អ្នកអាចរំពឹងថានឹងឃើញផ្ទាំងគ្រប់គ្រងបង្ហាញម៉ាស៊ីនមេ MCP ដែលបានចុះបញ្ជី រួមមានឧបករណ៍ធ្វើតេស្តធនធាន និងសំណើ។ ចំណុចអ៊ីនធឺណិតនេះអនុញ្ញាតឱ្យអ្នកអាចធ្វើតេស្តឧបករណ៍ជាផ្ទាល់ កំណត់មើលព័ត៌មានម៉ាស៊ីនមេ និងមើលការឆ្លើយតបពេលវេលាពិត ដែលធ្វើឱ្យងាយស្រួលក្នុងការបញ្ជាក់ និងកំណត់សុំពិធីយោបល់ការអនុវត្ត MCP របស់អ្នក។

នេះគឺជា​រូបភាពដែលវាអាចធ្វើបាន៖ ![Inspector](../../../../translated_images/km/connect.141db0b2bd05f096.webp)

អ្នកក៏អាចដំណើរការឧបករណ៍នេះជារបៀប CLI ដែលអ្នកត្រូវបន្ថែមattrbute `--cli`។ នេះគឺជាឧទាហរណ៍នៃការប្រើឧបករណ៍ក្នុងរបៀប "CLI" ដែលបញ្ជីឧបករណ៍ទាំងអស់នៅលើម៉ាស៊ីនមេ៖

```sh
npx @modelcontextprotocol/inspector --cli node build/index.js --method tools/list
```

### ការធ្វើតេស្តដោយដៃ

ក្រៅពីការប្រើប្រាស់ឧបករណ៍ inspector ដើម្បីធ្វើតេស្តសមត្ថភាពម៉ាស៊ីនមេ វិធីសាស្ត្រមួយដូចគ្នាគឺដំណើរការអ្នកប្រើប្រាស់ដែលអាចប្រើ HTTP បាន ដូចជា curl។

ជាមួយ curl អ្នកអាចធ្វើតេស្តម៉ាស៊ីនមេ MCP តាមរយៈសំណើ HTTP ដោយផ្ទាល់៖

```bash
# ឧទាហរណ៍៖ ព័ត៌មានបន្ថែមម៉ាស៊ីនបម្រើ
curl http://localhost:3000/v1/metadata

# ឧទាហរណ៍៖ ដំណើរការឧបករណ៍មួយ
curl -X POST http://localhost:3000/v1/tools/execute \
  -H "Content-Type: application/json" \
  -d '{"name": "calculator", "parameters": {"expression": "2+2"}}'
```

ដូចដែលអ្នកឃើញពីការប្រើប្រាស់ curl ខាងលើ អ្នកប្រើសំណើ POST ដើម្បីហៅឧបករណ៍មួយដោយប្រើផayload ដែលរួមមានឈ្មោះឧបករណ៍ និងប៉ារ៉ាម៉ែត្រ។ ប្រើវិធីសាស្ត្រណាមួយដែលសមរម្យសម្រាប់អ្នក។ ឧបករណ៍ CLI ជាទូទៅមានល្បឿនលឿនក្នុងការប្រើប្រាស់ និងងាយស្រួលសរសេរកូដជាស្គ្រីប ដែលអាចមានប្រយោជន៍ក្នុងបរិបទ CI/CD។

### ការធ្វើតេស្តឯកតា

បង្កើតតេស្តឯកតាសម្រាប់ឧបករណ៍ និងធនធានរបស់អ្នក ដើម្បីធានាថាវាដំណើរការដូចគេរំពឹងទុក។ នេះគឺជាគំរូកូដធ្វើតេស្តមួយ។

```python
import pytest

from mcp.server.fastmcp import FastMCP
from mcp.shared.memory import (
    create_connected_server_and_client_session as create_session,
)

# សម្គាល់ម៉ូឌុលទាំងមូលសម្រាប់ការប្រឡង async
pytestmark = pytest.mark.anyio


async def test_list_tools_cursor_parameter():
    """Test that the cursor parameter is accepted for list_tools.

    Note: FastMCP doesn't currently implement pagination, so this test
    only verifies that the cursor parameter is accepted by the client.
    """

 server = FastMCP("test")

    # បង្កើតឧបករណ៍សម្រាប់ការប្រឡងពីរប្រភេទ
    @server.tool(name="test_tool_1")
    async def test_tool_1() -> str:
        """First test tool"""
        return "Result 1"

    @server.tool(name="test_tool_2")
    async def test_tool_2() -> str:
        """Second test tool"""
        return "Result 2"

    async with create_session(server._mcp_server) as client_session:
        # សម្លាញ់ដោយមិនប្រើប៉ារ៉ាម៉ែត្រការបង្ហាញ (មិនដាក់)
        result1 = await client_session.list_tools()
        assert len(result1.tools) == 2

        # សម្លាញ់ដោយប្រើ cursor=None
        result2 = await client_session.list_tools(cursor=None)
        assert len(result2.tools) == 2

        # សម្លាញ់ដោយប្រើ cursor ជាស្រមោលអក្សរ
        result3 = await client_session.list_tools(cursor="some_cursor_value")
        assert len(result3.tools) == 2

        # សម្លាញ់ដោយប្រើ cursor ជាខ្សែអក្សរទទេ
        result4 = await client_session.list_tools(cursor="")
        assert len(result4.tools) == 2
    
```

កូដខាងលើធ្វើការដូចខាងក្រោម៖

- ទទួលអនុសាសន៍ការប្រើប្រាស់ rámára pytest ដែលអនុញ្ញាតឱ្យអ្នកបង្កើតតេស្តជាអនុគមន៍ និងប្រើប្រយោគ assert។
- បង្កើតម៉ាស៊ីនមេ MCP មានឧបករណ៍ពីរផ្សេងគ្នា។
- ប្រើប្រយោគ `assert` ដើម្បីពិនិត្យថាលក្ខខណ្ឌណាមួយត្រូវបានបំពេញ។

សូមមើល [ឯកសារពេញលេញនៅទីនេះ](https://github.com/modelcontextprotocol/python-sdk/blob/main/tests/client/test_list_methods_cursor.py)

ដោយផ្អែកលើឯកសារខាងលើ អ្នកអាចធ្វើតេស្តម៉ាស៊ីនមេរបស់អ្នកเอง ដើម្បីធានាសមត្ថភាពត្រូវបានបង្កើតឡើងដូចដែលគ្រោងទុក។

SDK ធំៗទាំងអស់មានផ្នែកធ្វើតេស្តដូចគ្នា ដូច្នេះអ្នកអាចបត់បែនទៅតាម runtime ដែលអ្នកជ្រើសរើសបាន។
