# កំពុងដំណើរការឧទាហរណ៍នេះ

អ្នកត្រូវបានណែនាំឱ្យតំឡើង `uv` ប៉ុន្តែមិនមែនជាការជាសំខាន់ទេ សូមមើល [សេចក្ដីណែនាំ](https://docs.astral.sh/uv/#highlights)

## -1- តំឡើងការពឹងផ្អែក

```bash
npm install
```

## -3- ដំណើរការឧទាហរណ៍


```bash
npm run build
```

## -4- សាកល្បងឧទាហរណ៍

ដោយមានម៉ាស៊ីនបម្រើដំណើរការនៅក្នុងផ្ទាំងបញ្ជាកាន់តែមួយ បើកផ្ទាំងបញ្ជាថ្មីមួយទៀតហើយរត់ពាក្យបញ្ជាដូចខាងក្រោម៖

```bash
npm run inspector
```

នេះគួរតែចាប់ផ្តើមម៉ាស៊ីនបម្រើវែបមួយជាមួយចំណុចប្រទាក់មើលឃើញដែលអនុញ្ញាតឱ្យអ្នកសាកល្បងឧទាហរណ៍។

ពេលម៉ាស៊ីនបម្រើបានភ្ជាប់៖ 

- សាកល្បងបញ្ជីឧបករណ៍ និងដំណើរការ `add` ជាមួយប៉ារ៉ាម៉ែត្រ 2 និង 4 អ្នកគួរតែឃើញលទ្ធផលជាចំនួន 6។
- ទៅកាន់ធនធាន និងរចនារូបធនធាន ហើយហៅ "greeting" បញ្ចូលឈ្មោះ ហើយអ្នកគួរតែឃើញសារស្វាគមន៍ជាមួយឈ្មោះដែលអ្នកបានផ្តល់។

### សាកល្បងក្នុងរបៀប CLI

អ្នកត្រួតពិនិត្យដែលអ្នកបានដំណើរការពិតជាជាកម្មវិធី Node.js មួយ ហើយ `mcp dev` គឺជាគ្រឿងបណ្តាក់ទ្រទ្រង់មួយនៅជុំវិញវា។

អ្នកអាចបញ្ចេញវាโดยផ្ទាល់ក្នុងរបៀប CLI ដោយរត់ពាក្យបញ្ជាដូចខាងក្រោម៖

```bash
npx @modelcontextprotocol/inspector --cli node ./build/index.js --method tools/list
```

នេះនឹងបញ្ជីឧបករណ៍ទាំងអស់ដែលមាននៅលើម៉ាស៊ីនបម្រើ។ អ្នកគួរតែឃើញលទ្ធផលដូចខាងក្រោម៖

```text
{
  "tools": [
    {
      "name": "add",
      "description": "Add two numbers",
      "inputSchema": {
        "type": "object",
        "properties": {
          "a": {
            "title": "A",
            "type": "integer"
          },
          "b": {
            "title": "B",
            "type": "integer"
          }
        },
        "required": [
          "a",
          "b"
        ],
        "title": "addArguments"
      }
    }
  ]
}
```

ដើម្បីហៅឧបករណ៍ សូមវាយ ៖

```bash
nnpx @modelcontextprotocol/inspector --cli node ./build/index.js --method tools/call --tool-name add --tool-arg a=1 --tool-arg b=2
```

អ្នកគួរតែឃើញលទ្ធផលដូចខាងក្រោម៖

```text
{
  "content": [
    {
      "type": "text",
      "text": "3"
    }
  ],
  "isError": false
}
```

> [!TIP]
> ជាទូទៅ វានឹងរហ័សជាងក្នុងការចាប់ផ្តើមកម្មវិធី ispector ក្នុងរបៀប CLI ជាងក្នុងកម្មវិធីកម្មវិធីរុករក។
> អានបន្ថែមអំពី ispector [នៅទីនេះ](https://github.com/modelcontextprotocol/inspector)។
