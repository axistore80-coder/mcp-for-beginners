# ការប្រតិបត្តិគំរូនេះ

អ្នកត្រូវបានផ្តល់អនុសាសន៍ឲ្យដំឡើង `uv` ប៉ុន្តែមិនបាច់ទេ၊ សូមមើល [សេចក្ដីណែនាំ](https://docs.astral.sh/uv/#highlights)

## -0- បង្កើតបរិស្ថានវើឌ្វល

```bash
python -m venv venv
```

## -1- បើកបរិស្ថានវើឌ្វល

```bash
venv\Scripts\activate
```

## -2- ដំឡើងអាស្រ័យភាព

```bash
pip install "mcp[cli]"
```

## -3- ជំហានរត់គំរូ


```bash
mcp run server.py
```

## -4- សាកល្បងគំរូ

ជាមួយម៉ាស៊ីនបម្រើដែលកំពុងដំណើរការនៅក្នុងផ្ទាំងបញ្ជា​មួយ ចូរបើកផ្ទាំងបញ្ជាថ្មីមួយ ហើយចូររត់ពាក្យបញ្ជាខាងក្រោម៖

```bash
mcp dev server.py
```

នេះគួរតែចាប់ផ្តើមម៉ាស៊ីនបម្រើវេបមួយដែលមានចំណុចផ្ទាល់ខ្លួនមួយ អនុញ្ញាតិឲ្យអ្នកសាកល្បងគំរូ។

ពេលម៉ាស៊ីនបម្រើត្រូវបានភ្ជាប់៖

- សាកល្បងបញ្ជីឧបករណ៍និងរត់ `add` ជាមួយអាគូម្មង ២ និង ៤ អ្នកគួរតែឃើញលទ្ធផលជា ៦។

- ចូលទៅកាន់ធនធាន និងគំរូធនធាន ហើយហៅ get_greeting បញ្ចូលឈ្មោះមួយ ហើយអ្នកគួរតែឃើញសារស្វាគមន៍ជាមួយឈ្មោះដែលអ្នកបានផ្តល់។

### ការសាកល្បងក្នុងរបៀប CLI

កម្មវិធីអ្នកត្រួតពិនិត្យដែលអ្នកបានរត់ពិតប្រាកដជាកម្មវិធី Node.js មួយ ហើយ `mcp dev` គឺជាកញ្ចប់នៅជុំវិញវា។

អ្នកអាចចាប់ផ្តើមវាត្រូវនៅក្នុងរបៀប CLI ដោយរត់ពាក្យបញ្ជាខាងក្រោម៖

```bash
npx @modelcontextprotocol/inspector --cli mcp run server.py --method tools/list
```

នេះនឹងបង្ហាញបញ្ជីឧបករណ៍ទាំងអស់ដែលមាននៅក្នុងម៉ាស៊ីនបម្រើ។ អ្នកគួរតែឃើញលទ្ធផលដូចខាងក្រោម៖

```text
{
  "tools": [
    {
      "name": "add",
      "description": "Add two numbers",
      "inputSchema": {
        "type": "object",
        "properties": {
          "a": {
            "title": "A",
            "type": "integer"
          },
          "b": {
            "title": "B",
            "type": "integer"
          }
        },
        "required": [
          "a",
          "b"
        ],
        "title": "addArguments"
      }
    }
  ]
}
```

ដើម្បីហៅឧបករណ៍ មួយ សូមវាយពាក្យបញ្ជា៖

```bash
npx @modelcontextprotocol/inspector --cli mcp run server.py --method tools/call --tool-name add --tool-arg a=1 --tool-arg b=2
```

អ្នកគួរតែឃើញលទ្ធផលដូចខាងក្រោម៖

```text
{
  "content": [
    {
      "type": "text",
      "text": "3"
    }
  ],
  "isError": false
}
```

> [!TIP]
> សម្រាប់ការប្រើប្រាស់ អ្នកត្រួតពិនិត្យក្នុងរបៀប CLI ជាទូទៅ លឿនជាងក្នុងកម្មវិធីរុករក។
> អានបន្ថែមអំពីអ្នកត្រួតពិនិត្យ [នៅទីនេះ](https://github.com/modelcontextprotocol/inspector).
