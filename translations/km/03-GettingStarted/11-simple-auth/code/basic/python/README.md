# រត់សំណុំឧទាហរណ៍

សំណុំឧទាហរណ៍នេះចាប់ផ្តើមម៉ាស៊ីនមេ MCP ជាមួយមេឌីយូអវែរ​មួយដែលពិនិត្យមើលសម្រាប់ចំណងជើង Authorization ត្រូវតែត្រឹមត្រូវ។

## ដំឡើងផ្នែកបំពាក់ផ្នែកបន្ថែម

```bash
pip install "mcp[cli]" 
```

## ចាប់ផ្តើមម៉ាស៊ីនមេ

```bash
python server.py
```

ចាប់ផ្តើមកម្មវិធីអតិថិជននៅក្នុងផ្ទាំងបញ្ជា​មួយផ្សេងទៀត

```bash
python client.py
```

អ្នកគួរតែឃើញលទ្ធផលមានរូបមន្តដូចខាងក្រោម៖

```text
2025-09-30 13:25:54 - mcp_client - INFO - Tool result: meta=None content=[TextContent(type='text', text='{\n  "current_time": "2025-09-30T13:25:54.311900",\n  "timezone": "UTC",\n  "timestamp": 1759238754.3119,\n  "formatted": "2025-09-30 13:25:54"\n}', annotations=None, meta=None)] structuredContent={'current_time': '2025-09-30T13:25:54.311900', 'timezone': 'UTC', 'timestamp': 1759238754.3119, 'formatted': '2025-09-30 13:25:54'} isError=False
```

នេះមានន័យថា ឯកសារទិន្នន័យដែលបានបញ្ជូនត្រូវបានអនុញ្ញាត។

សាកល្បងប្ដូរឯកសារទិន្នន័យនៅក្នុង `client.py` ទៅជា "secret-token2" បន្ទាប់មកអ្នកគួរតែឃើញអត្ថបទនេះជាផ្នែកនៃការឆ្លើយតប៖

```text
2025-09-30 13:27:44 - httpx - INFO - HTTP Request: POST http://localhost:8000/mcp "HTTP/1.1 403 Forbidden"
```

នេះមានន័យថា អ្នកបានផ្ទៀងផ្ទាត់ (អ្នកមានឯកសារទិន្នន័យ), ប៉ុន្តែវាមិនត្រឹមត្រូវ។
