# AGENTS.md

## ការពិពណ៌នាពីគម្រោង

**MCP សម្រាប់អ្នកចាប់ផ្តើម** គឺជាបរិយាកាសការសិក្សាតាមប្រភពបើកសម្រាប់ការសិក្សាពី Model Context Protocol (MCP) - ស៊ុមសំណុំច្បាប់ស្តង់ដាសម្រាប់អន្តរកម្មរវាងម៉ូឌែល AI និងកម្មវិធីអតិថិជន។ កំណត់ទីតាំងនេះផ្តល់នូវសំណុំព័ត៌មានសិក្សាលំម្ដាប់ពេញលេញជាមួយកូដតំណាងអនុវត្តដោយដៃ តាមភាសាបញ្ចូលកម្មវិធីជាច្រើន។

### បច្ចេកវិទ្យាចម្បង

- **ភាសាកម្មវិធី**: C#, Java, JavaScript, TypeScript, Python, Rust
- **ស៊ុមសំណុំ & SDKs**: 
  - MCP SDK (`@modelcontextprotocol/sdk`)
  - Spring Boot (Java)
  - FastMCP (Python)
  - LangChain4j (Java)
- **មូលដ្ឋានទិន្នន័យ**: PostgreSQL ជាមួយការបន្ថែម pgvector
- **វេទិកាពពក**: Azure (Container Apps, OpenAI, Content Safety, Application Insights)
- **ឧបករណ៍កសាង**: npm, Maven, pip, Cargo
- **ឯកសារ**: Markdown ជាមួយការបកប្រែភាសាច្រើនដោយស្វ័យប្រវត្តិ (48+ ភាសា)

### វីធានស្ថាបត្យកម្ម

- **១១ ម៉ូឌុលស្នូល (០០-១១)**: ផ្លូវការសិក្សាទាំងឡាយពីមូលដ្ឋានទៅកាន់ប្រធានបទជំរុញខ្ពស់
- **មន្ទីរពិសោធន៍បច្ចេកទេស**: លំហាត់អនុវត្តជាក់ស្តែងជាមួយកូដដំណោះស្រាយពេញលេញក្នុងភាសាច្រើន
- **គម្រោងម៉ូឌែល**: កម្មវិធីម៉ាស៊ីនបម្រើ MCP និងអតិថិជនអនុវត្តការ
- **ប្រព័ន្ធបកប្រែ**: ជួរប្រតិបត្តិការ GitHub Actions ដោយស្វ័យប្រវត្តិសម្រាប់គាំទ្រភាសាច្រើន
- **ទ្រព្យសម្បត្តិរូបភាព**: ទីតាំងផ្ទុករូបភាពកណ្តាលជាមួយកំណែបកប្រែ

## ពាក្យបញ្ជាបង្កើត

នេះគឺជាកំណត់ទីតាំងផ្តោតលើឯកសារ។ ការតំឡើងភាគច្រើនកើតឡើងនៅក្នុងគម្រោងទំនាក់ទំនង និងមន្ទីរសាកល្បងនីមួយៗ។

### ការតំឡើង Repository

```bash
# ចម្លងឃ្លាំងរបស់បណ្ណាសារ
git clone https://github.com/microsoft/mcp-for-beginners.git
cd mcp-for-beginners
```

### ការងារជាមួយគម្រោងទំនាក់ទំនង

គម្រោងទំនាក់ទំនងស្ថិតនៅ:
- `03-GettingStarted/samples/` - ឧទាហរណ៍ជាភាសាពិសេស
- `03-GettingStarted/01-first-server/solution/` - អនុវត្តកម្មវិធីម៉ាស៊ីនបម្រើដំបូង
- `03-GettingStarted/02-client/solution/` - អនុវត្តកម្មវិធីអតិថិជន
- `11-MCPServerHandsOnLabs/` - មន្ទីរពិសោធន៍បញ្ចូលមូលដ្ឋានទិន្នន័យពេញលេញ

គម្រោងទំនាក់ទំនងនីមួយៗមានសេចក្តីណែនាំតំឡើងផ្ទាល់ខ្លួន៖

#### គម្រោង TypeScript/JavaScript
```bash
cd <project-directory>
npm install
npm start
```

#### គម្រោង Python
```bash
cd <project-directory>
pip install -r requirements.txt
# ឬ
pip install -e .
python main.py
```

#### គម្រោង Java
```bash
cd <project-directory>
mvn clean install
mvn spring-boot:run
```

## ដំណើរការអភិវឌ្ឍន៍

### រចនាសម្ព័ន្ធឯកសារ

- **ម៉ូឌុល ០០-១១**: មាតិកាបណ្តុះបណ្តាលស្នូលតាមលំដាប់
- **translations/**: កំណែជាភាសានិមួយៗ (ផលិតដោយស្វ័យប្រវត្តិ មិនត្រូវកែបន្ថែមដោយដៃ)
- **translated_images/**: កំណែរូបភាពបែបបង្ហាញក្នុងភាសា (ផលិតដោយស្វ័យប្រវត្តិ)
- **images/**: រូបភាព និងគំនូរ​ដើម

### ការផ្លាស់ប្ដូរឯកសារ

1. កែសម្រួលឯកសារ markdown ជាភាសាអង់គ្លេសប៉ុណ្ណោះនៅក្នុងថតម៉ូឌុលរឺកន្លែងដើម (០០-១១)
2. បន្ទាន់សម្រួលរូបភាពនៅក្នុងថត `images/` ប្រសិនបើចាំបាច់
3. ប្រតិបត្តិការ GitHub Action co-op-translator នឹងបង្កើតការបកប្រែដោយស្វ័យប្រវត្តិ
4. ការបកប្រែត្រូវបានធ្វើឡើងម្តងទៀតនៅពេលផុសទៅសាខា main

### ការងារជាមួយការបកប្រែ

- **ការបកប្រែដោយស្វ័យប្រវត្តិ**: ប្រតិបត្តិការ GitHub Actions ដំណើរការការបកប្រែទាំងអស់
- **មិនត្រូវកែប្រែដោយដៃ** ឯកសារនៅក្នុងថត `translations/`
- ព័ត៌មានមេតា ពាក់ព័ន្ធនឹងការបកប្រែបានដាក់នៅក្នុងឯកសារបកប្រែទាំងអស់
- ភាសាគាំទ្រ: ៤៨+ ភាសា រួមមាន អារ៉ាប់, ចិន, បារាំង, អាល្លឺម៉ង់, ហ៊ីនឌូ, ជប៉ុន, ខូរ៉េ, ព័រទugues, រុស្ស៊ី, អេស្ប៉ាញ, និងភាសាផ្សេងទៀតជាច្រើន

## សេចក្តីណែនាំសម្រាប់ការធ្វើតេស្ត

### ការធ្វើតេស្តឯកសារ

ដោយសារតែនេះគឺជាកំណត់ទីតាំងផ្តោតលើឯកសារ សេចក្តីបញ្ជានេះផ្តោតលើ៖

1. **ផ្ទៀងផ្ទាត់តំណភ្ជាប់**: ប្រាកដថាតំណភ្ជាប់ទាំងអស់ខាងក្នុងដំណើរការល្អ
```bash
# ពិនិត្យមើលតំណ Markdown ខូច
find . -name "*.md" -type f | xargs grep -n "\[.*\](../../.*)"
```

2. **ផ្ទៀងផ្ទាត់គំរូកូដ**: សាកល្បងថាគំរូកូដអាចបញ្ចូល និងអនុវត្តបាន
```bash
# ផ្លាស់ទីទៅតំបន់​គំរូជាក់លាក់ ហើយបើកចាប់ផ្តើមការធ្វើតេស្តរបស់វា
cd 03-GettingStarted/samples/typescript
npm install && npm test
```

3. **ពិនិត្យរចនាសម្ព័ន្ធ Markdown**: ពិនិត្យភាពផ្ទៀងផ្ទាត់រចនាសម្ព័ន្ធ
```bash
# ប្រើ markdownlint ប្រសិនបើจำเป็น
npx markdownlint-cli2 "**/*.md" "#node_modules"
```

## សៀវភៅមាគ៌ាសម្រាប់រចនាប័ទ្មកូដ

### ស្ទីលឯកសារ

- ប្រើភាសាមួយច្បាស់លាស់ រួមទាំងសម្រាប់អ្នកចាប់ផ្តើម
- រួមបញ្ចូលឧទាហរណ៍កូដជាភាសាតិចៗនៅពេលមាន
- ធ្វើតាមគោលការណ៍ Markdown ល្អបំផុត៖
  - ប្រើកម្មវិធីស្លាកសំពាធ ATX (`#` សញ្ញា)
  - ប្រើបណ្ដុំកូដជាលក្ខណៈមួកសម្រាប់ភាសាកំណត់
  - រួមបញ្ចូលអត្ថបទ alt ដាក់ពណ៌នាសម្រាប់រូបភាព
  - រក្សាការពង្រឹងប្រវែងបន្ទាត់ឲ្យសមរម្យ (គ្មានកំណត់តឹង តែហត្ថការល្អ)

### ស្ទីលគំរូកូដ

- ប្រើសញ្ញាបញ្ជាក់លេខបញ្ចូលប្រភេទពេលគួរ
- រួមបញ្ចូល docstrings សម្រាប់មុខងារ និងថ្នាក់
- ប្រើមុខងារ Python សម័យទំនើប (3.8+)

#### Java
- តាមប្រព័ន្ធ Spring Boot
- ប្រើមុខងារ Java 21
- តាមរចនាសម្ព័ន្ធគម្រោង Maven មុនតម្លើង
- រួមបញ្ចូលមតិយោបល់ Javadoc

2. **បង្កើតសាខា**: ប្រើឈ្មោះសាខាដែលពិពណ៌នាថ្មីច្បាស់ (ឧ. `fix/typo-module-3`, `add/python-example`)
3. **ធ្វើការផ្លាស់ប្ដូរ**: កែសម្រួលឯកសារ markdown ជាភាសាអង់គ្លេសប៉ុណ្ណោះ (មិនបញ្ចូលកំណែបកប្រែ)
4. **សាកល្បងនៅក្នុងកុំព្យូទ័រផ្ទាល់**: ពិនិត្យមើលថា markdown បង្ហាញបានត្រឹមត្រូវ
5. **ដាក់ស្នើ PR**: ប្រើចំណងជើង និងសេចក្តីពិពណ៌នាច្បាស់លាស់
6. **CLA**: ចុះហត្ថលេខារួចថាអ្នកយល់ព្រមនឹង Microsoft Contributor License Agreement ពេលមានស្នើសុំ

### ទម្រង់ចំណងជើង PR

ប្រើចំណងជើងច្បាស់លាស់និងពិពណ៌នា៖
- `[Module XX] ការពិពណ៌នាខ្លី` សម្រាប់ការផ្លាស់ប្ដូរម៉ូឌុលជាក់លាក់
- `[Samples] ការពិពណ៌នា` សម្រាប់ការផ្លាស់ប្ដូរកូដគំរូ
- `[Docs] ការពិពណ៌នា` សម្រាប់កំណែលេខឯកសារទូទៅ

5. **១១**: មន្ទីរពិសោធន៍បញ្ចូលមូលដ្ឋានទិន្នន័យពេញលេញ (១៣ មន្ទីរសម្រាប់រៀបរាប់លំដាប់)

### បញ្ហាក្នុងការដោះស្រាយជាញឹកញាប់

**Q: PR របស់ខ្ញុំបរាជ័យក្នុងការត្រួតពិនិត្យការបកប្រែ**
A: ទំនាក់ទំនងថាអ្នកបានកែសម្រួលឯកសារ markdown ជាភាសាអង់គ្លេសប៉ុណ្ណោះនៅក្នុងថតម៉ូឌុលរឺកន្លែងដើម មិនមែនកំណែបកប្រែទេ។

**Q: ថាតើខ្ញុំដាក់ភាសាថ្មីមួយដូចម្តេច?**
A: ការគាំទ្រភាសាត្រូវបានគ្រប់គ្រងតាមប្រតិបត្តិការ co-op-translator។ បើកបញ្ហា issue ដើម្បីពិភាក្សាអំពីការបន្ថែមភាសាថ្មី។

**Q: គំរូកូដមិនដំណើរការ**
A: ប្រាកដថាអ្នកបានអនុវត្តតាមសេចក្តីណែនាំក្នុង README ពីគំរូជាក់លាក់។ ពិនិត្យថាមានកំណែគ្រឿងចក្រ និងសំណុំឡាយប៉ាដែលត្រូវបានដំឡើង។

**Q: រូបភាពមិនបង្ហាញ**
A: ពិនិត្យវិធីបញ្ជូនរូបភាពជ.relative និងប្រើស្លាកស្លុតបើកមុខ។ រូបភាពគួរត្រូវរក្សាទុកក្នុងថត `images/` ឬ `translated_images/` សម្រាប់កំណែបកប្រែ។
