# ការអនុវត្តជាក់ស្តែង

[![វិធីសាស្ត្រសាងសង់, សាកល្បង និងដាក់ពង្រីកកម្មវិធី MCP ជាមួយឧបករណ៍ និងដំណើរការពិតប្រាកដ](../../../translated_images/km/05.64bea204e25ca891.webp)](https://youtu.be/vCN9-mKBDfQ)

_(ចុចរូបភាពខាងលើដើម្បីមើលវីដេអូមេរៀននេះ)_

ការអនុវត្តជាក់ស្តែងគឺជាកន្លែងដែលអំណាចនៃ Model Context Protocol (MCP) ក្លាយជា​អារម្មណ៍បាន។ នៅពេលយើងយល់ពីទ្រឹស្តី និងសំណង់ខាងក្រោយ MCP គឺមានសារៈសំខាន់ ការ​ផ្តល់តម្លៃពិតប្រាកដចេញមកពេលអ្នកអនុវត្តគំនិតទាំងនេះដើម្បីសាងសង់ សាកល្បង និងដាក់ពង្រីកដំណោះស្រាយដែលដោះស្រាយបញ្ហានៅក្នុងពិភពជាក់ស្តែង។ សៀវភៅនេះជួយស្ពានចន្លោះចំណេះដឹងគំនិត និងការអភិវឌ្ឍដោយដៃ ស្វែងរកដំណើរការនាំអោយទទួលបានកម្មវិធីជាច្រើនជាប់ MCP។

មិនថាអ្នកកំពុងអភិវឌ្ឍជំនួយឆ្លាតវៃ, បញ្ចូល AI ទៅក្នុងដំណើរការអាជីវកម្ម, ឬបង្កើតឧបករណ៍ផ្ទាល់ខ្លួនសម្រាប់ដំណើរការទិន្នន័យ MCP ផ្តល់ចំណុចចាប់ផ្តើមបត់បែនបាន។ រចនាប័ទ្មមិនភ្ជាប់ភាសា និង SDKs ផ្លូវការរបស់វាដើម្បីភាសាកម្មវិធីពេញនិយមធ្វើឱ្យវាសមរម្យសម្រាប់អ្នកអភិវឌ្ឍជាច្រើនជួរដែលខុសគ្នា។ តាមរយៈការប្រើប្រាស់ SDKs ទាំងនេះ អ្នកអាចបង្កើតគំរូប្រើប្រាស់បានយ៉ាងរហ័ស តវ៉ាជាថ្មី និងអាចឡើងវាស្ថិតក្នុងវេទិកានានា និងបរិស្ថានផ្សេងៗ។

នៅក្នុងផ្នែកខាងក្រោម អ្នកនឹងឃើញឧទាហរណ៍ជាក់ស្តែង កូដគំរូ និងយុទ្ធសាស្ត្រដាក់ពង្រីកដែលបង្ហាញពីវិធីអនុវត្ត MCP ក្នុង C#, Java ជាមួយ Spring, TypeScript, JavaScript និង Python។ អ្នកនឹងបានរៀនពីវិធីសាកល្បង និងដោះស្រាយបញ្ហាសំរាប់ម៉ាស៊ីនមេ MCP របស់អ្នក គ្រប់គ្រង API និងដាក់ពង្រីកដំណោះស្រាយទៅកាន់មេឃ្លាំងដោយប្រើ Azure ទាំងនេះគឺធនធានហត្ថអនុវត្តដែលរចនាឡើងសម្រាប់ល្បឿនរៀន និងជួយអ្នកបង្កើតកម្មវិធី MCP មាំមួនដែលរួចរាល់សម្រាប់ផលិតកម្ម។

## សង្ខេប

មេរៀននេះផ្តោតលើផ្នែកអនុវត្តជាក់ស្តែងនៃ MCP ក្នុងភាសាកម្មវិធីច្រើនមុខ។ យើងនឹងស្វែងយល់ពីការប្រើប្រាស់ MCP SDK ក្នុង C#, Java ជាមួយ Spring, TypeScript, JavaScript និង Python ដើម្បីបង្កើតកម្មវិធីមាំមួន, សាកល្បង និងដោះស្រាយម៉ាស៊ីនមេ MCP, និងបង្កើតធនធាន, ពន្លឿន, និងឧបករណ៍ដែលប្រើប្រាស់ឡើងវិញ។

## គោលដៅសិក្សា

នៅចុងបញ្ចប់មេរៀននេះ អ្នកនឹងអាច៖

- អនុវត្តដំណោះស្រាយ MCP ដោយប្រើ SDKs ផ្លូវការក្នុងភាសាកម្មវិធីផ្សេងៗ
- សាកល្បង និងដោះស្រាយបញ្ហាម៉ាស៊ីនមេ MCP ដោយវិញវិល
- បង្កើត និងប្រើលក្ខណៈម៉ាស៊ីនមេ (ធនធាន, ពន្លឿន និងឧបករណ៍)
- សារសំខាន់លំហាត់ធ្វើការ MCP សម្រាប់បន្ទុកស្មុគស្មាញ
- អុីប៉ុងទំព័រអនុវត្ត MCP សម្រាប់កំណត់តម្លៃ និងភាពជឿនលឿន

## លក្ខណៈម៉ាស៊ីនមេសំខាន់ៗ

ម៉ាស៊ីនមេ MCP អាចអនុវត្តលក្ខណៈណាមួយក្នុងចំណោមដូចខាងក្រោម៖

### ធនធាន

ធនធានផ្តល់បរិបទ និងទិន្នន័យសម្រាប់អ្នកប្រើឬម៉ាស៊ីន AI ប្រើ៖

- ឃ្លាំងឯកសារ
- មូលដ្ឋានចំណេះដឹង
- ការប្រមូលទិន្នន័យដែលមានរចនាសម្ព័ន្ធ
- ប្រព័ន្ធរៀបចំឯកសារ

### ពន្លឿន

ពន្លឿនគឺជាសារ និងដំណើរការបង្កើតទម្រង់សម្រាប់អ្នកប្រើ៖

- គំរូសន្ទនាជាមុន
- លំនាំធ្វើអន្តរកម្មដែលបានណែនាំ
- រចនាសម្ព័ន្ធសន្ទនាពិសេស

### ឧបករណ៍

ឧបករណ៍គឺជាអនុគមន៍សម្រាប់ម៉ាស៊ីន AI អនុវត្ត៖

- ឧបករណ៍ដំណើរការទិន្នន័យ
- បញ្ចូល API ខាងក្រៅ
- សមត្ថភាពគណនា
- 功能ស្វែងរក

## គំរូអនុវត្តៈ C# Implementation

ឃ្លាំង SDK ផ្លូវការដែលមានឈ្មោះ C# មានគំរូអនុវត្តថ្មីៗជាច្រើនបង្ហាញពីផ្នែកជាចម្បងនៃ MCP៖

- **មុខងារ MCP Client ដែលចាស់តែមាន**៖ ឧទាហរណ៍សាមញ្ញបង្ហាញពីរបៀបបង្កើត MCP Client និងហៅឧបករណ៍
- **ម៉ាស៊ីនមេ MCP សាមញ្ញ**៖ អនុវត្តម៉ាស៊ីនមេអប្បបរមាជាមួយការចុះបញ្ជីឧបករណ៍មូលដ្ឋាន
- **ម៉ាស៊ីនមេ MCP ជំនាន់ខ្ពស់**៖ ម៉ាស៊ីនមេគ្រប់គ្រងឧបករណ៍ អត្តសញ្ញាណ និងការដោះស្រាយបញ្ហាលម្អិត
- **ការចូលរួម ASP.NET**៖ ឧទាហរណ៍បង្ហាញពីការលាតត្រដាងជាមួយ ASP.NET Core
- **លំនាំអនុវត្តឧបករណ៍**៖ លំនាំផ្សេងៗសម្រាប់អនុវត្តឧបករណ៍ជាមួយកម្រិតស្មុគស្មាញខុសគ្នា

SDK C# MCP កំពុងនៅក្នុងដំណាក់កាលជាមុនហើយ API អាចផ្លាស់ប្តូរ។ យើងនឹងបន្តបច្ចុប្បន្នភាពប្លុកនេះនៅពេល SDK វាស់វែងឡើង។

## គំរូអនុវត្តៈ Java ជាមួយ Spring Implementation

SDK Java ជាមួយ Spring ផ្តល់ជម្រើសអនុវត្ត MCP មានមុខងារថ្នាក់អាជីវកម្មដ៏រឹងមាំ។

### លក្ខណៈសំខាន់ៗ

- បញ្ចូល Spring Framework
- ភាពសុវត្ថិភាពប្រភេទត្រឹមត្រូវ
- ការគាំទ្រកម្មវិធី Reactive
- ការដោះស្រាយកំហុសគ្រប់គ្រងបានពេញលេញ

សម្រាប់គំរូអនុវត្ត Java ជាមួយ Spring សូមមើល [គំរូ Java ជាមួយ Spring](samples/java/containerapp/README.md) នៅក្នុងថតគំរូ។

## គំរូអនុវត្តៈ JavaScript Implementation

### លក្ខណៈសំខាន់ៗ

## ដាក់ពង្រីកម៉ាស៊ីនមេ MCP ពីចម្ងាយទៅ Azure

មើលទៅថាតើយើងអាចដាក់ពង្រីកគំរូដែលបានរៀបរាប់មុននេះទេ៖

1. ក្លូនឃ្លាំង

    ```bash
    git clone https://github.com/Azure-Samples/remote-mcp-apim-functions-python.git
    cd remote-mcp-apim-functions-python
    ```

1. ចុះបញ្ជីអ្នកផ្គត់ផ្គង់ធនធាន `Microsoft.App`។

   - ប្រសិនបើអ្នកប្រើ Azure CLI សូមរត់ `az provider register --namespace Microsoft.App --wait`។
   - ប្រសិនបើអ្នកប្រើ Azure PowerShell សូមរត់ `Register-AzResourceProvider -ProviderNamespace Microsoft.App`។ បន្ទាប់មករត់ `(Get-AzResourceProvider -ProviderNamespace Microsoft.App).RegistrationState` បន្ទាប់ពីពេលខ្លះដើម្បីពិនិត្យមើលថាការចុះបញ្ជីបានបញ្ចប់រួចចុះឬនៅ។

1. រត់បញ្ជា [azd](https://aka.ms/azd) នេះដើម្បីផ្គត់ផ្គង់សេវាកម្ម api management, function app (ជាមួយកូដ) និងធនធាន Azure ផ្សេងទៀតដែលត្រូវការ​ទាំងអស់

    ```shell
    azd up
    ```

    បញ្ជាការនេះគួរត្រូវដាក់ពង្រីកធនធានទាំងអស់នៅលើ Azure

### សាកល្បងម៉ាស៊ីនមេរបស់អ្នកជាមួយ MCP Inspector

1. នៅក្នុង **បង្អួច terminal ថ្មីមួយ** តំឡើង និងរត់ MCP Inspector

    ```shell
    npx @modelcontextprotocol/inspector
    ```

    អ្នកគួរតែឃើញផ្ទាំងប្រតិបត្តិភាពដូចជា៖

    ![ភ្ជាប់ទៅ Node inspector](../../../translated_images/km/connect.141db0b2bd05f096.webp)

1. ចុច CTRL ដើម្បីផ្ទុកកម្មវិធីជើងខែល MCP Inspector ពី URL ដែលកម្មវិធីបង្ហាញ (ឧ. [http://127.0.0.1:6274/#resources](http://127.0.0.1:6274/#resources))
1. កំណត់ប្រភេទនាំដឹកទៅ `SSE`
1. កំណត់ URL ទៅកាន់ចំណុចចេញ API Management SSE ដែលកំពុងដំណើរការរបស់អ្នកបង្ហាញបន្ទាប់ពី `azd up` ហើយ **ភ្ជាប់**៖

    ```shell
    https://<apim-servicename-from-azd-output>.azure-api.net/mcp/sse
    ```

1. **បញ្ជីឧបករណ៍**។ ចុចលើឧបករណ៍មួយ ហើយ **រត់ឧបករណ៍**។  

ប្រសិនបើជំហានទាំងអស់ដំណើរការត្រឹមត្រូវ អ្នកគួរតែភ្ជាប់ជាមួយម៉ាស៊ីនមេ MCP បានហើយ អ្នកបានអាចហៅឧបករណ៍មួយបាន។

## ម៉ាស៊ីនមេ MCP សម្រាប់ Azure

[Remote-mcp-functions](https://github.com/Azure-Samples/remote-mcp-functions-dotnet): បន្ទុកឃ្លាំងទាំងនេះជាគំរូចាប់ផ្តើមសម្រាប់សាងសង់និងដាក់ពង្រីកម៉ាស៊ីនមេ MCP ចម្ងាយ (Model Context Protocol) ដោយប្រើ Azure Functions ជាមួយ Python, C# .NET ឬ Node/TypeScript។

គំរូផ្តល់ដំណោះស្រាយពេញលេញដែលអនុញ្ញាតឱ្យអ្នកអភិវឌ្ឍក្រុមហ៊ុន៖

- សាងសង់ និងរត់ក្នុងកន្លែង៖ អភិវឌ្ឍ និងសាកល្បងម៉ាស៊ីនមេ MCP នៅលើម៉ាស៊ីនមូលដ្ឋាន
- ដាក់ពង្រីកទៅ Azure៖ រហ័សរហាត់ដាក់ពង្រីកទៅមេឃ្លាំងដោយបញ្ជា azd up ងាយស្រួល
- ភ្ជាប់ពីអតិថិជន៖ ភ្ជាប់ទៅម៉ាស៊ីនមេ MCP ពីអតិថិជនផ្សេងៗ រួមមាន VS Code នៅរបៀប Copilot agent និងឧបករណ៍ MCP Inspector

- អភិវឌ្ឍមូលដ្ឋាន៖ គាំទ្រសាកល្បង និងអភិវឌ្ឍមូលដ្ឋានពេញលេញ
