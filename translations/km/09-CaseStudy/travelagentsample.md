# ករណីសិក្សា៖ អ្នកចរណ៍ AI របស់ Azure – ការអនុវត្តវិចារណ.reference

## ទិដ្ឋភាពទូទៅ

[Azure AI Travel Agents](https://github.com/Azure-Samples/azure-ai-travel-agents) គឺជាដំណោះស្រាយយោងពេញលេញដែលបានបង្កើតដោយ Microsoft ដែលបង្ហាញពីរបៀបសាងសង់កម្មវិធីរៀបចំដំណើរកម្សាន្តដែលមានភ្នាក់ងារច្រើនដោយប្រើបច្ចេកវិទ្យា AI ដោយប្រើ Model Context Protocol (MCP), Azure OpenAI, និង Azure AI Search។ គម្រោងនេះបង្ហាញពីអនុវត្តបែបបទល្អបំផុតសម្រាប់សម្របសម្រួលភ្នាក់ងារបច្ចេកវិទ្យា AI ច្រើន ស្វែងយល់ទិន្នន័យសហគ្រាស និងផ្តល់ជាកម្មវិធីមានសុវត្ថិភាព និងបន្ថែមបានសម្រាប់ស្ថានភាពពិត។

## លក្ខណៈសម្បត្តិសំខាន់ៗ
- **ការសម្របសម្រួលភ្នាក់ងារច្រើន៖** ប្រើ MCP ដើម្បីអភិវឌ្ឍក្រុមភ្នាក់ងារពិសេស (ដូចជា អ្នកចរណ៍ព័ត៌មានហោះហើរ, សណ្ឋាគារ និងផែនដំណើរ) ដែលសហការគ្នា ដើម្បីបំពេញភារកិច្ចរៀបចំនីតិវិធីដំណើរកម្សាន្តស្មុគស្មាញ។
- **ការតភ្ជាប់ទិន្នន័យសហគ្រាស៖** តភ្ជាប់ទៅ Azure AI Search និងប្រភពទិន្នន័យសហគ្រាសផ្សេងទៀត ដើម្បីផ្តល់ព័ត៌មានទាន់សម័យ និងពាក់ព័ន្ធសម្រាប់ការផ្ដល់អនុសាសន៍ដំណើរកម្សាន្ត។
- **ស្ថាបត្យកម្មសុវត្ថិភាព និងអាចពង្រីក៖** ប្រើសេវាកម្ម Azure សម្រាប់ការផ្ទៀងផ្ទាត់អត្តសញ្ញាណ ការអនុញ្ញាត និងការដាក់ប្រាក់ប្រើដែលអាចពង្រីកបាន ដំណើរការតាមអនុគ្រោះប្រទេសសុវត្ថិភាព។
- **ឧបករណ៍បន្ថែម៖** អនុវត្តឧបករណ៍ MCP ដែលអាចប្រើឡើងវិញ និងគំរូសំណុំបែបបទចម្លើយ ដើម្បីអាចបញ្ជាក់ទៅមុខក្រោយនៅក្នុងដែនឬតម្រូវការអាជីវកម្មថ្មីៗ។
- **បទពិសោធន៍អ្នកប្រើប្រាស់៖** ផ្តល់ផ្ទាំងនិយាយជាមួយអ្នកប្រើប្រាស់ដើម្បីធ្វើប្រតិកម្មជាមួយភ្នាក់ងារដំណើរកម្សាន្ត ដោយថាមពលពី Azure OpenAI និង MCP។

## ស្ថាបត្យកម្ម
![Architecture](https://raw.githubusercontent.com/Azure-Samples/azure-ai-travel-agents/main/docs/ai-travel-agents-architecture-diagram.png)

### សេចក្ដីអធិប្បាយតារាងស្ថាបត្យកម្ម

ដំណោះស្រាយ Azure AI Travel Agents ត្រូវបានបង្កើតឡើងសម្រាប់ភាពលាយឡំ ភាពអាចពង្រីកបាន និងការតភ្ជាប់សុវត្ថិភាពរវាងភ្នាក់ងារបច្ចេកវិទ្យា AI ច្រើន និងប្រភពទិន្នន័យសហគ្រាស។ សមាសធាតុសំខាន់ៗ និងដំណើរការទិន្នន័យគឺដូចខាងក្រោម៖

- **ផ្ទាំងអ្នកប្រើប្រាស់៖** អ្នកប្រើប្រាស់ធ្វើប្រតិកម្មជាមួយប្រព័ន្ធតាម UI ពិភាក្សា (ដូចជា កញ្ចក់វេបសាយ ឬ bot Teams) ដែលផ្ញើសំនួរអ្នកប្រើ និងទទួលអនុសាសន៍ដំណើរកម្សាន្ត។
- **ម៉ាស៊ីនបម្រើ MCP៖** ដំណើរការជាអ្នកសម្របសម្រួលកណ្តាល ទទួលព័ត៍មានពីអ្នកប្រើគ្រប់គ្រងបរិបទ និងសម្របសម្រួលសកម្មភាពរបស់ភ្នាក់ងារពិសេស (ដូចជា FlightAgent, HotelAgent, ItineraryAgent) តាមរយៈ Model Context Protocol។
- **ភ្នាក់ងារ AI៖** ភ្នាក់ងារ​នីមួយៗ ប្រកួតប្រជែងផ្នែកជាក់លាក់ (ជាឧទាហរណ៍ យន្តហោះ សណ្ឋាគារ ផែនដំណើរ) ហើយត្រូវបានអនុវត្តជាឧបករណ៍ MCP។ ភ្នាក់ងារប្រើគំរូសំណុំបែបបទនិងตรรก៍ដើម្បីដំណើរការសំណើ និងបង្កើតចម្លើយ។
- **សេវាកម្ម Azure OpenAI:** ផ្តល់ការយល់ដឹងភាសាធម្មជាតិកម្រិតខ្ពស់ និងការបង្កើត ភាសា អនុញ្ញាតឲ្យភ្នាក់ងារបកស្រាយចេតនាអ្នកប្រើ និងបង្កើតចម្លើយសន្ទនា។
- **Azure AI Search និងទិន្នន័យសហគ្រាស៖** ភ្នាក់ងារសួរអំពី Azure AI Search និងប្រភពទិន្នន័យសហគ្រាសផ្សេងទៀត ដើម្បីយកព័ត៌មានទាន់សម័យអំពីជ Fahrzeuge, សណ្ឋាគារ និងជម្រើសដំណើរកម្សាន្ត។
- **ការផ្ទៀងផ្ទាត់ និងសុវត្ថិភាព៖** ផ្នែកនេះរួមបញ្ចូលជាមួយ Microsoft Entra ID សម្រាប់ការផ្ទៀងផ្ទាត់សុវត្ថិភាព និងអនុវត្តការត្រួតពិនិត្យចូលប្រើតិចតួចទៅធនធានគ្រប់អង្គភាព។
- **ការដាក់នៅលើ Container Apps របស់ Azure៖** គំរូនេះត្រូវបានរចនាឡើងសម្រាប់ដាក់ក្នុង Azure Container Apps ដើម្បីធានាថាអាចពង្រីក និងបរិហារណ្ធការ។

ស្ថាបត្យកម្មនេះអនុញ្ញាតឲ្យមានការសម្របសម្រួលយ៉ាងរហ័សរវាងភ្នាក់ងារ AI ច្រើន ការតភ្ជាប់សុវត្ថិភាពទៅទិន្នន័យសហគ្រាស និងផ្នែកប្រព័ន្ធដែលអាចបន្ថែមបានសម្រាប់សំណុំកម្មវិធី AI ផ្នែកជាក់លាក់។

## ការពិពណ៌នាជំហាន​ដោយជំហាននៃតារាងស្ថាបត្យកម្ម
សូមនឹកឃើញការរៀបចំដំណើរកំសាន្តធំមួយ ហើយមានក្រុមជំនាញជួយគ្រប់ផ្នែក។ ប្រព័ន្ធ Azure AI Travel Agents ធ្វើការដូចគ្នានេះ ដោយប្រើផ្នែកផ្សេងៗ (ដូចជា សមាជិកក្រុម) ដែលម្នាក់ៗមានភារកិច្ចពិសេស។ នេះជារបៀបដែលវាភ្ជាប់គ្នា៖

### ផ្ទាំងអ្នកប្រើប្រាស់ (UI):
គិតថាវាជាគ្រែមុខរបស់អ្នកចរណ៍ដំណើរកម្សាន្តរបស់អ្នក។ វាជាកន្លែងដែលអ្នក (អ្នកប្រើប្រាស់) សួរបញ្ហា ឬធ្វើសំណើ ដូចជា “សូមរកបើក្រឿងហោះហើរទៅទីក្រុងប៉ារីស។” វាអាចជាប្រអប់សន្ទនា​នៅលើ​គេហទំព័រ ឬកម្មវិធីផ្ញើសារ។

### ម៉ាស៊ីនបម្រើ MCP (អ្នកត្រួតពិនិត្យ):
ម៉ាស៊ីនបម្រើ MCP គឺដូចជា អ្នកគ្រប់គ្រងដែលស្តាប់សំណើររបស់អ្នកនៅធ្នេញមុខ និងសម្រេចថាអ្នកជំនាញណាគួរទទួលបន្ទុកគ្រប់ផ្នែក។ វាត្រួតពិនិត្យការសន្ទនារបស់អ្នក និងធានាថាភារកិច្ចទាំងអស់ដំណើរការល្អ។

### ភ្នាក់ងារ AI (ជំនួយការជំនាញ):
ភ្នាក់ងារនីមួយៗជាជំនាញក្នុងផ្នែកជាក់លាក់ – ម្នាក់ស្គាល់អំពីហោះហើរ, ម្នាក់ទៀតអំពីសណ្ឋាគារ និងម្នាក់ទៀតអំពីការរៀបចំផែនដំណើរ។ នៅពេលអ្នកស្នើដំណើរ MCP Server យកសំណើរបស់អ្នកទៅភ្នាក់ងារដែលសមស្រប។ ភ្នាក់ងារទាំងនេះប្រើចំណេះដឹង និងឧបករណ៍របស់ពួកគេ ដើម្បីរកជម្រើសល្អបំផុតសម្រាប់អ្នក។

### សេវាកម្ម Azure OpenAI (ជំនាញភាសា):
នេះដូចជាមានជំនាញភាសាដែលយល់អ្វីដែលអ្នកស្នើ ដោយមិនគិតពីរបៀបអ្នកសរសេរ។ វាជួយឲ្យភ្នាក់ងារយល់សំណើររបស់អ្នក និងឆ្លើយតបជាភាសាសន្ទនា។

### Azure AI Search និងទិន្នន័យសហគ្រាស (បណ្ណាល័យព័ត៍មាន):
សូមនឹកឃើញបណ្ណាល័យធំបំផុត ដែលមានព័ត៌មានដំណើរកំសាន្តទាន់សម័យទាំងអស់ – តារាងហោះហើរ, សណ្ឋាគារស្ថិតនៅ និងជម្រើសដទៃទៀត។ ភ្នាក់ងារស្វែងរកបណ្ណាល័យនេះដើម្បីយកចម្លើយត្រឹមត្រូវបំផុតសម្រាប់អ្នក។

### ការផ្ទៀងផ្ទាត់ និងសុវត្ថិភាព (អ្នកចាំច្រកសុវត្ថិភាព):
ដូចជា អ្នកចាំច្រក សម្រាប់ពិនិត្យមើលអ្នកណាអាចចូលតំបន់ជាក់លាក់បានក៏ប៉ុន្តែនេះធានាថា មានតែអ្នកដែលមានសិទ្ធិបើកចូល និងភ្នាក់ងារតំរូវការសម្របសម្រួលត្រឹមត្រូវ ដើម្បីរក្សាអោយទិន្នន័យរបស់អ្នកមានសុវត្ថិភាព និងឯកជន។

### ការដាក់នៅលើ Azure Container Apps (អគារខ្លួន):
ជំនួយការនិងឧបករណ៍ទាំងនេះរួមគ្នាក្នុងអគារដែលមានសុវត្ថិភាព អាចពង្រីកបាន (ពពក)។ នេះមានន័យថាប្រព័ន្ធអាចគ្រប់គ្រងអ្នកប្រើជាច្រើនពេលតែមួយ និងតែងតែមានស្រាប់នៅពេលអ្នកត្រូវការវា។

## វារួមគ្នាដូចម្តេច៖

អ្នកចាប់ផ្តើមដោយសួរបញ្ហានៅកន្លែងទទួល (UI)។
អ្នកគ្រប់គ្រង (MCP Server) សម្រេចថាអ្នកជំនាញណាគួរជួយអ្នក។
អ្នកជំនាញប្រើអ្នកបកប្រែភាសា (OpenAI) ដើម្បីយល់សំណើររបស់អ្នក និងបណ្ណាល័យ (AI Search) ដើម្បីរកចម្លើយល្អបំផុត។
អ្នកចាំច្រកសុវត្ថិភាព (Authentication) ត្រួតពិនិត្យទាំងអស់ឲ្យមានសុវត្ថិភាព។
ទាំងនេះកើតឡើងនៅក្នុងអគារដែលអាចពង្រីកបាន និងទំនុកចិត្តបាន (Azure Container Apps) ដូច្នេះបទពិសោធន៍របស់អ្នកល្អ និងមានសុវត្ថិភាព។
ក្រុមការងារនេះអនុញ្ញាតឲ្យប្រព័ន្ធជួយអ្នករៀបចំដំណើរបានយ៉ាងរហ័ស និងសុវត្ថិភាព ដូចជាក្រុមអ្នកចរណ៍ដំណើរអាជ្ញាសិទ្ធិដែលធ្វើការជាមួយគ្នា​នៅក្នុងការិយាល័យទំនើប។

## ការអនុវត្តបច្ចេកទេស
- **ម៉ាស៊ីនបម្រើ MCP៖** ផ្ដល់តុល្យភាពទ្រឹស្តី សម្របសម្រួលឧបករណ៍ភ្នាក់ងារ និងគ្រប់គ្រងបរិបទសម្រាប់ស៊េរីការងាររៀបចំដំណើរកម្សាន្តច្រើនជំហាន។
- **ភ្នាក់ងារ៖** ភ្នាក់ងារនីមួយៗ (ដូចជា FlightAgent, HotelAgent) ត្រូវបានអនុវត្តជាឧបករណ៍ MCP មានគំរូសំណុំបែបបទ និងตรรก្យារបស់ខ្លួន។
- **ការតភ្ជាប់ Azure៖** ប្រើ Azure OpenAI សម្រាប់ការយល់ដឹងភាសាធម្មជាតិ និង Azure AI Search សម្រាប់ស្វែងរកទិន្នន័យ។
- **សុវត្ថិភាព៖** រួមបញ្ចូល Microsoft Entra ID សម្រាប់ការផ្ទៀងផ្ទាត់ និងអនុវត្តសិទ្ធិចូលប្រើតិចតួចទៅធនធានគ្រប់អង្គភាព។
- **ការដាក់ពាក្យ៖** គាំទ្រការដាក់នៅលើ Azure Container Apps សម្រាប់ភាពអាចពង្រីក និងប្រសិទ្ធភាពប្រតិបត្តិការ។

## លទ្ធផល និងផលប៉ះពាល់
- បង្ហាញពីរបៀបប្រើ MCP ដើម្បីបង្កើតការសម្របសម្រួលភ្នាក់ងារ AI ច្រើនក្នុងស្ថានភាពផលិតកម្មពិតប្រាកដ។
- ជំរុញការអភិវឌ្ឍដំណោះស្រាយ ដោយផ្ដល់គំរូដែលអាចប្រើឡើងវិញសម្រាប់ការសម្របសម្រួលភ្នាក់ងារ ការតភ្ជាប់ទិន្នន័យ និងការដាក់ពាក្យមានសុវត្ថិភាព។
- ធ្វើជាគំរូសម្រាប់ការសាងសង់កម្មវិធី AI មានឈ្មោះច្បាស់សម្រាប់ដែនជាក់លាក់ ដោយប្រើ MCP និងសេវាកម្ម Azure។
