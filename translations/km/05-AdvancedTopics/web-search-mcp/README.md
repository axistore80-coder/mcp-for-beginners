# ថ្នាក់សិក្សា៖ ការបង្កើតម៉ាស៊ីនមេ Web Search MCP

ជំពូកនេះបង្ហាញពីរបៀបបង្កើតភ្នាក់ងារតុល្យភាពAIពិតប្រាកដដែលថែមចូលជាមួយ API ខាងក្រៅ ដោះស្រាយប្រភេទទិន្នន័យចម្រុះ គ្រប់គ្រងកំហុស និងរៀបចំឧបករណ៍ច្រើនៗទាំងអស់ក្នុងទ្រង់ទ្រាយស្រេចរៀបចំសម្រាប់ផលិតកម្ម។ អ្នកនឹងឃើញ៖

- **ការបន្តផ្សំជាមួយ API ខាងក្រៅដែលទាមទារការផ្ទៀងផ្ទាត់**
- **ការដោះស្រាយប្រភេទទិន្នន័យផ្សេងៗពីចំណុចចូលជាច្រើន**
- **យុទ្ធសាស្ត្រការដោះស្រាយកំហុសយ៉ាងរឹងមាំ និងកំណត់ហេតុ**
- **ការរៀបចំឧបករណ៍ច្រើនៗនៅក្នុងម៉ាស៊ីនមេតែមួយ**

នៅចុងនេះ អ្នកនឹងមានបទពិសោធន៍អនុវត្តន៍ជាមួយលំនាំ និងម៉ូដែលល្អបំផុតដែលមានសារៈសំខាន់សម្រាប់កម្មវិធី AI និង LLM ជាន់ខ្ពស់។

## ការណែនាំ

ក្នុងថ្នាក់សិក្សានេះ អ្នកនឹងរៀនពីរបៀបបង្កើតម៉ាស៊ីនមេ MCP និងអតិថិជនដែលពង្រីកសមត្ថភាព LLM ជាមួយទិន្នន័យវែបប្រកាន់ពេលវេលាពិតដោយប្រើ SerpAPI។ នេះជាជំនាញសំខាន់សម្រាប់អភិវឌ្ឍភ្នាក់ងារតុល្យភាព AI ដែលអាចចូលដំណើរការទិន្នន័យថ្មីពីវែប។

## គោលបំណងរៀន

នៅចុងថ្នាក់សិក្សានេះ អ្នកនឹងអាច៖

- បន្តផ្សំ API ខាងក្រៅ (ដូចជា SerpAPI) អោយមានសុវត្ថិភាពទៅក្នុងម៉ាស៊ីនមេ MCP
- អនុវត្តឧបករណ៍ច្រើនសម្រាប់ស្វែងរកវែប, ព័ត៌មាន, ផលិតផល, និង សំណួរ-ចម្លើយ
- បំបែក និងបំពាក់ទិន្នន័យរចនាសម្ព័ន្ធសម្រាប់ការប្រើប្រាស់ LLM
- ដោះស្រាយកំហុស និងគ្រប់គ្រងកំណួតអត្រា API ដោយមានប្រសិទ្ធភាព
- បង្កើត និងសាកល្បងអតិថិជន MCP ដែលស្វ័យប្រវត្តិ និងអន្តរកម្ម

## ម៉ាស៊ីនមេ Web Search MCP

ផ្នែកនេះណែនាំអំពីស្ថាបត្យកម្ម និងលក្ខណៈពិសេសនៃម៉ាស៊ីនមេ Web Search MCP។ អ្នកនឹងឃើញពីរបៀបដែល FastMCP និង SerpAPI ត្រូវបានប្រើរួមគ្នាដើម្បីពង្រីកសមត្ថភាព LLM ជាមួយទិន្នន័យវែបពេលវេលាពិត។

### ទិដ្ឋភាពទូទៅ

ការអនុវត្ដនេះមានឧបករណ៍បួនដែលបង្ហាញពីសមត្ថភាព MCP ក្នុងការដោះស្រាយភារកិច្ច(api)គ្រប់គ្រងដោយ API ខាងក្រៅដោយមានសុវត្ថិភាព និងប្រសិទ្ធភាព៖

- **general_search**: សម្រាប់កំណត់លទ្ធផលគ្រប់ទំហំនៃវែប
- **news_search**: សម្រាប់សារព័ត៌មានកាលពីថ្មីៗ
- **product_search**: សម្រាប់ទិន្នន័យអេឡិចត្រូនិចពាណិជ្ជកម្ម
- **qna**: សម្រាប់សំណួរ និងចម្លើយរង

### លក្ខណៈពិសេស
- **ឧទាហរណ៍កូដ**: រួមបញ្ចូលប្លុកកូដជាភាសារផ្សេងៗសម្រាប់ Python (ហើយងាយស្រួលពង្រីកទៅភាសាផ្សេងទៀត) ដោយប្រើគន្លងកូដសម្រាប់ភាពច្បាស់លាស់

### Python

```python
# ការប្រើប្រាស់ឧទាហរណ៍នៃឧបករណ៍ស្វែងរកទូទៅ
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

async def run_search():
    server_params = StdioServerParameters(
        command="python",
        args=["server.py"],
    )
    async with stdio_client(server_params) as (reader, writer):
        async with ClientSession(reader, writer) as session:
            await session.initialize()
            result = await session.call_tool("general_search", arguments={"query": "open source LLMs"})
            print(result)
```

---

មុនដំណើរការអតិថិជន វាជួយយល់ថាអ្វីដែលម៉ាស៊ីនមេកំពុងធ្វើ។ ឯកសារ [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py) អនុវត្តម៉ាស៊ីនមេ MCP ដែលបង្ហាញឧបករណ៍សម្រាប់វែប ព័ត៌មាន ផលិតផល និងសំណួរ-ចម្លើយដោយបន្តផ្សំជាមួយ SerpAPI។ វាដោះស្រាយសំណើចូល គ្រប់គ្រងការហៅ API បំបែកចម្លើយ និងបញ្ចូនលទ្ធផលរចនាសម្ព័ន្ធទៅអតិថិជន។

អ្នកអាចពិនិត្យការអនុវត្តពេញលេញនៅក្នុង [`server.py`](../../../../05-AdvancedTopics/web-search-mcp/server.py)។

នេះជាឧទាហរណ៍ខ្លឹមសារពីរបៀបដែលម៉ាស៊ីនមេកំណត់ និងចុះឈ្មោះឧបករណ៍មួយ៖

### ម៉ាស៊ីនមេ Python

```python
# server.py (ដំណាក់កាល)
from mcp.server import MCPServer, Tool

async def general_search(query: str):
    # ...ការអនុវត្ត...

server = MCPServer()
server.add_tool(Tool("general_search", general_search))

if __name__ == "__main__":
    server.run()
```

---

- **ការបន្តផ្សំ API ខាងក្រៅ**: បង្ហាញការគ្រប់គ្រងក្បួន API និងសំណើខាងក្រៅដោយសុវត្ថិភាព
- **ការបំបែកទិន្នន័យរចនាសម្ព័ន្ធ**: បង្ហាញរបៀបបម្លែងចម្លើយ API ទៅទ្រង់ទ្រាយឆមាសម្រាប់ LLM
- **ការដោះស្រាយកំហុស**: ការដោះស្រាយកំហុសយ៉ាងរឹងមាំជាមួយកំណត់ហេតុសាកសម
- **អតិថិជនអន្តរកម្ម**: រួមបញ្ចូលការសាកល្បងស្វ័យប្រវត្តិ និងរបៀបអន្តរកម្មសម្រាប់ការសាកល្បង
- **ការគ្រប់គ្រងបរិបទ**: ប្រើប្រាស់ MCP Context សម្រាប់កំណត់ហេតុ និងតាមដានសំណើ

### របៀបអតិថិជន

- **របៀបធម្មតា**៖ ដំណើរការតេស្តស្វ័យ ប្រើឧបករណ៍ទាំងអស់ និងបញ្ជាក់ចម្លើយ។ វាមានប្រយោជន៍សម្រាប់ពិនិត្យនៅចុះមួយដល់ថាថ្នាក់មេ និងឧបករណ៍ដំណើរការត្រឹមត្រូវ។
- **របៀបអន្តរកម្ម**៖ ចាប់ផ្តើមចំណុចផ្ទាំងប្រើប្រាស់ដែលអាចជ្រើសរើសឧបករណ៍ដោយដៃ បញ្ចូលសំណួរផ្ទាល់ខ្លួន និងមើលលទ្ធផលក្នុងពេលវេលាពិត។ វាសមសម្រាប់ការស្វែងយល់សមត្ថភាពម៉ាស៊ីនមេនិងសាកល្បងនូវទិន្នន័យផ្សេងៗ។

- **ការរៀបចំឧបករណ៍ច្រើន**៖ នេះមានន័យថារត់ឧបករណ៍ជាច្រើនផ្សេងគ្នា (ដូចជាស្វែងរកវែប ព័ត៌មាន ផលិតផល និងសំណួរ-ចម្លើយ) នៅក្នុងម៉ាស៊ីនមេ MCP តែមួយ។ វាអនុញ្ញាតឲ្យម៉ាស៊ីនមេរបស់អ្នកដោះស្រាយភារកិច្ចជាច្រើន មិនត្រឹមតែ១។
- **ការគ្រប់គ្រងកំណួតអត្រា API**៖ API ផ្សេងៗ (ដូចជា SerpAPI) មួយច្រើនបានកំណត់ចំនួនសំណើដែលអ្នកអាចធ្វើក្នុងពេលកំណត់។ កូដល្អពិនិត្យកំណត់ចំនួននេះ ព្រមទាំងដោះស្រាយវាយ៉ាងសមរម្យ ដើម្បីឲ្យកម្មវិធីអ្នកមិនបាក់ប្រសិនបើយកំណត់ត្រូវបានឈប់។
- **ការបំបែកទិន្នន័យរចនាសម្ព័ន្ធ**៖ ចម្លើយ API ជាញឹកញាប់មានរចនាសម្ព័ន្ធស្មុគស្មាញ។ គំនិតនេះគឺបម្លែងចម្លើយក្នងទ្រង់ទ្រាយស្អាត ងាយស្រួលប្រើសម្រាប់ LLM ឬកម្មវិធីផ្សេងៗ។
- **ការស្តារកំហុស**៖ ពេលខ្លះមានអ្វីមួយបំផ្លាញ - ប្រហែលបណ្តាញខ្សែប្រហោង ឬ API មិនចេញចម្លើយដែលអ្នករំពឹងទុក។ ការស្តារកំហុសមានន័យថាកូដអ្នកអាចដោះស្រាយបញ្ហាទាំងនេះ ហើយនៅតែផ្តល់ព័ត៌មានមានប្រយោជន៍ជំនួសការបាក់។
- **ការត្រួតពិនិត្យប៉ារ៉ាម៉ែត្រ**៖ នេះពាក់ព័ន្ធនឹងការត្រួតពិនិត្យថាការបញ្ចូលទៅឧបករណ៍របស់អ្នកត្រឹមត្រូវ និងសុវត្ថិភាព។ វារួមបញ្ចូលការកំណត់តម្លៃលំនាំ និងធានាប្រភេទ ថ្លៃខ្ពស់ ក្នុងការជួយទប់ស្កាត់កំហុស និងភាពច្របូកច្របល់។

ផ្នែកនេះនឹងជួយអ្នកវាយតម្លៃ និងដោះស្រាយបញ្ហារឿយៗដែលអ្នកប្រទះឃើញខណៈពេលធ្វើការជាមួយម៉ាស៊ីនមេ Web Search MCP។ ប្រសិនបើអ្នកជួបកំហុស ឬអវត្តមានបង្កើតឲ្យឈប់លើប្រតិបត្តិការជាមួយម៉ាស៊ីនមេ Web Search MCP ផ្នែកជួសជុលនេះផ្តល់ដំណោះស្រាយសម្រាប់បញ្ហាធម្មតា។ សូមពិនិត្យមើលគន្លឹះទាំងនេះមុនពេលស្វែងរកជំនួយបន្ថែម - ពួកវាច្រើននៅតែជួយដោះស្រាយបញ្ហារបស់អ្នកយ៉ាងរហ័ស។

## ជួសជុលបញ្ហា

នៅពេលធ្វើការជាមួយម៉ាស៊ីនមេ Web Search MCP អ្នកប្រហែលជាប្រឈមមុខនឹងបញ្ហា—នេះគឺធម្មតានៅពេលអភិវឌ្ឍជាមួយ API ខាងក្រៅ និងឧបករណ៍ថ្មី។ ផ្នែកនេះផ្តល់ដំណោះស្រាយជាក់ស្តែងសម្រាប់បញ្ហាធម្មតាមាញ់ ដើម្បីឲ្យអ្នកអាចត្រឡប់មកលើផ្លូវបានយ៉ាងឆាប់រហ័ស។ ប្រសិនបើអ្នកជួបកំហុស សូមចាប់ផ្តើមនៅទីនេះ៖ គន្លឹះខាងក្រោមនិយាយពីបញ្ហាដែលអ្នកប្រើប្រាស់ភាគច្រើនប្រឈមមុខ និងច្រើនសម្រួលដោះស្រាយបញ្ហារបស់អ្នកដោយគ្មានជំនួយបន្ថែម។

### បញ្ហាទូទៅ

ខាងក្រោមជាបញ្ហាទូទៅមួយចំនួនដែលអ្នកប្រើប្រាស់ជួបប្រទៈជាមួយពណ៌នាយ៉ាងច្បាស់ និងជំហានដោះស្រាយ៖

1. **គ្មាន SERPAPI_KEY នៅក្នុងឯកសារ .env**
   - ប្រសិនបើអ្នកឃើញកំហុស `SERPAPI_KEY environment variable not found` មានន័យកម្មវិធីរបស់អ្នកមិនអាចរកឃើញកូនសោ API ដើម្បីចូល​ដំណើរការ SerpAPI។ ដើម្បីជួសជុល សូមបង្កើតឯកសារ `.env` នៅកន្លែងដើមគម្រោងរបស់អ្នក (បើមិនមាន) ហើយបន្ថែមបន្ទាត់មួយដូចជា `SERPAPI_KEY=your_serpapi_key_here`។ សូមប្រាកដថាប្តូរពាក្យ `your_serpapi_key_here` ជាមួយកូនសោពិតប្រាកដរបស់អ្នកពីវេបសាយ SerpAPI។

2. **កំហុសថា មិនរក Module មិនឃើញ**
   - កំហុសដូចជា `ModuleNotFoundError: No module named 'httpx'` មានន័យថាបណ្ណាល័យ Python និរន្តរមួយខ្វាច់។ វាបង្ហាញថាអ្នកមិនបានដំឡើងទាំងអស់ចាំបាច់ទេ។ ដើម្បីដោះស្រាយ សូមរត់ `pip install -r requirements.txt` ក្នុងពហុកំណត់បញ្ជារបស់អ្នកដើម្បីដំឡើងបណ្ណាល័យទាំងអស់ដែលគម្រោងដំណើរការ។

3. **បញ្ហាការតភ្ជាប់**
   - ប្រសិនបើអ្នកបានកំហុសដូចជា `Error during client execution` គេបង្ហាញថាអតិថិជនមិនអាចភ្ជាប់ទៅម៉ាស៊ីនមេ ឬម៉ាស៊ីនមេមិនដំណើរការត្រឹមត្រូវ។ សូមពិនិត្យមើលថាអតិថិជន និងម៉ាស៊ីនមេមានសមត្ថភាពគ្នា និងថា `server.py` មានទីតាំងហើយដំណើរការត្រឹមត្រូវ។ ការចាប់ផ្តើមឡើងវិញម៉ាស៊ីនមេ និងអតិថិជនអាចជួយបាន​ផងដែរ។

4. **កំហុស SerpAPI**
   - កំហុស `Search API returned error status: 401` មានន័យថាកូនសោ SerpAPI របស់អ្នកគ្មាន ឬមិនត្រឹមត្រូវ ឬផុតកំណត់។ សូមចូលទៅផ្ទាំងគ្រប់គ្រង SerpAPI របស់អ្នក ពិនិត្យកូនសោ ហើយធ្វើបច្ចុប្បន្នភាពឯកសារ `.env` ប្រសិនបើចាំបាច់។ បើកូនសោត្រឹមត្រូវ ប៉ុន្តែអ្នកនៅតែឃើញកំហុសនេះ សូមពិនិត្យថា ថ្នាក់ឥតគិតថ្លៃរបស់អ្នកមិនបានប្រើប្រាស់Quota រួចហើយ។

### របៀប Debug

ដោយលំនាំដើម កម្មវិធីកំណត់បញ្ចូលមានតែកំណត់ព័ត៌មានសំខាន់ៗប៉ុណ្ណោះ។ ប្រសិនបើអ្នកចង់មើលពត៌មានលម្អិតបន្ថែមពីអ្វីកំពុងកើតឡើង (ឧ. ដើម្បីវាយតម្លៃបញ្ហាកំប្លែង) អ្នកអាចបើករបៀប DEBUG។ វានឹងបង្ហាញរបាយការណ៍លម្អិតពីសកម្មភាព HTTP, ចម្លើយ និងព័ត៌មានក្នុងផ្នែកខាងក្នុងផ្សេងៗ។

**ឧទាហរណ៍៖ លទ្ធផលធម្មតា**
```plaintext
2025-06-01 10:15:23,456 - __main__ - INFO - Calling general_search with params: {'query': 'open source LLMs'}
2025-06-01 10:15:24,123 - __main__ - INFO - Successfully called general_search

GENERAL_SEARCH RESULTS:
... (search results here) ...
```

**ឧទាហរណ៍៖ លទ្ធផល DEBUG**
```plaintext
2025-06-01 10:15:23,456 - __main__ - INFO - Calling general_search with params: {'query': 'open source LLMs'}
2025-06-01 10:15:23,457 - httpx - DEBUG - HTTP Request: GET https://serpapi.com/search ...
2025-06-01 10:15:23,458 - httpx - DEBUG - HTTP Response: 200 OK ...
2025-06-01 10:15:24,123 - __main__ - INFO - Successfully called general_search

GENERAL_SEARCH RESULTS:
... (search results here) ...
```

សូមចាប់អារម្មណ៍ថារបៀប DEBUG រួមបញ្ចូលបន្ទាត់បន្ថែមអំពីសំណើ HTTP ពីការឆ្លើយតប និងពត៌មានផ្សេងៗខាងក្នុង។ វាជួយយល់ពីកំហុសបានយ៉ាងមានប្រសិទ្ធភាព។

ដើម្បីបើករបៀប DEBUG សូមកំណត់កម្រិតកំណត់ហេតុទៅ DEBUG នៅខាងលើឯកសារ `client.py` រឺ `server.py` របស់អ្នក៖

# [Python](#tab/python-debug)
```python
# នៅខាងលើរបស់ client.py ឬ server.py របស់អ្នក
import logging
logging.basicConfig(
    level=logging.DEBUG,  # ប្ដូរពី INFO ទៅ DEBUG
    format="%(asctime)s - %(name)s - %(levelname)s - %(message)s"
)
```
