## ថ្ងៃទី 28 ខែមករា 2026

### ការពិនិត្យភាពសមរម្យរបស់ MCP Specification 2025-11-25

#### ការកែលម្អគំនិតមូលដ្ឋាន (01-CoreConcepts/)
- **Client Primitive ថ្មី - Roots**៖ បន្ថែមឯកសារប្រកបដោយការលម្អិតលើ client primitive Roots ដែលអនុញ្ញាតឲ្យម៉ាស៊ីនបម្រើយល់ពីភាពដែនកំណត់របស់ប្រព័ន្ធគ្រប់គ្រងឯកសារ និងសិទ្ធិចូលដំណើរការ
- **ការអនុស្សាវរីយ៍ឧបករណ៍**៖ បន្ថែមឯកសារអំពីការអក្សរពីពលកម្មឧបករណ៍ (`readOnlyHint`, `destructiveHint`) ដើម្បីធ្វើការសំរេចចិត្តប្រតិបត្តិការល្អប្រសើរជាងមុន
- **ការហៅឧបករណ៍ក្នុងការស្ដារតំរូវការ**៖ ធ្វើបច្ចុប្បន្នភាពឯកសារស្ដារតំរូវការជាមួយប៉ារ៉ាម៉ែត្រ `tools` និង `toolChoice` សម្រាប់ការហៅឧបករណ៍ជាមួយម៉ូដែលបើកពន្លឺនៅពេលស្ដារតំរូវការ
- **ការចេញសេចក្តីស្នើសុំ URL Mode**៖ បន្ថែមឯកសារអំពីការចេញសំណើបែប URL សម្រាប់កិច្ចការប្រតិបត្តិការបណ្តាញក្រៅឌីណាមិចដោយម៉ាស៊ីនបម្រើ
- **បិទកិច្ចការសាកល្បង (កំពុងសាកល្បង)**៖ បន្ថែមផ្នែកថ្មី documenting លើគម្រោងកិច្ចការសាកល្បងដែលជាការបិទប្រតិបត្តិការជាយូរអង្វែង និងការទាញយកលទ្ធផលយឺត
- **គាំទ្រពិសេសរូបតំណាង**៖ បានពិពណ៌នាថាឧបករណ៍ ធនធាន គំរូធនធាន និងវេយ្យាករណ៍នាពេលនេះអាចរួមបញ្ចូលរូបតំណាងជាមetadata បន្ថែម

#### ការត្រួតពិនិត្យភាពសម្របសម្រួលនៃការបញ្ជាក់
- **កំណែ Protocol**៖ បញ្ជាក់ថាឯកសារទាំងអស់យោងទៅលើ MCP Specification 2025-11-25
- **ការតម្រឹមស្ថាបត្យកម្ម**៖ ផ្តល់ការបញ្ជាក់ជាក់លាក់លើស្ថាបត្យកម្មពីរដប់ (Data Layer + Transport Layer)
- **ឯកសារ Primitive**៖ វាយតម្លៃ server primitives (Resources, Prompts, Tools) និង client primitives (Sampling, Elicitation, Logging, Roots)
- **មធ្យោបាយដឹកជញ្ជូន**៖ ការត្រួតពិនិត្យក្រោមផ្លូវ STDIO និង HTTP Streaming មានភាពត្រឹមត្រូវ
- **ការណែនាំសុវត្ថិភាព**៖ បញ្ជាក់ការផ្គួចផ្គង់ជាមួយការណែនាំគោលការណ៍សុវត្ថិភាព MCP បច្ចុប្បន្ន

#### លក្ខណៈ MCP 2025-11-25 សំខាន់ៗបានរាយការណ៍
- **ការស្វែងរកជាមួយ OpenID Connect**៖ ការរកឃើញសេវាកម្ម auth តាម OIDC
- **ឯកសារព័ត៌មាន Clients OAuth ID**៖ ផ្លូវការបញ្ចូលការចុះបញ្ជី client
- **JSON Schema 2020-12**៖ ជាភាសាសម្រាប់កំណត់ MCP schema
- **ប្រព័ន្ធដំណាក់កាល SDK**៖ កំណត់ទាមទារផ្លូវការសម្រាប់ការគាំទ្រលក្ខណៈលក្ខណៈ SDK និងការថែទាំ
- **រចនាសម្ព័ន្ធអភិបាលកម្ម**៖ កំណត់ក្រុមបុគ្គលិកសម្រាប់ការដឹកនាំអភិបាលកម្ម MCP

  - ធ្វើបច្ចុប្បន្នភាពយោងគ្រប់កំណែ spec ពី 2025-06-18 ទៅ 2025-11-25
  - ធ្វើបច្ចុប្បន្នភាពកាលបរិច្ឆេទឯកសារពី 18 ខែសីហា 2025 ទៅ 18 ខែធ្នូ 2025
  - ធ្វើត្រួតពិនិត្យឲ្យ URL សារព័ត៌មានទាំងអស់គឺដាក់បង្ហាញឆ្នាំបច្ចុប្បន្ន

  - **ដំណោះស្រាយសុវត្ថិភាព Microsoft**៖ ត្រួតពិនិត្យពាក្យសំគាល់និងតំណភ្ជាប់ពាក់ព័ន្ធនឹង Prompt Shields (មុននេះគឺ "ការស្ទង់ហានិភ័យ Jailbreak"), Azure Content Safety, Microsoft Entra ID, និង Azure Key Vault
  - **សុវត្ថិភាព OAuth 2.1**៖ បញ្ជាក់សម្របសម្រួលជាមួយវិធីសាស្ត្រសុវត្ថិភាព OAuth ថ្មីៗ
  - **ស្តង់ដារ OWASP**៖ បានត្រួតពិនិត្យថា OWASP Top 10 សម្រាប់ LLMs កំណត់រួចហើយនៅតែទាន់សម័យ
  - **សេវាកម្ម Azure**៖ ធ្វើត្រួតពិនិត្យលីងឯកសារជាច្រើនរបស់ Microsoft Azure និងវិធីសាស្ត្រសុវត្ថិភាពល្អបំផុត
- **ការផ្គូផ្គងទៅស្តង់ដារ**៖ រួមបញ្ចូលស្តង់ដារសុវត្ថិភាពគ្រប់ចំណុចដែលបានយោង
  - NIST AI Risk Management Framework

  - ស៊ុមសុវត្ថិភាព និងការអនុវត្តនៅ Azure
- **ធនធានអនុវត្ត**៖ ត្រួតពិនិត្យឧបករណ៍ក៏ដូចជាឯកសារណែនាំអនុវត្តទាំងអស់

  - **គំរូអនុវត្តន៍**៖ ស្ថានភាពពិតប្រាកដដែលទម្លាប់ម៉ាស៊ីនបម្រើកម្រិតទាបជួយបង្កើតមុខងារកម្រិតខ្ពស់និងរចនាសម្ព័ន្ធជាក់លាក់

#### ការផ្ទៀងផ្ទាត់សាមញ្ញ (03-GettingStarted/11-simple-auth)
- **ជំពូកថ្មីបានបន្ថែម**៖ មគ្គុទេសក៍ជំហានតាមជំហានដើម្បីអនុវត្តការផ្ទៀងផ្ទាត់សាមញ្ញក្នុងម៉ាស៊ីនបម្រើ MCP
  - **គំនិតស្តីពី Auth**៖ ពណ៌នាច្បាស់ពីការផ្ទៀងផ្ទាត់ និងការអនុញ្ញាត និងការគ្រប់គ្រងអត្តសញ្ញាណ
  - **អនុវត្ត Auth យ៉ាងសាមញ្ញ**៖ គំរូការផ្ទៀងផ្ទាត់ដោយប្រើMiddleware ក្នុង Python (Starlette) និង TypeScript (Express) ជាមួយឧទាហរណ៍កូដ
  - **ការរីកចម្រើនទៅកាន់សុវត្ថិភាពកម្រិតខ្ពស់**៖ ជំនួយការចាប់ផ្តើមពីAuthសាមញ្ញទៅOAuth 2.1 និង RBAC ជាមួយយោងទៅមូឌុលសុវត្ថិភាពកម្រិតខ្ពស់

ការបន្ថែមទាំងនេះផ្តល់នូវការណែនាំដែលអនុវត្តបានជាក់ស្តែង ដើម្បីសាងសង់ម៉ាស៊ីនបម្រើ MCP ដែលរឹងមាំ សុវត្ថិ និងប្រែប្រួលឆាប់រហ័ស ដើម្បីភ្ជាប់គំនិតមូលដ្ឋានទៅគំរូផលិតកម្មកម្រិតខ្ពស់។

  - Servers ត្រូវបានពង្រឹងជាមួយស្ថានភាពផ្សារភ្ជាប់ក្នុងផ្លូវកន្លែង និងឆ្ងាយ
- **ការកែសម្រួលដើមទាំងស្រុង**: បានប្ដូរញឹកញាប់សម្រាប់ Primitive របស់ម៉ាស៊ីនបម្រើ និងម៉ាស៊ីនភ្ញៀវ
  - Server Primitives: ធនធាន (ប្រភពទិន្នន័យ), ព្រីម៉птом (គំរូ), ឧបករណ៍ (មុខងារអាចដំណើរការ) ជាមួយការពន្យល់លម្អិត និងឧទាហរណ៍
  - Client Primitives: ការទាញយក (ការបញ្ចប់ LLM), ការស្នើសុំ (ការបញ្ចូលអ្នកប្រើ), ការចុះបញ្ជី (ការត្រួតពិនិត្យ/តាមដាន)
  - បានធ្វើបច្ចុប្បន្នភាពជាមួយលំនាំវិធីសាស្ត្រស្វែងរក (`*/list`), ទទួលយក (`*/get`), និងអនុវត្ត (`*/call`)
- **រចនាសម្ព័ន្ធព្រឹត្តិប័ត្រ**: បានណែនាំម៉ូដែលរចនាសម្ព័ន្ធពីរដាន់
  - ស្រទាប់ទិន្នន័យ: ដំណាក់កាល JSON-RPC 2.0 ជា​មូលដ្ឋាន​ជាមួយ​ការគ្រប់គ្រងជីវិត ត្រឹមត្រូវ និង primitive
  - ស្រទាប់ផងដឹកជញ្ជូន: STDIO (ក្នុងបរិវេណ), និង Streamable HTTP ជាមួយ SSE (ឆ្ងាយ) ជា​និរន្តរដ្ឋផងដឹកជញ្ជូន
- **ស៊ុមសុវត្ថិភាព**: គោលការណ៍សុវត្ថិភាពទូលំទូលាយរួមបញ្ចូលការយល់ព្រមអ្នកប្រើយ៉ាងច្បាស់ ការពារឯកជនភាពទិន្នន័យ សុវត្ថិភាពក្នុងការអនុវត្តឧបករណ៍ និងសុវត្ថិភាពស្រទាប់ផៃ
- **លំនាំការទំនាក់ទំនង**: បានធ្វើបច្ចុប្បន្នភាពសារ​ព្រឹត្តិប័ត្រដើម្បីបង្ហាញដំណើរការចាប់ផ្តើម ស្វែងរក អនុវត្ត និងដំណឹងជូន
- **ឧទាហរណ៍កូដ**: បានធ្វើបច្ចុប្បន្នភាពឧទាហរណ៍ភាសាច្រើន (.NET, Java, Python, JavaScript) ដើម្បីបង្ហាញលំនាំ MCP SDK បច្ចុប្បន្ន

#### សុវត្ថិភាព (02-Security/) - ការកែសម្រួលសុវត្ថិភាពទូលំទូលាយ  
- **ការសម្របសម្រួលតាមស្តង់ដារ**: សម្របសម្រួលពេញលេញតាមតម្រូវការសុវត្ថិភាព MCP Specification 2025-06-18
- **​ការវិវឌ្ឍន៍ការផ្ទៀងផ្ទាត់​អត្តសញ្ញាណ**: បានរៀបរាប់ពីការវិវឌ្ឍចេញពីម៉ាស៊ីនបម្រើ OAuth ផ្ទាល់ខ្លួនសម្រាប់ផ្ទេរជាប្រភេទតំណាងផ្តល់អត្តសញ្ញាណក្រៅ (Microsoft Entra ID)
- **វិភាគគ្រោះថ្នាក់ AI ដ៏ពិសេស**: បានពង្រឹងការគ្របដណ្តប់លើវ៉ាទ័រវាយប្រហារអាយអាយទំនើប
  - ស្ថានភាពវាយប្រហារដោយបញ្ចូលព្រីម៉птомជាក់លាក់ជាមួយឧទាហរណ៍ពិតប្រាកដ
  - យន្តការវាយប្រហារត្រួតពិនិត្យឧបករណ៍ និងលំនាំវាយប្រហាររបស់ “rug pull”
  - ការវាយប្រហារបំពុលបង្អួចបរិបទ និងបំភាន់ម៉ូដែល
- **ដំណោះស្រាយសុវត្ថិភាព Microsoft AI**: គ្របដណ្តប់ទាំងមូលលើប្រព័ន្ធសុវត្ថិភាព Microsoft
  - AI Prompt Shields ជាមួយកំណត់សញ្ញាលំនាំបញ្ចេញ ភ្លឺស្ទើង និងបិទបាំងណែនាំ
  - លំនាំសម្របសម្រួល Azure Content Safety
  - GitHub Advanced Security សម្រាប់ការពារច្រកផ្គត់ផ្គង់
- **ការកាត់បន្ថយគ្រោះថ្នាក់កម្រិតខ្ពស់**: ត្រួតពិនិត្យសុវត្ថិភាពលម្អិតសម្រាប់
  - ការជ្រុះបំផ្លាញសវនកម្មជាមួយស្ថានភាពវាយប្រហារពិសេសរបស់ MCP និងតម្រូវឲ្យមាន session ID រចនាសម្ព័ន្ធគ្រប់គ្រាន់
  - បញ្ហាអ្នកតំណាងនិយមបំភាន់នៅក្នុងស្ថានភាពម៉ាស៊ីនភ្នាក់ងារពីរនៃ MCP ជាមួយតម្រូវការយល់ព្រមច្បាស់លាស់
  - របាំងវ៉ាទ័រត្រូវបានប្រើ Token ក្នុងការចេញផ្សាយជាមួយត្រួតពិនិត្យត្រឹមត្រូវអនុវត្តមុខងារ
