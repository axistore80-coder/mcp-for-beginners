# ប្រវត្តិបម្រែបម្រួល៖ នៅលើមេរៀន MCP សម្រាប់អ្នកចាប់ផ្ដើម

ឯកសារនេះជាឯកសារកត់ត្រានៃការផ្លាស់ប្តូរដ៏សំខាន់ទាំងអស់នៃមេរៀន Model Context Protocol (MCP) សម្រាប់អ្នកចាប់ផ្ដើម។ ការផ្លាស់ប្តូរត្រូវបានចុះបញ្ជីតាមលំដាប់ពេលវេលាត្រឡប់ក្រោយ (ការផ្លាស់ប្តូរថ្មីៗនៅដើម)។

## មេសា ១១, ២០២៦

### មេរៀនថ្មី ការជួសជុលឯកសារ និងការអាប់ដេត dependencies

#### បន្ថែមមាតិកាមេរៀនថ្មី

**ម៉ូឌុល ០៥ - ប្រធានបទខ្ពស់**
- **មេរៀន 5.17៖ ការគិតពហុភ្នាក់ងារប្រឆាំងគ្នាជាមួយ MCP** (`05-AdvancedTopics/mcp-adversarial-agents/README.md`): មេរៀនថ្មីពិស្តារពីលំនាំវាយតវ៉ាប្រឆាំងសម្រាប់ប្រព័ន្ធពហុភ្នាក់ងារ
  - រូបប្លង់ស្ថាបត្យកម្ម Mermaid៖ ភ្នាក់ងារពីររួចទៅម៉ាស៊ីនមេ MCP ចែករំលែក → សេចក្ដីអត្ថាធិប្បាយវាយតវ៉ា → វិនិច្ឆ័យ → សេចក្ដីសម្រេច
  - ម៉ាស៊ីនមេឧបករណ៍ MCP ចែករំលែក (`web_search` + `run_python`) ដែលបានអនុវត្តនៅក្នុង Python និង TypeScript
  - បញ្ជាកម្មប្រព័ន្ធប្រឆាំង (FOR / AGAINST / Judge) រួមមានតម្រូវការប្រើឧបករណ៍យ៉ាងច្បាស់
  - អ្នកសម្របសម្រួលវាយតវ៉ា ក្នុង Python, TypeScript និង C# រៀបចំវគ្គ និងផ្ញើរបញ្ចប់ជំនួប
  - ការតភ្ជាប់ MCP `ClientSession` សម្រាប់អ្នកសម្របសម្រួលទៅការហៅឧបករណ៍ពិត
  - តារាងករណីប្រើប្រាស់ (ការរកឃើញហាល់លូស៊ីណាស់, គំរោងគំរាម, ពិនិត្យការរចនាអេភីអាយ, ការផ្ទៀងផ្ទាត់ព័ត៌មាន, ជ្រើសរើសបច្ចេកវិទ្យា)
  - ការពិចារណាសុវត្ថិភាព៖ ការដំណើរការទាញយកក្នុងផ្សែង, ការផ្ទៀងផ្ទាត់ការហៅឧបករណ៍, ការកំណត់អត្រា, ការចុះបញ្ជី audit
  - លំហាត់មានរចនាសម្ព័ន្ធជាមួយ៣ករណីអនុវត្ត (ការពិនិត្យកូដ, សំឡេងស្ដីការស្ថាបត្យកម្ម, ការគ្រប់គ្រងមាតិកា)

#### ការកែលម្អចំណេះដឹងគោលការណ៍ (01-CoreConcepts/)
- **Primitive Client ថ្មី - Roots**: បន្ថែមឯកសារពិស្តារពី primitive client Roots ដែលអនុញ្ញាតឲ្យម៉ាស៊ីនមេយល់ដឹងពីដែនកំណត់ប្រព័ន្ធឯកសារ និងសិទ្ធិចូលប្រើ
- **សម្គាល់ឧបករណ៍**: បន្ថែមឯកសារអំពីសម្គាល់ឧបករណ៍អាកប្បកិរិយា (`readOnlyHint`, `destructiveHint`) សម្រាប់ការជ្រើសរើសដំណើរការឧបករណ៍ល្អប្រសើរ
- **ការហៅឧបករណ៍ក្នុងការសោតសំឡេង**: កែប្រែឯកសារ Sampling ដើម្បីរួមបញ្ចូល `tools` និង `toolChoice` ជាប៉ារ៉ាម៉ែត្រ សម្រាប់ការហៅឧបករណ៍ដោយម៉ូដែលនៅពេលស្នើសុំសោត
- **ការលើកឡើងរបៀប URL**: បន្ថែមឯកសារពីការលើកឡើងប្រភេទ URL សម្រាប់ការបញ្ជូនបណ្តោះអាសន្នក់តាមបណ្តាញក្រៅពីម៉ាស៊ីនមេ
- **បច្ចេកវិទ្យាការងារ (ជាសាកល្បង)**: បន្ថែមផ្នែកឯកសារដែលពន្យល់ពីមុខងារបច្ចេកវិទ្យាការងារសាកល្បង សម្រាប់វាំងននការបញ្ចូលបញ្ជារឿងនិងការទាញយកលទ្ធផលយឺត
- **ការគាំទ្រជាមួយរូបតំណាង**: ឯកសារបានដាក់សំគាល់ថា ឧបករណ៍ធនធាន គំរូធនធាន និង prompt អាចបន្ថែមរូបតំណាងជាតម្រង់មេតាដាតាបន្ថែមបាន

    - ការការពារវាយប្រហារ Confused Deputy ជាមួយ PKCE និងការផ្ទៀងផ្ទាត់ការយល់ព្រមអ្នកប្រើ
    - ការការពារចូរប៊ីតូកែនជាមួយការផ្ទៀងផ្ទាត់ទស្សនិកជន និងការគ្រប់គ្រងប៊ីតូកែនយ៉ាងសុវត្ថិភាព
    - ការការពារសម័យ hijacking ជាមួយការចងអក្សរករ និងវិភាគអាកប្បកិរិយា
  - **ការរួមបញ្ចូលសុវត្ថិភាពសហគ្រាស**: ការត្រួតពិនិត្យ Azure Application Insights, បំពង់រកឃើញគ្រោះថ្នាក់ និងសុវត្ថិភាពខ្សែផ្គត់ផ្គង់
  - **បញ្ជីត្រួតពិនិត្យអនុវត្តន៍**: ការបែងចែកច្បាស់លាស់គ្រប់គ្រងសុវត្ថិភាពត្រូវបានបញ្ជាក់ និងអនុវត្តន៍ផ្តល់អត្ថប្រយោជន៍ Microsoft security ecosystem

### គុណភាពឯកសារ និងការស្របតាមស្តង់ដា
- **យោងបញ្ជាក់ស៊ឹពឺស៊ី**: កែប្រែបញ្ជាក់ទាំងអស់ទៅ MCP Specification 2025-06-18 បច្ចុប្បន្ន
- **Microsoft Security Ecosystem**: ការណែនាំរួមបញ្ចូលពង្រឹងជើង
- **ការអនុវត្តជាក់លាក់**: បន្ថែមឧទាហរណ៍កូដលម្អិតក្នុង .NET, Java, និង Python ជាមួយគំរូសហគ្រាស
- **ការរៀបចំប្រើសរើសធនធាន**: បែងចែកធនធានផ្លូវការនិងស្តង់ដារ និងមគ្គុទេសក៍អនុវត្តន៍
- **សញ្ញាទស្សនៈ**: សម្គាល់ច្បាស់ចំនួនការទាមទារ ត្រូវប្រតិបត្ដិ និងអនុញ្ញាត

#### គំនិតមូលដ្ឋាន (01-CoreConcepts/) - ការផ្លាស់ប្តូរទាន់សម័យពេញលេញ
- **ការធ្វើបច្ចុប្បន្នភាពកំណែប្រព័ន្ធ**: បានធ្វើបច្ចុប្បន្នភាពទៅ MCP Specification 2025-06-18 ជាមួយកំណែផ្អែកលើកាលបរិច្ឆេទ (ទ្រង់ទ្រាយ YYYY-MM-DD)
- **កំណែផ្លាស់ប្តូរស្ថាបត្យកម្ម**: ពណ៌នាបន្ថែមពី Hosts, Clients និង Servers ដើម្បីផ្តល់សម្រួលលើរចនាសម្ព័ន្ធ MCP បច្ចុប្បន្ន
  - Hosts កំណត់ច្បាស់ថាជាកម្មវិធី AI ដែលសម្របសម្រួលការភ្ជាប់ MCP client ច្រើន
  - Clients ពិពណ៌នាថាជាឧបករណ៍ភ្ជាប់ប្រតិបត្តិការតាមប្រព័ន្ធព្រំភាគហ៊ុនមួយ-ទៅ-មួយ
  - Servers បានសម្រួលជាមួយស្នូលបច្ចេកទេសដំណាក់កាលមូលដ្ឋាន និងចម្ងាយ
- **ការផ្លាស់ប្តូរចាស្ថាបត្យកម្ម primitive**: ការកែប្រែពេញលេញសម្រាប់ប្រភេទ server និង client primitives
  - Server Primitives: ធនធាន (ប្រភពទិន្នន័យ), prompts (គំរូ), ឧបករណ៍ (មុខងារអនុវត្ត) ជាមួយការពិពណ៌នារួម និងឧទាហរណ៍
  - Client Primitives: ការគំរូ (LLM បញ្ចប់), ការចុះផ្សាយ (ការបញ្ចូលអ្នកប្រើ), ការចុះបញ្ជីប្រវត្តិ (debugging/monitoring)
  - បានធ្វើបច្ចុប្បន្នភាពសម្រាប់ការស្វែងរក (`*/list`), ទាញយក (`*/get`), និងអនុវត្ត (`*/call`) វិធីសាស្រ្ត
- **ស្ថាបត្យកម្មប្រព័ន្ធផ្ទេរ**: បង្ហាញម៉ូឌែលស្ថាបត្យកម្មពីរជាន់
  - ស្រទាប់ទិន្នន័យ: មូលដ្ឋាន JSON-RPC 2.0 ជាមួយការគ្រប់គ្រងជីវិត និង primitives
  - ស្រទាប់ដឹកជញ្ជូន: STDIO (ក្នុងតំបន់) និង Streamable HTTP ជាមួយ SSE សម្រាប់ដឹកជញ្ជូនពីចម្ងាយ
- **ស៊ឹវហ្វ្រេមសុវត្ថិភាព**: គោលការណ៍សុវត្ថិភាពទូលំទូលាយមានការយល់ព្រមអ្នកប្រើច្បាស់លាស់, ការការពារយុទ្ធសាស្រ្តទិន្នន័យឯកជន, សុវត្ថិភាពការប្រតិបត្តិការ​ឧបករណ៍ និងសុវត្ថិភាពស្រទាប់ដឹកជញ្ជូន
- **លំនាំសារប្រព័ន្ធ**: ប្រព័ន្ធលំនាំ protocol ថ្មីបង្ហាញការចាប់ផ្តើម, ស្វែងរក, អនុវត្ត និងការជូនដំណឹង
- **ឧទាហរណ៍កូដ**: បានធ្វើបច្ចុប្បន្នភាពឧទាហរណ៍ច្រើនភាសា (.NET, Java, Python, JavaScript) តាមលំនាំ MCP SDK បច្ចុប្បន្ន

#### សុវត្ថិភាព (02-Security/) - ការកែប្រែសុវត្ថិភាពទូលំទូលាយ  
- **ការស្របតាមស្តង់ដា**: ស្របច្បាប់ពេញលេញជាមួយ MCP Specification 2025-06-18
- **ការវឌ្ឍន៍ការផ្ទៀងផ្ទាត់អត្តសញ្ញាណ**: ឯកសារពីប្លង់កែលម OAuth ផ្ទាល់ខ្លួនទៅការចាត់ចែង identity ពីផ្គត់ផ្គង់ខាងក្រៅ (Microsoft Entra ID)
- **វិភាគគ្រោះថ្នាក់ AI ជាក់លាក់**: ការត្រួតពិនិត្យបន្ថែមលើវ៉ិចទ័រវាយប្រហារ AI ទំនើប
  - សមាសភាពវាយប្រហារចាក់ប្រូមជាក់លាក់ជាមួយឧទាហរណ៍ពិត
  - គ្រឿងញៀនឧបករណ៍និងការវាយប្រហារល្បែងបោកប្រាស់
  - ការបុកបាញ់រប្រអប់បរិបទ និងការបាក់បែកគំរូ
- **ដំណោះស្រាយសុវត្ថិភាព AI Microsoft**: រាប់បញ្ចូលលម្អិត Microsoft security ecosystem
  - AI Prompt Shields ជាមួយបច្ចេកទេសរកឃើញកម្រិតខ្ពស់, លេខាធិការ spotlighting និងអ្នកកំណត់ delimiter
  - ការរួមបញ្ចូល Azure Content Safety
  - GitHub Advanced Security សម្រាប់ការពារខ្សែផ្គត់ផ្គង់
- **ការកាត់បន្ថយគ្រោះថ្នាក់កម្រិតខ្ពស់**: ឧបករណ៍គ្រប់គ្រងសុវត្ថិភាពសម្រាប់
  - ការចូលរួម hijacking ជាមួយសិនារីយ៉ូជាក់លាក់ MCP និងការទាមទាររ៉ឺម៉ោត session ID អក្សរករ
  - បញ្ហា Confused deputy សម្រាប់ MCP proxy ជាមួយការយល់ព្រមច្បាស់លាស់
  - ខ្សែប៊ីតូកែនបុកបាញ់ជាមួយការត្រួតពិនិត្យច្បាស់លាស់
- **សុវត្ថិភាពខ្សែផ្គត់ផ្គង់**: ពង្រីកការពារខ្សែផ្គត់ផ្គង់ AI រួមមានម៉ូឌែលមូលដ្ឋាន, សេវាកម្ម embeddings, អ្នកផ្គត់ផ្គង់ context និង API ពីភាគីទីបី
- **សុវត្ថិភាពមូលដ្ឋាន**: ការរួមបញ្ចូលល្អជាមួយស្តង់ដារសុវត្ថិភាពសហគ្រាស រួមមានស្ថាបត្យកម្មមិនទុកចិត្ត និង Microsoft security ecosystem
- **ការរៀបចំប្រើសរើសធនធាន**: បែងចែកតំណភ្ជាប់ធនធានដោយប្រភេទ (ឯកសារផ្លូវការ, ស្តង់ដារ, ការស្រាវជ្រាវ, ដំណោះស្រាយ Microsoft, មគ្គុទេសក៍អនុវត្តន៍)

### ការកែលម្អគុណភាពឯកសារ
- **គោលបំណងរៀនរចនាសម្ព័ន្ធ**: ពង្រីកគោលបំណងរៀនជាក់លាក់ និងអាចអនុវត្តបាន
- **ការតភ្ជាប់គ្នារវាងប្រធានបទ**: បន្ថែមតំណភ្ជាប់រវាងប្រធានសុវត្ថិភាព និងគំនិតមូលដ្ឋាន

- **ការណែនាំអនុវត្ត**: បន្ថែមមគ្គុទេសក៍អនុវត្តជាក់លាក់ទូលំទូលាយទាំងពីរផ្នែក
