# 🚀 ម៉ាស៊ីនបម្រើ MCP ជាមួយ PostgreSQL - មគ្គុទេសក៍រៀនពេញលេញ

## 🧠 មើលទើបពី ផ្លូវរៀនបញ្ចូលមូលដ្ឋានទិន្នន័យ MCP

មគ្គុទេសក៍រៀនពេញលេញនេះ ស្វែងបង្រៀនអ្នកធ្វើម៉ាស៊ីនបម្រើ **Model Context Protocol (MCP)** ដែលមានភាពរួបរួមជាមួយមូលដ្ឋានទិន្នន័យតាមរយៈការអនុវត្តវិភាគលក់តាមអាជីវកម្ម។ អ្នកនឹងរៀនលំនាំប្រកបដោយវិជ្ជាជីវៈ រួមបញ្ចូល **សន្តិសុខជាន់ជួរ (Row Level Security - RLS)**, **ស្វែងរកអត្ថន័យ**, **ការរួមបញ្ចូល Azure AI** និង **ការចូលដំណើរការទិន្នន័យពហុជួលក**

មិនថាអ្នកជាអ្នកអភិវឌ្ឍផ្នែកខាងក្រោយ, វិស្វករជំនាញ AI ឬស្ថាបនិកទិន្នន័យ ក៏ដោយ មគ្គុទេសក៍នេះផ្តល់ការរៀនមានរចនា និងគំរូពិតប្រាកដរួមនឹងហ្វឹកហាត់ដៃ ដែលនាំអ្នកឆ្លងកាត់ម៉ាស៊ីនបម្រើ MCP ក្នុង https://github.com/microsoft/MCP-Server-and-PostgreSQL-Sample-Retail ។

- **មូលដ្ឋានទិន្នន័យលក់ពហុតារាង** មានការបញ្ជាទិញអតិថិជន, ផលិតផល និងសារពើភ័ណ្ឌ

- **ស្វែងរកផលិតផលមានអត្ថន័យ** ប្រើ embedding Azure OpenAI
- **បញ្ចូល AI Chat ក្នុង VS Code** សម្រាប់សំណួរភាសាធម្មជាតិ

- **តាមដានជាស្ថេរភាព** ជាមួយ Application Insights

## 🎯 លក្ខខណ្ឌមុនសម្រាប់ការរៀន

ដើម្បីទទួលបានអត្ថប្រយោជន៍ខ្ពស់ពីផ្លូវរៀននេះ អ្នកគួរតែមាន៖

- **បទពិសោធន៍កម្មវិធី**: រៀនភាសា Python (សំណូមពរជាមុន) ឬភាសាដូចគ្នា
- **ចំនេះដឹងមូលដ្ឋានទិន្នន័យ**: យល់ដឹងមូលដ្ឋានអំពី SQL និងមូលដ្ឋានទិន្នន័យទាក់ទងគ្នា
- **មូលដ្ឋានអំពី API**: យល់ដឹងអំពី REST API និងគំនិត HTTP
- **ឧបករណ៍អភិវឌ្ឍ**: ប្រើប្រាស់បន្ទាត់ពាក្យបញ្ជា, Git និងកម្មវិធីកែសម្រួលកូដ
- **មូលដ្ឋានពពក**: (ជាជម្រើស) ចំណេះដឹងមូលដ្ឋានអំពី Azure ឬបណ្តាញពពកដូចគ្នា
- **ស្គាល់ Docker**: (ជាជម្រើស) យល់ដឹងពីមូលដ្ឋាន containerization

### មគ្គុទេសក៍សិក្សា

លាបនីមួយៗរួមមាន៖
- **គោលដៅរៀនច្បាស់លាស់** - អ្វីដែលអ្នកនឹងសម្រេច
- **ណែនាំជំហ៊ានតាមជំហ៊ាន** - មគ្គុទេសក៍អនុវត្តលម្អិត
- **គំរូកូដ** - ឧទាហរណ៍រួមមានការពន្យល់
- **ហ្វឹកហាត់** - ឱកាសប្រើប្រាស់ដោយផ្ទាល់
- **មគ្គុទេសក៍ដោះស្រាយបញ្ហា** - បញ្ហាទូទៅ និងដំណោះស្រាយ
- **ធនធានបន្ថែម** - វិភាគទិន្នន័យបន្ថែមនិងការស្វែងរកឆ្ងាយ

### ការត្រួតពិនិត្យលក្ខខណ្ឌមុន

មុនចាប់ផ្ដើមលាបណាមួយ អ្នកនឹងទទួលបាន៖
- **ចំណេះដឹងចាំបាច់** - អ្វីដែលអ្នកត្រូវដឹងមុន
- **ត្រួតពិនិត្យបរិដ្ឋាន** - វិធីសាស្ត្រត្រួតពិនិត្យបរិដ្ឋានរបស់អ្នក
- **ការវាយតម្លៃពេលវេលា** - ពេលវេលាគ្រោងទុកសម្រាប់បញ្ចប់
- **លទ្ធផលរៀន** - អ្វីដែលអ្នកនឹងយល់ដឹងបន្ទាប់ពីបញ្ចប់

## 🛠️ របៀបប្រើផ្លូវនេះដោយប្រសិទ្ធភាព

### រៀនតាមលំដាប់ (ណែនាំ)

ធ្វើការងារតាមលាបជាលំដាប់ ដើម្បីយល់ដឹងពេញលេញ៖

1. **អានទិដ្ឋភាពទូលំទូលាយ** - យល់ថាអ្នកនឹងរៀនអ្វី
2. **ពិនិត្យលក្ខខណ្ឌមុន** - ធានាថាអ្នកមានចំណេះដឹងចាំបាច់
3. **តាមដានណែនាំជំហ៊ាន** - អនុវត្តពេលអ្នករៀន
4. **បញ្ចប់ហ្វឹកហាត់** - ថ្កោលទុកចិត្តការយល់ដឹង
5. **ពិនិត្យទស្សនៈសំខាន់ៗ** - ផ្ទៀងផ្ទាត់លទ្ធផលសិក្សា

### រៀនផ្តោតគោលដៅ

បើអ្នកត្រូវការជំនាញជាក់លាក់៖

- **បញ្ចូលមូលដ្ឋានទិន្នន័យ**: ផ្តោតលាប 04-06
- **អនុវត្តសុវត្ថិភាព**: ផ្តោតលាប 02, 08, 12
- **AI/ស្វែងរកអត្ថន័យ**: ជ្រាបលាប 07
- **ដាក់បញ្ចូលផលិតកម្ម**: សិក្សាលាប 10-12

### ហ្វឹកហាត់​ដៃ

លាបទាំងឡាយរួមមាន៖
- **គំរូកូដដំណើរការ** - ចម្លងកូដ, កែលម្អ និងសាកល្បង
- **ករណីជាក់ស្តែងពីជិត** - ករណីនៃវិភាគលក់
- **ភាពស្មុគស្មាញជាបន្តបន្ទាប់** - កសាងពីសាមញ្ញទៅខ្ពស់
- **ជំហានបញ្ជាក់** - បញ្ជាក់ការងារឲ្យបានត្រឹមត្រូវ
