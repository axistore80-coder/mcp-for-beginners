## ទិដ្ឋភាពទូទៅ

**Model Context Protocol (MCP)** អនុញ្ញាតឲ្យជំនួយការបញ្ញាសិប្បនិម្មិតចូលដំណើរការ និងអន្តរកម្មជាមួយប្រភពទិន្នន័យក្រៅបានយ៉ាងសុវត្ថិភាពក្នុងពេលវេលាច្រេីន។ ពេលរួមបញ្ចូលជាមួយការរួមបញ្ចូលទិន្នន័យ MCP សមត្ថភាពដ៏ខ្លាំងបើកចំហសម្រាប់កម្មវិធី AI ដោយផ្អែកលើទិន្នន័យ។

ផ្លូវការសិក្សានេះបង្រៀនអ្នករបៀបបង្កើតម៉ាស៊ីនមេ MCP ដែលមានសមត្ថភាពប្រើប្រាស់បានក្នុងការផលិត ដែលភ្ជាប់ជំនួយការបញ្ញាសិប្បនិម្មិតទៅទិន្នន័យលក់រាយតាម PostgreSQL ដោយអនុវត្តលំនាំជំនួញដូចជា Row Level Security, ស្វែងរកអត្ថន័យ និងការចូលដំណើរការទិន្នន័យពហុអ្នកជួល។

- **ការដោះស្រាយកំហុស**: ការលក់បញ្ហាដោយមេតិកាល ហើយចុះបញ្ជីកំហុស

- **ធុងការតភ្ជាប់**: ការគ្រប់គ្រងតភ្ជាប់ asyncpg ប្រសិទ្ធិភាព
- **អ្នកផ្គត់ផ្គង់ស្កីម៉ា**: ការស្វែងរកស្កីម៉ាតារាងដោយឡើងវិញដោយ δυναμικά
- **អ្នកអនុវត្តសំណួរ**: ការអនុវត្ត SQL សុវត្ថិភាពជាមួយបរិបទ RLS
- **ការគ្រប់គ្រងប្រតិបត្តិការ**: គោលការណ៍ ACID និងការដោះស្រាយបញ្ហាចុះវិញ

#### **3. ស្រទាប់សុវត្ថិភាព**
- **Row Level Security**: PostgreSQL RLS សម្រាប់ការផ្ទេរទិន្នន័យពហុអ្នកជួល
- **អត្តសញ្ញាណអ្នកប្រើប្រាស់**: ការផ្ទៀងផ្ទាត់អ្នកគ្រប់គ្រងហាង និងការអនុញ្ញាត
- **ការគ្រប់គ្រងចូលប្រើ**: សិទ្ធិជាក់លាក់ និងការត្រួតពិនិត្យប្រតិបត្តិការ
- **ការផ្ទៀងផ្ទាត់បញ្ចូល**: បញ្ឈប់ការបញ្ចូល SQL injection និងផ្ទៀងផ្ទាត់សំណួរ

#### **4. ស្រទាប់បន្ថែម AI**
- **ស្វែងរកអត្ថន័យ**: ការដាក់បញ្ចូលវ៉ិចទ័រសម្រាប់ស្វែងរកផលិតផល
- **អន្តរកម្ម Azure OpenAI**: ការបង្កើតបញ្ចូលអត្ថបទ
- **គណិតវិទ្យាចម្លើយស្រដៀង**: ស្វែងរកផ្នែកស្រដៀងគ្នា pgvector cosine similarity
- **បង្កើតផលិតភាពស្វែងរក**: ការបញ្ចូលសន្ទស្សន៍ និងកែប្រែប្រសិទ្ធភាព

## 🔒 សុវត្ថិភាព និងពហុភាពជួល បញ្ជ្រាបជ្រាលជ្រៅ

ការអនុវត្តរបស់យើងផ្តោតលើសុវត្ថិភាពតម្រង់តម្រង់កម្រិតសហគ្រាស៖

### សុវត្ថិភាពជួរដេក (RLS)

PostgreSQL RLS ធានាការផ្ទេរទិន្នន័យឯកលក្ខណៈ៖

```sql
-- Store managers see only their store's data
CREATE POLICY store_manager_policy ON retail.orders
  FOR ALL TO store_managers
  USING (store_id = get_current_user_store());

-- Regional managers see multiple stores
CREATE POLICY regional_manager_policy ON retail.orders
  FOR ALL TO regional_managers
  USING (store_id = ANY(get_user_store_list()));
```

### ការគ្រប់គ្រងអត្តសញ្ញាណអ្នកប្រើ

ការតភ្ជាប់ MCP នីមួយៗរួមបញ្ចូល៖  
- **លេខសម្គាល់អ្នកគ្រប់គ្រងហាង**: អត្តសញ្ញាណលម្អិតសម្រាប់បរិបទ RLS  
- **ការចាត់តួនាទី**: សិទ្ធិ និងកម្រិតចូលប្រើ  
- **ការគ្រប់គ្រងសម័យ**: សញ្ញាអត្តសញ្ញាណសុវត្ថិភាព  
- **ការចុះបញ្ជីតាមដាន**: ប្រវត្តិការចូលប្រើពេញលេញ

### ការការពារទិន្នន័យ

ស្រទាប់សុវត្ថិភាពច្រើន៖  
- **ការបញ្ចូលការតភ្ជាប់រួមបញ្ចូល**: TLS សម្រាប់គ្រប់ការតភ្ជាប់ទិន្នន័យ  
- **ការការពារចូលបញ្ចូល SQL injection**: សំណួរប៉ារ៉ាម៉ែត្រត្រឹមត្រូវតែប៉ុណ្ណោះ  
- **ការផ្ទៀងផ្ទាត់បញ្ចូល**: ផ្ទៀងផ្ទាត់សំណើពេញលេញ  
- **ការដោះស្រាយកំហុស**: មិនប្រើទិន្នន័យភាគីបីក្នុងសារខុស
