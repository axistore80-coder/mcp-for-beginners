ഈ ഡോക്യുമെന്റ് മോഡൽ കോൺടെക്സ് പ്രോട്ടോക്കോൾ (MCP) for Beginners പാഠ്യക്രമത്തിലെ പ്രധാന മാറ്റങ്ങൾ രേഖപ്പെടുത്തുന്നതിനുള്ളതാണ്. മാറ്റങ്ങൾ കാലക്രമത്തിൻറ വിരുദ്ധ顺序 (പുതിയമാറ്റങ്ങൾ ആദ്യത്തായി) രേഖപ്പെടുത്തിയിട്ടുണ്ട്.

- **പുതിയ ക്ലയന്റ് പ്രിമിറ്റീവ് - Roots**: റൂട്ട്സ് ക്ലയന്റ് പ്രിമിറ്റീവിന്റെ സമഗ്ര ഡോക്യുമെന്റ് ചേർത്തു, സർവറുകൾ ഫയൽസിസ്റ്റം അതിര്‍വരമ്പുകളും പ്രവേശന അവകാശങ്ങളും മനസ്സിലാക്കാൻ സഹായിക്കുന്നു  
- **ടൂൾ അനോട്ടേഷനുകൾ**: കൂടുതൽ മെച്ചപ്പെട്ട ടൂൾ നിർവഹണ തീരുമാനങ്ങൾക്ക് ടൂൾ പെരുമാറ്റാനോട്ടേഷനുകളെപ്പറ്റി (`readOnlyHint`, `destructiveHint`) ഡോക്യുമെന്റ് ചേർത്തു  
- **സാമ്പിളിംഗ്-ൽ ടൂൾ വിളിക്കൽ**: സാമ്പിളിംഗ് അഭ്യർത്ഥനകളിൽ മോഡൽ നയിക്കുന്ന ടൂൾ വിളിക്കലിനായി `tools` ആൻഡ് `toolChoice` പാരാമീറ്ററുകൾ ചേർന്നു  
- **URL മോഡ് പ്രചോദനം**: സെർവർ ആരംഭിക്കുന്ന എൻറിൻറൽ വെബ് ഇടപെടലുകൾക്കായുള്ള URL അടിസ്ഥാന elicitation ഡോക്യൂമെന്റ് ചേർത്തു  
- **ടാസ്കുകൾ (പരീക്ഷണ)**: ദീർഘനഷ്ടമായ നിർവഹണ റാപ്പേഴുകളും വbrookതിരുത്തൽ ഫല ശേഖരണവും പഠിപ്പിക്കുന്ന പരീക്ഷണ ടാസ്ക് ഫീച്ചർ ഡോക്യുമെന്റ് ചേർത്തു  
- **ഇക്കോണുകൾ പിന്തുണ**: ടൂളുകൾ, റിസോഴ്സുകൾ, റിസോഴ്സ് ടെംപ്ലേറ്റുകൾ, പ്രോംപ്റ്റുകൾ ഇനി അധിക മെറ്റാഡാറ്റയായി ഐക്കോണുകൾ ഉൾക്കൊള്ളാം  

#### ഡോക്യുമെന്റ് പുതുക്കലുകൾ

- **study_guide.md**: ടാസ്കുകളും ടൂൾ അനോട്ടേഷനുകളും Core Concepts സെക്ഷനിൽ ഉൾപ്പെടുത്തി; ഡോക്യുമെന്റ് ടൈംസ്റ്റാമ്പ് പുതുക്കി  

#### സ്‌പെസിഫിക്കേഷൻ അനുസരണം പരിശോധന  
- **പ്രോട്ടോക്കോൾ പതിപ്പ്**: എല്ലാ ഡോക്യുമെന്റ് రిఫറൻസുകളും MCP സ്‌പെസിഫിക്കേഷൻ 2025-11-25-നോട് പൊരുത്തപ്പെടുന്നു എന്ന് സ്ഥിരീകരിച്ചു  
- **ആർക്കിടെക്ചർ പൊരുത്തം**: രണ്ട് ലെയർ ആർക്കിടെക്ചർ (ഡാറ്റ ലെയർ + ട്രാൻസ്പോർട്ട് ലെയർ) ഡോക്യുമെന്റേഷൻ ശരിയാണ് എന്ന് സ്ഥിരീകരിച്ചു  
- **പ്രിമിറ്റീവുകൾ**: സർവർ പ്രിമിറ്റീവുകൾ (റിസോഴ്സുകൾ, പ്രോംപ്റ്റുകൾ, ടൂളുകൾ) ക്ലയന്റ് പ്രിമിറ്റീവുകൾ (സാമ്പിളിംഗ്, elicitation, ലോഗിംഗ്, റൂട്ട്സ്) പരിശോധന നടത്തി  
- **ട്രാൻസ്പോർട്ട് മാർഗങ്ങൾ**: STDIO, Streamable HTTP ട്രാൻസ്പോർട്ട് ഡോക്യുമെന്റേഷൻ കൃത്യമാണെന്ന് സ്ഥിരീകരിച്ചു  
- **സുരക്ഷാ നിർദ്ദേശങ്ങൾ**: MCP സുരക്ഷാ മികച്ച പരിപാലനവുമായി പൊരുത്തപ്പെടുന്നുവെന്ന് ഉറപ്പാക്കി  

#### MCP 2025-11-25 മുഖ്യ ഫീച്ചറുകൾ ഡോക്യുമെന്റ് ചെയ്തു

- **OAuth Client ID മെറ്റാഡാറ്റ ഡോക്യുമെന്റുകൾ**: ശുപാർശ ചെയ്ത ക്ലയന്റ് രജിസ്‌ട്രേഷൻ മാർഗം  
- **JSON Schema 2020-12**: MCP സ്കീമ നിർവചനങ്ങളിലെ ഡിഫോൾട്ട് ഡയലക്ട്  
- **SDK ടിയറിംഗ് സിസ്റ്റം**: SDK ഫീച്ചർ പിന്തുണക്കും പരിപാലനത്തിന്നുമുള്ള ഔദ്യോഗിക ആവശ്യങ്ങൾ  
- **ഭരണഘടന**: MCP ഗവൺസൻസിലുണ്ടായ വർകിങ് ഗ്രൂപ്പുകളും ഇന്ററെസ്റ്റ് ഗ്രൂപ്പുകളും ഔദ്യോഗികമായി നിലനിർത്തുന്നു

#### ഉയർന്ന നിരക്കിലുള്ള സെർവർ ഉപയോഗം (03-GettingStarted/10-advanced)
- **പുതിയ അധ്യായം ചേർത്തു**: സാധാരണ, ന്യുനപദസേനകളും ഉൾപ്പെടുന്ന ഉയർന്ന നിരക്കിലുള്ള MCP സെർവർ ഉപയോഗത്തിന് സമഗ്ര ഗൈഡ് അവതരിപ്പിച്ചു.
  - **സാധാരണ സെർവർ vs. ന്യുനപദ സെർവർ**: രണ്ട് സമീപനങ്ങൾക്കും Python, TypeScript കോഡ് ഉദാഹരണങ്ങളോടെ വിശദീകരണം.
  - **ഹാൻഡ്ലർ-അടിത്തറ ഡിസൈൻ**: സ്കെയിലബിള്‍, ഫ്ലെക്സിബിള്‍ സെർവർ നടപ്പാക്കലിനായി ഹാൻഡ്ലർ അടിസ്ഥാന ടൂൾ/രിസോഴ്സ്/പ്രോംപ്റ്റ് മാനേജ്മെന്റ് വിശദീകരണം.
  - **പ്രായോഗിക പാറ്റേണുകൾ**: ഉയർന്ന നിരക്കിലുള്ള ഫീച്ചറുകൾക്കും ആർക്കിടെക്ചർ ആവശ്യങ്ങൾക്കും സഹായകമായ സാൻദർഭിക പാറ്റേണുകൾ.
  
#### ലളിതമായ ഓതന്റിക്കേഷൻ (03-GettingStarted/11-simple-auth)
- **പുതിയ അധ്യായം ചേർത്തു**: MCP സർവറുകളിൽ ലളിതമായ ഓതന്റിക്കേഷൻ നടപ്പാക്കാനുള്ള സ്റ്റെപ്പ്-ബ്-സ്റ്റെപ്പ് ഗൈഡ്.
  - **ഓതന്റെ കോൺസെപ്റ്റുകൾ**: ഓതന്റിക്കേഷൻ, ഓതറൈസേഷൻ, ക്രെഡൻഷ്യൽ കൈകാര്യം എന്നിവയുടെ വ്യക്തമായ വിശദീകരണം.
  - **ബേസിക് ഓത് നടപ്പാക്കൽ**: Python (Starlette) & TypeScript (Express) മിഡിൽവെയർ-അടിസ്ഥാന ഓതന്റിക്കേഷൻ പാറ്റേണുകൾ, കോഡ് സാമ്പിളുകൾ ഉൾപ്പെടെ.
  - **ഉയർന്ന സുരക്ഷയിലേക്ക് പുരോഗതി**: ലളിതമായ ഓത്ത് തുടങ്ങികൊണ്ട് OAuth 2.1, RBAC എന്നിവയ്ക്കായി മെച്ചപ്പെട്ട സുരക്ഷാ മോഡ്യൂളുകളിൽ മാർഗനിർദ്ദേശം.
  
ഈ ചേർപ്പുകൾ MCP സർവറുകളെ കൂടുതൽ മുറുക്കിയ, സുരക്ഷിതവും ഫ്ലെക്സിബിൾ ആക്കാൻ പ്രായോഗിക, ഹാൻഡ്‌സോൺ മാർഗനിർദ്ദേശം നൽകുന്നു, അടിസ്ഥാന ആശയങ്ങളെയും ഉയർന്ന ഉത്പാദന പാറ്റേണുകളെയും സമീപിക്കുന്നു.

#### കോർ ആശയങ്ങൾ (01-CoreConcepts/) - പൂർണ്ണ ആധുനീകരണം  
- **പ്രോട്ടോകോൾ പതിപ്പ് അപ്‌ഡേറ്റ്**: MCP സ്പെസിഫിക്കേഷൻ 2025-06-18 എന്ന തീയതി അടിസ്ഥാനമാക്കി പതിപ്പ് കാണിച്ച് അപ്‌ഡേറ്റ് ചെയ്തു  
- **ആർക്കിടെക്ചർ പരിഷ്കരണം**: ഹൈസ്റ്റ്സ്, ക്ലയന്റുകൾ, സർവ്വറുകൾ എന്നിവയുടെ വിശദീകരണം MCP ആർക്കിടെക്ചർ മാതൃകാകൾക്ക് അനുയോജ്യമായി വികസിപ്പിച്ചു  
  - ഹൈസ്റ്റുകൾ ഇപ്പോൾ AI ആപ്ലിക്കേഷനുകളായി, ബഹുപദിച്ച് MCP ക്ലയന്റ് ബന്ധങ്ങൾ ഏകോപിപ്പിക്കുന്നു എന്ന് വ്യക്തമായി നിർവചിച്ചു  
  - ക്ലയന്റുകൾ പ്രോട്ടോകോൾ കണക്റ്ററുകളായി, ഓരോ സർവ്വറുമായി ഒരുമിച്ച് ബന്ധം നിലനിർത്തുന്നു എന്ന് വിശദീകരിച്ചു  
  - സർവ്വറുകൾ പ്രാദേശികവും ബഹിരാകാശവുമായ മൊത്തം വിന്യാസ സാഹചര്യങ്ങൾ ഉൾപ്പെടുത്തി വിപുലീകരിച്ചു  
- **പ്രിമിറ്റീവ് പുനഃസംഘടന**: സർവ്വർ, ക്ലയന്റ് പ്രിമിറ്റീവുകളുടെ പൂർണ്ണ പരിഷ്കരണം  
  - സർവ്വർ പ്രിമിറ്റീവുകൾ: സ്രോതസ്സ് (ഡാറ്റാ), പ്രോംപ്റ്റുകൾ (ടെംപ്ലേറ്റുകൾ), ടൂളുകൾ (നിർവഹണ ഫംഗ്ഷനുകൾ) വിശദീകരണത്തോടും ഉദാഹരണങ്ങളുമായുള്ള  
  - ക്ലയന്റ് പ്രിമിറ്റീവുകൾ: സാംപ്ലിംഗ് (LLM പൂർത്തീകരണങ്ങൾ), എലിസിറ്റേഷൻ (ഉപയോക്തൃ ഇൻപുട്ട്), ലോപ്പിങ്ങ് (ഡീബഗ്/നിരീക്ഷണം)  
  - നിലവിലുള്ള ഡിസ്കവർ (`*/list`), ലഭിക്കുക (`*/get`), നിർവഹിക്കുക (`*/call`) രീതി മാതൃകകൾക്ക് അനുയോജ്യമായി അപ്‌ഡേറ്റ് ചെയ്തിരിക്കുന്നു  
- **പ്രോട്ടോകോൾ ആർക്കിടെക്ചർ**: രണ്ട്-പാളി ആർക്കിടെക്ചർ മാതൃക അവതരിപ്പിച്ചു  
  - ഡാറ്റാ പാളി: JSON-RPC 2.0 അടിസ്ഥാനത്തോടെ ജീവിതചക്രം നിയന്ത്രണവും പ്രിമിറ്റീവുകളും  
  - ട്രാൻസ്പോർട്ട് പാളി: STDIO (പ്രാദേശിക) ഒപ്പം Streamable HTTP എന്ന SSE (ബഹിരാകാശം) ട്രാൻസ്പോർട്ട് രീതികൾ  
- **സുരക്ഷാ ഫ്രെയിംവർക്ക്**: വ്യക്തമായ ഉപയോക്തൃ സമ്മതം, ഡാറ്റാ ഗോപനീയത സംരക്ഷണം, ടൂൾ നിർവഹണ സുരക്ഷ, ട്രാൻസ്പോർട്ട് പാളി സുരക്ഷ തുടങ്ങിയ സമഗ്ര സുരക്ഷാ സിദ്ധാന്തങ്ങൾ  
- **സംവാദ മാതൃകകൾ**: പ്രോട്ടോകോൾ സന്ദേശങ്ങൾ നിർവചനശേഷം, കണ്ടെത്തൽ, നിർവഹണം, അറിയിപ്പ് പ്രവാഹം എന്നിവ കാണിക്കുന്നു  
- **കോഡ് ഉദാഹരണങ്ങൾ**: MCP SDK മാതൃകകൾക്ക് അനുയോജ്യമായി .NET, ജാവ, പൈതൺ, ജാവാസ്ക്രിപ്റ്റ് ഭാഷകളിൽ നസംസ്കരിച്ച നിരവധി ഭാഷാ ഉദാഹരണങ്ങൾ  

#### സുരക്ഷ (02-Security/) - സമഗ്ര സുരക്ഷാ പുനഃസംഘടന  
- **സ്റ്റാൻഡേർഡുകൾക്ക് യോജിപ്പ്**: MCP സ്പെസിഫിക്കേഷൻ 2025-06-18 സുരക്ഷാ ആവശ്യകതകളുമായി പൂർണ്ണമായ അന്തർഗതമാക്കി

- **AI-സവിശേഷ ഭീഷണി വിശകലനം**: ആധുനിക AI ആക്രമണ മാർഗങ്ങൾ വിശദീകരിച്ചു  
  - prompt injection ആക്രമണത്തിന് പ്രത്യേകമായ യഥാർത്ഥ ലോക ഉദാഹരണങ്ങൾ  
  - ടൂൾ വിഷബാധ കാണിക്കുന്ന രീതികളും "രഗ് പുൾ" ആക്രമണ മാതൃകകളും  
  - context window വിഷബാധയും മോഡൽ ആശ്രയക്കേട് ആക്രമണങ്ങളും  
- **Microsoft AI സുരക്ഷാ പരിഹാരങ്ങൾ**: Microsoft സുരക്ഷാ പരിസ്ഥിതിയുടെ സമഗ്ര പരിചയം  
  - AI Prompt Shields: പ്രഗത്ഭ കണ്ടെത്തലുകൾ, സ്പോട്ട്‌ലൈറ്റിംഗ്, ഡെലിമിറ്റർ സാങ്കേതികതകൾ  
  - Azure Content Safety ഇന്റഗ്രേഷൻ മാതൃകകൾ  
  - GitHub Advanced Security വഴി സമ്പ്രദായ ശൃംഖല സംരക്ഷണം  
- **പ്രഗത്ഭ ഭീഷണി നിയന്ത്രണം**:  
  - MCP-സവിശേഷ സെഷൻ ഹাইজാക്കിങ് ആക്രമണ മാതൃകകൾ, ക്രിപ്‌ടോഗ്രാഫിക് സെഷൻ ഐഡി ആവശ്യകതകളോടും  
  - MCP പ്രോക്സി സാഹചര്യങ്ങളിലുള്ള Confused Deputy പ്രശ്നങ്ങൾ വ്യക്തമായ സമ്മതം ആവശ്യകതയോടും  
  - ടോക്കൺ പാസ്ത്രൂ ദുർബലതകൾ നിർബന്ധിത പരിശോധന നിയന്ത്രണങ്ങളോടും  
- **സമ്പ്രദായ ശൃംഖല സുരക്ഷ**: ഫൗണ്ടേഷൻ മോഡലുകൾ, ایم്ബഡ്ഡിംഗ്സ് സേവനങ്ങൾ, context പ്രൊവൈഡറുകൾ, മൂന്നാംകക്ഷി API കളുടെ ഉൾപ്പെടെയുള്ള AI സമ്പ്രദായ ശൃംഖല പരിരക്ഷണം വ്യാപിച്ചു  
- **ഫൗണ്ടേഷൻ സുരക്ഷ**: സൂന്യ വിശ്വാസ ആർക്കിടെക്ചറികളും Microsoft സുരക്ഷാ പരിസ്ഥിതിയുമായുള്ള സംയോജനം മെച്ചപ്പെടുത്തി  
- **സ്രോതസ്സ് ക്രമീകരണം**: ഔദ്യോഗിക ഡോക്സുകൾ, സ്റ്റാൻഡേർഡുകൾ, ഗവേഷണം, Microsoft പരിഹാരങ്ങൾ, നടപ്പാക്കൽ മാർഗനിർദ്ദേശങ്ങൾ എന്നിങ്ങനെ വിഭാഗീകരിച്ച് ലിങ്കുകൾ കൂട്ടിച്ചേർത്തു

- MCP ക്ലയന്റുകളും ടൂളുകളുമായി ബന്ധപ്പെട്ട പുതിയ ഭാഗങ്ങൾ ചേർത്തു, ജനപ്രിയ MCP സർവറുകൾ കൂടി  
- Visual Curriculum Map ആക്കം മുഴുവനായി അപ്ഡേറ്റ് ചെയ്തു  
- പ്രഗത്ഭ വിഷയങ്ങളുടെ വിവരണങ്ങൾ കൂടുതൽ പ്രത്യേകഭാഗങ്ങൾ ഉൾപ്പെടുത്തി മെച്ചപ്പെടുത്തി  
- യാഥാർത്ഥ്യ ഉദാഹരണങ്ങൾ പ്രതിഫലിപ്പിക്കുന്ന കേസ് സ്റ്റഡീസുകൾ അപ്ഡേറ്റ് ചെയ്തു  
- ഈ സമഗ്രമായ ചേഞ്ച്‌ലോഗ് ചേർത്തു  

### കമ്മ്യൂണിറ്റി സംഭാവനകൾ (06-CommunityContributions/)  
- ചിത്രം സൃഷ്‌ടിക്കുന്ന MCP സർവറുകളെക്കുറിച്ച് വിശദമായ വിവരം ചേർത്തു  
- VSCode-ൽ Claude ഉപയോഗിക്കുന്ന സമഗ്ര വിഭാഗം ചേർത്തു  
- Cline ടെർമിനൽ ക്ലയന്റ് സജ്ജീകരണവും ഉപയോഗ നിർദ്ദേശങ്ങളുമടങ്ങിയവ ചേർത്തു  
- MCP ക്ലയന്റ് വിഭാഗം എല്ലാ ജനപ്രിയ ക്ലയന്റ് ഓപ്ഷനുകളോടും പുതുക്കി  
- കൂടുതൽ ക്രമീകരിച്ചിട്ടുള്ള മെച്ചപ്പെട്ട സംഭാവനാ ഉദാഹരണങ്ങൾ ചേർത്തു  

### പ്രഗത്ഭ വിഷയങ്ങൾ (05-AdvancedTopics/)  
- എല്ലാ പ്രത്യേകവിഷയ പോളഡറുകളും സുസ്ഥിര നാമകരണം ഉപയോഗിച്ച് ക്രമീകരിച്ചു  
- കോൺടെക്സ്റ്റ് എഞ്ചിനീയറിംഗ് സാമഗ്രികളും ഉദാഹരണങ്ങളും ചേർത്തു  
- Foundry ഏജന്റ് ഇന്റഗ്രേഷൻ ഡോക്യുമെന്റേഷൻ ചേർത്തു  
- Entra ID സുരക്ഷാ സംയോജനം ഡോക്യുമെന്റേഷൻ മെച്ചപ്പെടുത്തി

### തുടങ്ങൽ (03-GettingStarted/)  
- ആദ്യ സർവർ നടപ്പാക്കൽ ഉദാഹരണങ്ങൾ സൃഷ്ടിച്ചു  
- ക്ലയന്റ് വികസന മാർഗ നിർദ്ദേശങ്ങൾ ചേർത്തു  
- LLM ക്ലയന്റ് ഇന്റഗ്രേഷൻ നിർദ്ദേശങ്ങൾ ഉൾപ്പെടുത്തി  
- VS Code ഇന്റഗ്രേഷൻ ഡോക്യുമെന്റേഷൻ ചേർത്തു  
- Server-Sent Events (SSE) സർവർ ഉദാഹരണങ്ങൾ നടപ്പാക്കി

- MCP സന്ദേശോപാധി മാതൃകകൾ രേഖപ്പെടുത്തി
