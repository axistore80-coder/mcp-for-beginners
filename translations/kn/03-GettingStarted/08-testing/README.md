## ಪರೀಕ್ಷೆ ಮತ್ತು ಡಿಬಗಿಂಗ್

ನಿಮ್ಮ MCP ಸರ್ವರ್ ಅನ್ನು ಪರೀಕ್ಷೆ ಮಾಡುವ ಮೊದಲು, ಲಭ್ಯವಿರುವ ಸಾಧನಗಳು ಮತ್ತು ಡಿಬಗ್ ಮಾಡುವ ಶ್ರೇಷ್ಠ ಪದ್ಧತಿಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು ಮುಖ್ಯ. ಪರಿಣಾಮಕಾರಿ ಪರೀಕ್ಷೆ ನಿಮ್ಮ ಸರ್ವರ್ ನಿರೀಕ್ಷೆಯಂತೆ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತಿರುವುದನ್ನು ಖಚಿತಪಡಿಸುತ್ತದೆ ಮತ್ತು ಸಮಸ್ಯೆಗಳನ್ನು ತ್ವರಿತವಾಗಿ ಗುರುತಿಸಿ ಪರಿಹರಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ. ಕೆಳಗಿನ ವಿಭಾಗದಲ್ಲಿ ನಿಮ್ಮ MCP ಅನುಷ್ಠಾನವನ್ನು ಪರಿಶೀಲಿಸಲು ಶಿಫಾರಸು ಮಾಡಲಾದ ವಿಧಾನಗಳನ್ನು ವಿವರಿಸಲಾಗಿದೆ.

## ಅವಲೋಕನ

ಈ ಪಾಠವು ಸರಿಯಾದ ಪರೀಕ್ಷಾ ವಿಧಾನವನ್ನು ಆಯ್ಕೆಮಾಡುವುದು ಮತ್ತು ಪರಿಣಾಮಕಾರಿಯಾದ ಪರೀಕ್ಷಾ ಸಾಧನವನ್ನು ಕುರಿತಂತೆ ಒಳಗೊಂಡಿದೆ.

## ಕಲಿಕೆ ಗುರಿಗಳು

ಈ ಪಾಠದ ಅಂತ್ಯಕ್ಕೆ, ನೀವು ಸಾಧ್ಯವಾಗುತ್ತೀರಿ:

- ವಿಸ್ತೃತ ಪರೀಕ್ಷಾ ವಿಧಾನಗಳನ್ನು ವರ್ಣಿಸಲು.
- ವಿಭಿನ್ನ ಉಪಕರಣಗಳನ್ನು ಬಳಸಿ ನಿಮ್ಮ ಕೋಡ್ ಅನ್ನು ಪರಿಣಾಮಕಾರಿಯಾಗಿ ಪರೀಕ್ಷಿಸಲು.


## MCP ಸರ್ವರ್‌ಗಳನ್ನು ಪರೀಕ್ಷಿಸುವಿಕೆ

MCP ನಿಮ್ಮ ಸರ್ವರ್ಗಳನ್ನು ಪರೀಕ್ಷಿಸಲು ಮತ್ತು ಡಿಬಗ್ ಮಾಡಲು ಉಪಕರಣಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ:

- **MCP ಇನ್ಸ್‌ಪೆಕ್ಟರ್**: CLI ಸಾಧನವಾಗಿಯೂ ಮತ್ತು ದೃಶ್ಯ ಸಾಧನವಾಗಿಯೂ ಚಾಲನೆ ಮಾಡಬಹುದಾದ ಕಮಾಂಡ್ ಲೈನ್ ಟೂಲ್.
- **ಮ್ಯಾನುಯಲ್ ಪರೀಕ್ಷೆ**: curl ನಂತಹ ಸಾಧನ ಬಳಸಿಕೊಂಡು ವೆಬ್ ವಿನಂತಿಗಳನ್ನು ಚಾಲನೆ ಮಾಡಬಹುದು, ಆದರೆ ಯಾವುದಾದರೂ HTTPಚಾಲನೆ ಸಾಧ್ಯವಿರುವ ಸಾಧನ ಉಪಯೋಗಿಸಬಹುದು.
- **ಯುನಿಟ್ ಪರೀಕ್ಷೆ**: ಸರ್ವರ್ ಮತ್ತು ಕ್ಲೈಂಟ್ ವೈಶಿಷ್ಟ್ಯಗಳನ್ನು ಪರೀಕ್ಷಿಸಲು ನಿಮ್ಮ ಇಚ್ಛಿತ ಪರೀಕ್ಷಾ ಫ್ರೇಮ್ವರ್ಕ್ ಅನ್ನು ಬಳಸಬಹುದು.

### MCP ಇನ್ಸ್‌ಪೆಕ್ಟರ್ ಬಳಕೆ

ಈ ಸಾಧನದ ಬಳಕೆ ನಾವು ಹಿಂದಿನ ಪಾಠಗಳಲ್ಲಿ ವಿವರಿಸಿದ್ದೇವೆ, ಆದರೆ ಇಲ್ಲಿ ಅದನ್ನು ಸ್ವಲ್ಪ ಮಟ್ಟಿನಲ್ಲಿ ಚರ್ಚಿಸೋಣ. ಇದು Node.js ನಲ್ಲಿ ನಿರ್ಮಿತ ಸಾಧನ ಮತ್ತು ನೀವು `npx` ಸ್ವಚ್ಛ ಚಲಾಯಿಸಿದರುಕ (executable) ಅನ್ನು ಕರೆಸಿ ಬಳಸಬಹುದು, ಅದು ತಾತ್ಕಾಲಿಕವಾಗಿ ಸಾಧನವನ್ನು ಡೌನ್‌ಲೋಡ್ ಮತ್ತು ಇನ್‌ಸ್ಟಾಲ್ ಮಾಡಿ ನಿಮ್ಮ ವಿನಂತಿಯನ್ನು ಚಲಾಯಿಸಿದ ಬಳಿಕ ಸ್ವತಃ ಸ್ವಚ್ಛಗೊಳಿಸಲಿದೆ.

[MCP ಇನ್ಸ್‌ಪೆಕ್ಟರ್](https://github.com/modelcontextprotocol/inspector) ನಿಮಗೆ ಸಹಾಯ ಮಾಡುತ್ತದೆ:

- **ಸರ್ವರ್ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಕಂಡುಹಿಡಿದುಕೊಳ್ಳುವುದು**: ಲಭ್ಯವಿರುವ ಸಂಪನ್ಮೂಲಗಳು, ಉಪಕರಣಗಳು ಮತ್ತು ಪ್ರಾಂಪ್ಟ್ಗಳನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಪತ್ತೆಹಚ್ಚಿ
- **ಪರೀಕ್ಷಾ ಉಪಕರಣ ಕಾರ್ಯನಿರ್ವಹಣೆ**: ವಿಭಿನ್ನ ಪರಿಮಾಣಗಳನ್ನು ಪ್ರಯತ್ನಿಸಿ ಮತ್ತು ನೈಜ-ಸಮಯ ಪ್ರತಿಕ್ರಿಯೆಗಳನ್ನು ನೋಡಿ
- **ಸರ್ವರ್ ಮೆಟಾಡೇಟಾವನ್ನು ವೀಕ್ಷಿಸು**: ಸರ್ವರ್ ಮಾಹಿತಿ, ಸ್ಕೀಮಾಗಳು ಮತ್ತು ಸಂರಚನೆಗಳನ್ನು ಪರಿಶೀಲಿಸಿ

ಈ ಸಾಧನದ ಸಾಮಾನ್ಯ ಚಾಲನೆ ಹೀಗೆ ಕಾಣಿಸುತ್ತుంది:

```bash
npx @modelcontextprotocol/inspector node build/index.js
```

ಮೇಲಿನ ಆಜ್ಞೆಯು ఒక MCP ಮತ್ತು ಅದರ ದೃಶ್ಯ ಇಂಟರ್ಫೇಸ್ ಅನ್ನು ಪ್ರಾರಂಭಿಸಿ ನಿಮ್ಮ ಬ್ರೌಸರ್ನಲ್ಲಿ ಸ್ಥಳೀಯ ವೆಬ್ ಇಂಟರ್ಫೇಸನ್ನು ಚಾಲನೆ ಮಾಡುತ್ತದೆ. ನೀವು ನೋಡಿಯಲ್ಲದೆ ನಿಮ್ಮ ನೋಂದಾಯಿತ MCP ಸರ್ವರ್‌ಗಳು, ಅವುಗಳ ಲಭ್ಯವಿರುವ ಉಪಕರಣಗಳು, ಸಂಪನ್ಮೂಲಗಳು ಮತ್ತು ಪ್ರಾಂಪ್ಟ್ಗಳ ಡ್ಯಾಶ್ಬೋರ್ಡ್‌ ಅನ್ನು ಕಾಣಬಹುದು. ಈ ಇಂಟರ್ಫೇಸ್ ನಿಮಗೆ ಪರಸ್ಪರ ಸಂವಾದಾತ್ಮಕವಾಗಿ ಉಪಕರಣದ ಕಾರ್ಯಾಚರಣೆ ಪರಿಶೀಲನೆ, ಸರ್ವರ್ ಮೆಟಾಡೇಟಾ ಪರಿಶೀಲನೆ, ಮತ್ತು ನೈಜ-ಸಮಯ ಪ್ರತಿಕ್ರಿಯೆಗಳ ವೀಕ್ಷಣೆಯನ್ನು ಸುಗಮಗೊಳಿಸುತ್ತದೆ, ಇದರಿಂದ ನಿಮ್ಮ MCP ಸರ್ವರ್ ಅನುಷ್ಠಾನಗಳನ್ನು ದೃಢೀಕರಿಸಲು ಮತ್ತು ಡಿಬಗ್ ಮಾಡಲು ಸಹಾಯವಾಗುತ್ತದೆ.

ಇದು ಹೀಗೆ ಕಾಣಬಹುದು: ![Inspector](../../../../translated_images/kn/connect.141db0b2bd05f096.webp)

ನೀವು ಈ ಸಾಧನವನ್ನು CLI ಮೋಡ್‌ನಲ್ಲಿ ಕೂಡ ಚಾಲನೆ ಮಾಡಬಹುದು, ಆ ಸಂದರ್ಭದಲ್ಲಿ ನೀವು `--cli` ಗುಣಲಕ್ಷಣವನ್ನು ಸೇರಿಸುತ್ತೀರಿ. "CLI" ಮೋಡ್‌ನಲ್ಲಿ ಸಾಧನವನ್ನು ಚಾಲನೆ ಮಾಡುವ ಉದಾಹರಣೆ ಇದಾಗಿದೆ, ಇದರಿಂದ ಸರ್ವರ್‌ನ ಎಲ್ಲಾ ಉಪಕರಣಗಳ ಪಟ್ಟಿಯನ್ನು ಕಾಣಬಹುದು:

```sh
npx @modelcontextprotocol/inspector --cli node build/index.js --method tools/list
```

### ಮ್ಯಾನುಯಲ್ ಪರೀಕ್ಷೆ

ಸರ್ವರ್ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಪರೀಕ್ಷಿಸಲು ಇನ್ಸ್‌ಪೆಕ್ಟರ್ ಸಾಧನವನ್ನು ನಡೆಸುವುದರ ಹೊರಗಿನ ಮತ್ತೊಂದು ಸಮಾನ ವಿಧಾನವೆಂದರೆ HTTP ಬಳಕೆ ಮಾಡಬಹುದಾದ ಕ್ಲೈಂಟ್, ಉದಾಹರಣೆಗೆ curl ಅನ್ನು ಬಳಸುವದು.

curl ಮೂಲಕ, ನೀವು MCP ಸರ್ವರ್‌ಗಳನ್ನು ನೇರವಾಗಿ HTTP ವಿನಂತಿಗಳ ಮೂಲಕ ಪರೀಕ್ಷಿಸಬಹುದು:

```bash
# ಉದಾಹರಣೆ: ಪರೀಕ್ಷಾ ಸರ್ವರ್ ಮೆಟಾಡೇಟಾ
curl http://localhost:3000/v1/metadata

# ಉದಾಹರಣೆ: ಒಂದು ಉಪಕರಣವನ್ನು ಕಾರ್ಯಗತಗೊಳಿಸಿ
curl -X POST http://localhost:3000/v1/tools/execute \
  -H "Content-Type: application/json" \
  -d '{"name": "calculator", "parameters": {"expression": "2+2"}}'
```

ಮೇಲಿನ curl ಬಳಕೆಯಿಂದ ನೀವು ನೋಡಬಹುದು, ನೀವು ಪೋಸ್ಟ್ ವಿನಂತಿಯನ್ನು ಉಪಕರಣವನ್ನು ಕರೆಯಲು ಬಳಕೆಮಾಡುತ್ತೀರಿ, ಇದರ ಪೇಲೋಡ್‌ನಲ್ಲಿ ಉಪಕರಣದ ಹೆಸರು ಮತ್ತು ಅದರ ಪರಿಮಾಣಗಳಿವೆ. ನಿಮ್ಮ ಅಗತ್ಯಕ್ಕೆ ತಕ್ಕಂತಹ ವಿಧಾನವನ್ನು ಉಪಯೋಗಿಸಿ. CLI ಉಪಕರಣಗಳು ಸಾಮಾನ್ಯವಾಗಿ ವೇಗವಾಗಿ ಬಳಸಬಹುದಾಗಿವೆ ಮತ್ತು ಅವುಗಳಿಗೆ ಸ್ಕ್ರಿಪ್ಟಿಂಗ್ ಅನುವಯಿಸುತ್ತದೆ, ಇದು CI/CD ಪರಿಸರದಲ್ಲಿ ಉಪಯುಕ್ತವಾಗಬಹುದು.

### ಯುನಿಟ್ ಪರೀಕ್ಷೆ

ನಿಮ್ಮ ಉಪಕರಣಗಳು ಮತ್ತು ಸಂಪನ್ಮೂಲಗಳು ನಿರೀಕ್ಷೆಯಂತೆ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತವೆ ಎಂದು ಖಚಿತಪಡಿಸಲು ಯುನಿಟ್ ಪರೀಕ್ಷೆಗಳನ್ನು ರಚಿಸಿ. ಕೆಳಗಿನ ಉದಾಹರಣೆಯಾಗಿ ಪರೀಕ್ಷಾ ಕೋಡ್ ನೀಡಲಾಗಿದೆ.

```python
import pytest

from mcp.server.fastmcp import FastMCP
from mcp.shared.memory import (
    create_connected_server_and_client_session as create_session,
)

# ಮಡ್ಯೂಲ್ ಅನ್ನು ಪೂರ್ಣವಾಗಿ ಅಸಿಂಕ್ರೋನ್ ಪರೀಕ್ಷೆಗಾಗಿ ಗುರುತಿಸಿ
pytestmark = pytest.mark.anyio


async def test_list_tools_cursor_parameter():
    """Test that the cursor parameter is accepted for list_tools.

    Note: FastMCP doesn't currently implement pagination, so this test
    only verifies that the cursor parameter is accepted by the client.
    """

 server = FastMCP("test")

    # ಕೆಲವು ಪರೀಕ್ಷಾ ಸಾಧನಗಳನ್ನು ರಚಿಸಿ
    @server.tool(name="test_tool_1")
    async def test_tool_1() -> str:
        """First test tool"""
        return "Result 1"

    @server.tool(name="test_tool_2")
    async def test_tool_2() -> str:
        """Second test tool"""
        return "Result 2"

    async with create_session(server._mcp_server) as client_session:
        # ಕರ್ಸರ್ ಪ್ಯಾರಾಮೀಟರ್ ಇಲ್ಲದೆ ಪರೀಕ್ಷೆ ಮಾಡಿ (ಬಿಟ್ಟಿಡಲಾಗಿದೆ)
        result1 = await client_session.list_tools()
        assert len(result1.tools) == 2

        # ಕರ್ಸರ್=None ಜೊತೆ ಪರೀಕ್ಷೆ ಮಾಡಿ
        result2 = await client_session.list_tools(cursor=None)
        assert len(result2.tools) == 2

        # ಕರ್ಸರ್ ಅನ್ನು ಸ್ಟ್ರಿಂಗ್ ಆಗಿ ಪರೀಕ್ಷೆ ಮಾಡಿ
        result3 = await client_session.list_tools(cursor="some_cursor_value")
        assert len(result3.tools) == 2

        # ಖಾಲಿ ಸ್ಟ್ರಿಂಗ್ ಕರ್ಸರ್ ಜೊತೆಗೆ ಪರೀಕ್ಷೆ ಮಾಡಿ
        result4 = await client_session.list_tools(cursor="")
        assert len(result4.tools) == 2
    
```

ಕೆಳಗಿನ ಕೋಡ್ ಹೀಗೆ ಮಾಡುತ್ತದೆ:

- pytest ಫ್ರೇಮ್ವರ್ಕ್ ಬಳಸಿದೆ, ಇದು ನಿಮಗೆ ಫಂಕ್ಷನ್‌ಗಳಾಗಿ ಪರೀಕ್ಷೆಗಳನ್ನು ರಚಿಸಲು ಹಾಗೂ assert ಸೂಚನೆಯನ್ನು ಉಪಯೋಗಿಸಲು ಅನುಮತಿಸುತ್ತದೆ.
- ಎರಡು ವಿಭಿನ್ನ ಉಪಕರಣಗಳನ್ನು ಹೊಂದಿರುವ MCP ಸರ್ವರ್ ಅನ್ನು ರಚಿಸುತ್ತದೆ.
- ಕೆಲವು ಶರತ್ತುಗಳು ಪೂರೈಸಲ್ಪಟ್ಟಿರುವುದನ್ನು ಖಚಿತಪಡಿಸಲು `assert` ಸೂಚನೆಯನ್ನು ಬಳಸುತ್ತದೆ.

ಇದಲ್ಲದೆ, [ಇಲ್ಲಿ ಸಂಪೂರ್ಣ ಫೈಲ್ ನೋಡಿ](https://github.com/modelcontextprotocol/python-sdk/blob/main/tests/client/test_list_methods_cursor.py)

ಮೇಲಿನ ಫೈಲ್ ಆಧಾರದಿಂದ, ನೀವು ನಿಮ್ಮ ಸ್ವಂತ ಸರ್ವರ್ ಅನ್ನು ಪರೀಕ್ಷಿಸಿ ಸಾಮರ್ಥ್ಯಗಳು ಸೃಷ್ಟಿಯಾಗಿರುವುದನ್ನು ಪರಿಶೀಲಿಸಬಹುದು.

ಎಲ್ಲಾ ಪ್ರಮುಖ SDKಗಳಿಗೆ ಸಮಾನ ಪರೀಕ್ಷಾ ವಿಭಾಗಗಳಿವೆ ಹಾಗಾಗಿ ನೀವು ನಿಮ್ಮ ಆರಿಸಿಕೊಂಡ ರನ್ ಟೈಮ್‌ಗೆ ಹೊಂದಿಕೊಳ್ಳಬಹುದು.
