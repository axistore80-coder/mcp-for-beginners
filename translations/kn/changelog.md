# ಚೇಂಜ್‌ಲॉग್: MCP ಆರಂಭಿಕರಿಗಾಗಿ ಪಠ್ಯಕ್ರಮ

ಈ ದಾಖಲೆವು ಮಾದರಿ ವಿಷಯ ಪ್ರೋಟೋಕಾಲ್ (MCP) ಆರಂಭಿಕರಿಗಾಗಿ ಪಠ್ಯಕ్రమಕ್ಕೆ ಮಾಡಿದ ಪ್ರಮುಖ ಬದಲಾವಣೆಗಳ ದಾಖಲೆಯಾಗಿದೆ. ಬದಲಾವಣೆಗಳು ಕಾಲಕ್ರಮದಲ್ಲಿ ತಿದ್ದುಪಡಿ (ಹೊಸ ಬದಲಾವಣೆಗಳು ಮೊದಲು) ದಾಖಲಿಸಲಾಗಿವೆ.

## ಏಪ್ರಿಲ್ 11, 2026

### ಹೊಸ ಪಾಠ, ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ತಿದ್ದುಪಡಿ ಮತ್ತು ಅವಲಂಬನೆ ನವೀಕರಣಗಳು

#### ಹೊಸ ಪಠ್ಯಕ್ರಮ ವಿಷಯವನ್ನು ಸೇರಿಸಲಾಗಿದೆ

**ಮಾಡ್ಯೂಲ್ 05 - ಆಧುನಿಕ ವಿಷಯಗಳು**
- **ಪಾಠ 5.17: MCP ನೊಂದಿಗೆ ವಿರೋಧಾತ್ಮಕ ಬಹು-ಏಜೆಂಟ್ ತರ್ಕ** (`05-AdvancedTopics/mcp-adversarial-agents/README.md`): ಬಹು-ಏಜೆಂಟ್ ವ್ಯವಸ್ಥೆಗಳು ಒಳಗೊಂಡ ವಿರೋಧಾತ್ಮಕ ಚರ್ಚಾ ಮಾದರಿಯನ್ನು ಸಮಗ್ರವಾಗಿ ವಿವರಿಸುವ ಹೊಸ ಮಾರ್ಗದರ್ಶಿ
  - ಮರಮೇಡ್ ವಾಸ್ತುಶಿಲ್ಪ ರೇಖಾಚಿತ್ರ: ಎರಡು ಏಜೆಂಟ್‌ಗಳು → ಹಂಚಿಕೆಯ MCP ಸರ್ವರ್ → ಚರ್ಚಾ ಲಿಖಿತ → ತೀರ್ಪುಕಾರ → ತೀರ್ಪು
  - Python ಮತ್ತು TypeScript ನಲ್ಲಿ ಅನುಷ್ಠಾನಗೊಳಿಸಿದ ಹಂಚಿಕೆಯ MCP ಉಪಕರಣ ಸರ್ವರ್ (`web_search` + `run_python`)
  - ವಿರುದ್ಧ ವ್ಯವಸ್ಥೆ ಪ್ರಾಂಪ್ಟ್‌ಗಳು (FOR / AGAINST / ತೀರ್ಪುಕಾರ) ಸ್ಪಷ್ಟ ಉಪಕರಣ ಬಳಕೆಯ ಅಗತ್ಯತೆಗಳೊಂದಿಗೆ
  - Python, TypeScript ಮತ್ತು C# ನಲ್ಲಿ ತರ್ಕ ಸಂಚಾಲಕ, ಸುತ್ತುಗಳನ್ನು ನಿರ್ವಹಿಸುವ ಮತ್ತು ವಾದಗಳನ್ನು ಮಾರ್ಗದರ್ಶಿಸುವ ಕಾರ್ಯ
  - ತರ್ಕ ಸಂಚಾಲಕಕ್ಕೆ MCP `ClientSession` ಮಡಗಿಸಿಕೊಂಡು ಪ್ರಸಕ್ತ ಉಪಕರಣ ಕರೆಗಳಿಗೆ ಸಂಪರ್ಕ
  - ಉಪಯೋಗದ ಉದಾಹರಣೆಪಟ್ಟಿ (ಹಲುಸಿನೇಶನ್ ಪತ್ತೆ, ಬೆದರಿಕೆಯ ಮಾದರೀಕರಣ, API ವಿನ್ಯಾಸ ಪರಿಶೀಲನೆ, ನಿಜಸ್ವರೂಪ ಪರಿಶೀಲನೆ, ತಂತ್ರಜ್ಞಾನ ಆಯ್ಕೆ)
  - ಭದ್ರತಾ ಪರಿಗಣನೆಗಳು: ಸ್ಯಾಂಡ್‌ಬಾಕ್ಸ್ ನಿರ್ವಹಣೆ, ಉಪಕರಣ-ಕರೆ ಮಾನ್ಯತೆ, ದರ ನಿಯಂತ್ರಣ, ಲಾಗಿಂಗ್ ಪರಿಶೀಲನೆ
  - ಮೂರು ವೈಯಕ್ತಿಕ ಸಂದರ್ಭದಲ್ಲಿ ರಚಿಸಿದ ವ್ಯಾಯಾಮ (ಕೋಡ್ ಸಮೀಕ್ಷೆ, ವಾಸ್ತುಶಿಲ್ಪ ನಿರ್ಣಯ, ವಿಷಯ ನಿಯಂತ್ರಣ)

#### ಕೋಡ್ ಮತ್ತು ಭದ್ರತಾ ತಿದ್ದುಪಡಿ

**ಮಾಡ್ಯೂಲ್ 05 - ವಿರೋಧಾತ್ಮಕ ಏಜೆಂಟ್‌ಗಳು (`mcp-adversarial-agents`)**
- **ಭದ್ರತಾ ತಿದ್ದುಪಡಿ — ಕಮಾಂಡ್ ಇಂಜೆಕ್ಷನ್**: TypeScript `run_python` ಉಪಕರಣದಲ್ಲಿ `execSync` ಶೆಲ್ ಇಂಟರ್‌ಪೋಲೇಷನ್ ಬದಲಾಗಿ `execFile` + `promisify` ಬಳಸಿ ಕಮಾಂಡ್ ಇಂಜೆಕ್ಷನ್ ಸಾಥ್ಯವನ್ನು ತೆಗೆದುಹಾಕಲಾಗಿದೆ (LLM ನಿಯಂತ್ರಿತ ಕೋಡ್ ಈಗ ಶೆಲ್ ಭಾಗವಿಲ್ಲದೆ ನೇರ `argv` ಅಂಶವಾಗಿ ಕಳುಹಿಸು)
- **MCP ಉಪಕರಣ ಲೂಪ್ ವೈರಿಂಗ್**: Python ತರ್ಕ ಸಂಚಾಲಕವನ್ನು `AsyncAnthropic` ಕ್ಲೈಂಟ್ ಬಳಸಿ (ಅವರೋಧಕ sync `Anthropic` ಬದಲಿಗೆ), ಪ್ರತಿ ಏಜೆಂಟ್ ತಿರುವಿಗೆ ನೇರ `ClientSession` ಹೊರತುಪಡಿಸಿ, ಪ್ರತಿ ಸುತ್ತಿನಲ್ಲಿ `session.list_tools()` ಮೂಲಕ ಉಪಕರಣ ವ್ಯಾಖ್ಯಾನಗಳನ್ನು ಪಡೆಯುತ್ತಿರುವುದು, ಹಾಗೂ ಮಾರುಕಟ್ಟೆಯಲ್ಲಿ `session.call_tool()` ಮೂಲಕ `tool_use` ಬ್ಲಾಕ್ಗಳನ್ನು ಸರಣಿಮಾಡುತ್ತಿರುವುದು

### MCP ಸ್ಪೆಸಿಫಿಕೇಶನ್ 2025-11-25 ಅನುಕೂಲ ವಿಮರ್ಶೆ

#### ಕೋರ್ ಸಂಪ್ರತಿಗಳಿಗೆ ಸುಧಾರಣೆ (01-CoreConcepts/)
- **ಹೊಸ ಕ್ಲೈಂಟ್ ಪ್ರಿಮಾನಿ - ರೂಟ್ಗಳು**: ಫೈಲ್ ಸಿಸ್ಟಮ್ ಗಡಿಭಾಗಗಳು ಮತ್ತು ಪ್ರವೇಶ ಅನುಮತಿಗಳನ್ನು ಗ್ರಹಿಸಲು ಸರ್ವರ್‌ಗಳಿಗೆ ಅವಕಾಶಿಸುವ ಕಾಂಪ್ರೀಹೆನ್ಸಿವ್ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಸೇರಿಸಲಾಗಿದೆ  
- **ಉಪಕರಣ ಸೂಚಿಕೆಗಳು**: ಉತ್ತಮ ಉಪಕರಣ ನಿರ್ವಹಣೆಗೆ `readOnlyHint`, `destructiveHint` ಉಪಕರಣ ವರ್ತನೆ ಸೂಚಿಕೆಗಳ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಸೇರಿಸಲಾಗಿದೆ  
- **ಸ್ಯಾಂಪ್ಲಿಂಗ್‌ನಲ್ಲಿ ಉಪಕರಣ ಕರೆಗಳನ್ನು ಸೇರಿಸಲಾಗಿದೆ**: ಮಾದರಿ ಚಾಲಿತ ಉಪಕರಣ ಸಲ್ಲಿಕೆಗಾಗಿ `tools` ಮತ್ತು `toolChoice` ಪ್ಯಾರಾಮೀಟರ್‌ಗಳ ಮಾಹಿತಿಯನ್ನು ಸೇರ್ಪಡೆ ಮಾಡಲಾಗಿದೆ  
- **URL ಮೋಡ್ ಈಲಿಸಿಟೇಶನ್**: ಸರ್ವರ್ ಪ್ರೇರಿತ ಹೊರಗಿನ ವೆಬ್ ಸಂವಹನಕ್ಕಾಗಿ URL ಆಧಾರಿತ ಈಲಿಸಿಟೇಶನ್ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಸೇರಿಸಲಾಗಿದೆ  
- **ಕಾರ್ಯ (ಪ್ರಯೋಗಾತ್ಮಕ)**: ಸ್ಥಾಯಿ ನಿರ್ವಹಣಾ ರಾಪರ್‌ಗಳು ಮತ್ತು ವಿಲಂಬಿತ ಫಲಿತಾಂಶ ಪಡೆಯುವ ಕಾರ್ಯಗಳ ಬಗ್ಗೆ ಹೊಸ ವಿಭಾಗ ಸೇರಿಸಲಾಗಿದೆ  
- **ಐಕಾನ್ ಬೆಂಬಲ**: ಉಪಕರಣಗಳು, ಸಂಪನ್ಮೂಲಗಳು, ಸಂಪನ್ಮೂಲ ಟೆಂಪ್ಲೇಟುಗಳು ಮತ್ತು ಪ್ರಾಂಪ್ಟ್‌ಗಳು ಹೆಚ್ಚುವರಿ ಮೆಟಾಡೇಟಾ ರೂಪে ಐಕಾನ್ ಗಳನ್ನು ಒಳಗೊಂಡಿರುವುದಾಗಿ ಸೂಚಿಸಲಾಗಿದೆ

#### ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ನವೀಕರಣಗಳು
- **README.md**: MCP ಸ್ಪೆಸಿಫಿಕೇಶನ್ 2025-11-25 ಆವೃತ್ತಿ ಉಲ್ಲೇಖ ಮತ್ತು ದಿನಾಂಕ ಆಧಾರಿತ ಆವೃತ್ತಿ ವ್ಯಾಖ್ಯಾನ ಸೇರಿಸಲಾಗಿದೆ  
- **study_guide.md**: ಕೋರ್ ಸಂಪ್ರತಿಗಳ ವಿಭಾಗದಲ್ಲಿ ಕಾರ್ಯಗಳು ಮತ್ತು ಉಪಕರಣ ಸೂಚಿಕೆಗಳ ಪಠ್ಯಕ್ರಮ ನಕ್ಷೆ ಅಪ್‌ಡೇಟ್ ಮಾಡಲಾಗಿದೆ; ಡಾಕ್ಯುಮೆಂಟ್ ಸಮಯ ಟೀಕೆ ನವೀಕರಿಸಲಾಗಿದೆ

#### ಸ್ಪೆಸಿಫಿಕೇಶನ್ ಅನುಕೂಲ ಪರಿಶೀಲನೆ
- **ಪ್ರೋಟೋಕಾಲ್ ಆವೃತ್ತಿ**: ಎಲ್ಲಾ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಪ್ರಸ್ತುತ MCP ಸ್ಪೆಸಿಫಿಕೇಶನ್ 2025-11-25 ಸಬೇಕು ಎಂದು ದೃಢಪಡಿಸಲಾಗಿದೆ  
- **ವಾಸ್ತುಶಿಲ್ಪ ಹೊಂದಾಣಿಕೆ**: ಡೇಟಾ ಲೇಯರ್ + ಟ್ರಾನ್ಸ್ಪೋರ್ಟ್ ಲೇಯರ್ ಎರಡು-ಪಟಲ ವಾಸ್ತುಶಿಲ್ಪ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಸಾಕ್ಷ್ಯೀಕರಿಸಲಾಗಿದೆ  
- **ಪ್ರಿಮಾನಿಗಳ ಡಾಕ್ಯುಮೆಂಟೇಶನ್**: ಸರ್ವರ್ ಪ್ರಿಮಾನಿಗಳು (ಸಂಪನ್ಮೂಲಗಳು, ಪ್ರಾಂಪ್ಟ್ಗಳು, ಉಪಕರಣಗಳು) ಮತ್ತು ಕ್ಲೈಂಟ್ ಪ್ರಿಮಾನಿಗಳು (ಸ್ಯಾಂಪ್ಲಿಂಗ್, ಈಲಿಸಿಟೇಶನ್, ಲಾಗಿಂಗ್, ರೂಟ್ಸ್) ಪರಿಶೀಲಿಸಲಾಗಿದೆ  
- **ಟ್ರಾನ್ಸ್ಪೋರ್ಟ್ ವ್ಯವಸ್ಥೆಗಳು**: STDIO ಮತ್ತು ಸ್ಟ್ರೀಮಬಲ್ HTTP ಟ್ರಾನ್ಸ್ಪೋರ್ಟ್ ಡಾಕ್ಯುಮೆಂಟ್ ಸಮಗ್ರತೆ ಪರಿಶೀಲನೆಯನ್ನು ಖಚಿತಪಡಿಸಲಾಗಿದೆ  
- **ಭದ್ರತಾ ಮಾರ್ಗದರ್ಶನ**: ಪ್ರಸ್ತುತ MCP ಭದ್ರತಾ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳಿಗೆ ಅನುಗುಣವಾಗಿ ದೃಢೀಕರಿಸಲಾಗಿದೆ

#### ಪ್ರಮುಖ MCP 2025-11-25 ವೈಶಿಷ್ಟ್ಯಗಳು ಡಾಕ್ಯುಮೆಂಟ್ ಆಗಿವೆ

- **ನಿರ್ವಹಣಾ ರಚನೆ**: MCP ಸರ್ಕಾರದಲ್ಲಿ ಕಾರ್ಯಸಂಪಾದಕ ಗುಂಪುಗಳು ಮತ್ತು ಆಸಕ್ತಿಯ ಗುಂಪುಗಳ ಅಧಿಕೃತ ರಚನೆ

#### ಮೂಲ ತತ್ವಗಳು (01-CoreConcepts/) - ಸಂಪೂರ್ಣ ಆಧುನೀಕರಣ  
- **ಪ್ರೋಟೊಕಾಲ್ ಆವೃತ್ತಿ ನವೀಕರಣ**: MCP ವಿಶೇಷಣ 2025-06-18 ದಿನಾಂಕ ಆಧಾರದ ಆವೃತ್ತಿಗೆ ನವೀಕರಿಸಲಾಗಿದೆ (YYYY-MM-DD ಫಾರ್ಮ್ಯಾಟ್)  
- **ವಾಸ್ತುಶಿಲ್ಪ ತಿದ್ದುಪಡಿ**: Hosts, Clients ಮತ್ತು Servers ಬಗ್ಗೆ ಪ್ರಸ್ತುತ MCP ವಾಸ್ತುಶಿಲ್ಪ ಮಾದರಿಗಳ ವಿವರವನ್ನು ಹೆಚ್ಚು ನಿಖರಗೊಳಿಸಲಾಗಿದೆ  
  - Hosts ಈಗ ಹಲವಾರು MCP ಕ್ಲೈಂಟ್ ಸಂಪರ್ಕಗಳನ್ನು ಸಂಯೋಜಿಸುವ ಎಐ ಅಪ್ಲಿಕೇಶನ್‌ಗಳಾಗಿ ಸ್ಪಷ್ಟವಾಗಿ ವ್ಯಾಖ್ಯಾನಿಸಲಾಗಿದೆ  
  - Clients ಪ್ರೋಟೊಕಾಲ್ ಸಂಪರ್ಕಕಗಳಾಗಿ ರಚಿಸಲಾಗಿದೆ, ಪ್ರತಿಯೊಂದು ಸರ್ವರ್ ಸಂಪರ್ಕದೊಂದಿಗೆ ಒಂದೊಂದು ಸಂಬಂಧಗಳನ್ನು ನಿರ್ವಹಿಸುವುದು  
  - Servers ಸ್ಥಳೀಯ ಮತ್ತು ದೂರದ ಜಾಗದಲ್ಲಿ ನಿಯೋಜನೆಯ ದೃಶ್ಯಾವಳಿ ನೀಡಲಾಗಿದೆ  
- **ಅಡಿಪಾಯ ಪುನರ್ ರಚನೆ**: ಸರ್ವರ್ ಮತ್ತು ಕ್ಲೈಂಟ್ ಅಡಿಪಾಯಗಳ ಸಂಪೂರ್ಣ ಮರುಬಳಕೆ  
  - ಸರ್ವರ್ ಅಡಿಪಾಯಗಳು: ಸಂಪನ್ಮೂಲಗಳು (ಡೇಟಾ ಮೂಲಗಳು), ಪ್ರಾಂಪ್ಟ್‌ಗಳು (ಟೆಂಪ್ಲೇಟುಗಳು), ಉಪಕರಣಗಳು (ಕರ್ಮ ನಿರ್ವಹಣೆ ಕಾರ್ಯಗಳು) ವಿವರವಾದ ವಿವರಣೆ ಮತ್ತು ಉದಾಹರಣೆಗಳೊಂದಿಗೆ  
  - ಕ್ಲೈಂಟ್ ಅಡಿಪಾಯಗಳು: ಸಂಗ್ರಹ (LLM ಪೂರ್ಣಗೊಳಣೆ), ಉತ್ಸಾಹ (ಬಳಕೆದಾರ ಇನ್ಪುಟ್), ಲಾಗಿಂಗ್ (ಡೆಬಗ್/ಮಾನಿಟರಿಂಗ್)  
  - ಪ್ರಸ್ತುತ ಕಂಡುಹಿಡಿಯುವ (`*/list`), ಪಡೆದುಕೊಳ್ಳುವ (`*/get`), ಮತ್ತು ಕಾರ್ಯನಿರ್ವಹಿಸುವ (`*/call`) ವಿಧಾನ ಮಾದರಿಗಳನ್ನು ನವೀಕರಿಸಲಾಗಿದೆ  
- **ಪ್ರೋಟೊಕಾಲ್ ವಾಸ್ತುಶಿಲ್ಪ**: ಎರಡು-ಪಾತಿ ವಾಸ್ತುಶಿಲ್ಪ ಮಾದರಿಯನ್ನು ಪರಿಚಯಿಸಲಾಗಿದೆ  
  - ಡೇಟಾ ಪದರ: JSON-RPC 2.0 ಆಧಾರಿತ ಜೀವನಚಕ್ರ ನಿರ್ವಹಣೆ ಮತ್ತು ಅಡಿಪಾಯಗಳೊಂದಿಗೆ  
  - ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ ಪದರ: STDIO (ಸ್ಥಳೀಯ) ಮತ್ತು Streamable HTTP SSE (ದೂರದ) ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ ಯಂತ್ರಗಳು  
- **ಭದ್ರತಾ ಚಟುವಟಿಕೆ ಕ್ರಮ**: ಬಳಕೆದಾರ ಸಮ್ಮತಿಯನ್ನು ಸ್ಪಷ್ಟಪಡಿಸುವ, ಡೇಟಾ ಹೊಂದಾಣಿಕೆ ರಕ್ಷಿಸುವ, ಉಪಕರಣ ಕಾರ್ಯನಿರ್ವಾಹ ಸುರಕ್ಷತೆ ಮತ್ತು ಟ್ರಾನ್ಸ್‌ಪೋರ್ಟ್ ಭದ್ರತೆಗೆ ಸಂಬಂಧಿಸಿದ ಸಂಪೂರ್ಣ ಭದ್ರತಾ ತತ್ವಗಳೊಂದಿಗೆ  
- **ಸಂವಹನ ಮಾದರಿಗಳು**: ಪ್ರಾರಂಭ, ಅನ್ವೇಷಣೆ, ಕಾರ್ಯನಿರ್ವಹಣೆ ಮತ್ತು ತಿಳಿವು ಮೊಬೈಲೆಟ್ ಅನ್ನು ಪ್ರದರ್ಶಿಸುವ ಪ್ರೋಟೊಕಾಲ್ ಸಂದೇಶಗಳನ್ನು ನವೀಕರಿಸಲಾಗಿದೆ  
- **ಕೋಡ್ ಉದಾಹರಣೆಗಳು**: ಪ್ರಸ್ತುತ MCP SDK ಮಾದರಿಗಳಿಗೆ ಹೊಂದಿಕೊಳ್ಳುವಂತೆ  ಪ್ರತಿ ಭಾಷಾ ಉದಾಹರಣೆಗಳು (.NET, Java, Python, JavaScript) ನವೀಕರಿಸಲಾಗಿದೆ

- **ಸುಧಾರಿತ ಧಮಕಿ ತಡೆ**:  
  - MCP-ನಿರ್ದಿಷ್ಟ ಹೈಜೆಕಿಂಗ್ ದಾಳಿ ದೃಶ್ಯಗಳು ಮತ್ತು ಕ್ರಿಪ್ಟೋಗ್ರಾಫಿಕ್ ಸೆಷನ್ ಐಡಿ ಅಗತ್ಯಗಳೊಂದಿಗೆ ಸೆಷನ್ ಹೈಜೆಕಿಂಗ್  
  - MCP ಪ್ರಾಕ್ಸಿ ದೃಶ್ಯಗಳಲ್ಲಿ Confused Deputy ಸಮಸ್ಯೆಗಳು ಹಾಗೂ ಸ್ಪಷ್ಟ ಒಪ್ಪಂದ ಅಗತ್ಯಗಳೊಂದಿಗೆ  
  - ಟೋಕನ್ ಪಾಸ್ಥ್ರೂ ಬಲಹೀನತೆಗಳಿಗೆ ಕಡ್ಡಾಯ ಪರಿಶೀಲನಾ ನಿಯಂತ್ರಣಗಳು  
- **ಸರಬರಾಜು ಸರಪಳಿ ಭದ್ರತೆ**: ಸ್ಥಾಪನೆಯಲ್ಲಿನ ಮಾದರಿಗಳು, ಎಂಬೆಡಿಂಗ್ ಸೇವೆಗಳು, ಸಂಧರ್ಭ ಪೂರೈಕೆದಾರರು ಮತ್ತು ಮೂರನೇ-ಪಕ್ಷ API ಗಳನ್ನು ಒಳಗೊಂಡಂತೆ ಎಐ ಸರಬರಾಜು ಸರಪಳಿಗೆ ವಿಸ್ತಾರವಾದ ವ್ಯಾಪ್ತಿ  
- **ಪಾಯ್ದು ಭದ್ರತೆ**: ಶೂನ್ಯ ನಂಬಿಕೆಯ ವಾಸ್ತುಶಿಲ್ಪ ಮತ್ತು Microsoft ಭದ್ರತಾ ಪರಿಸರ ಸೇರಿದಂತೆ ಎಂಟರಪ್ರೈಸ್ ಭದ್ರತಾ ಮಾದರಿಗಳ ಸಹಕಾರ  
- **ಸಂಪನ್ಮೂಲ ಸಂಘಠನ**: ಪ್ರಕಾರ (ಅಧಿಕೃತ ಡಾಕ್ಸ್, ಮಾನದಂಡಗಳು, ಸಂಶೋಧನೆ, Microsoft ಪರಿಹಾರಗಳು, ಅನುಷ್ಠಾನ ಮಾರ್ಗದರ್ಶಿಗಳು)ನ ಪ್ರಕಾರ ಸಮಗ್ರ ಸಂಪನ್ಮೂಲ ಲಿಂಕ್ ವರ್ಗೀಕರಣ

### ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಗುಣಮಟ್ಟ ಸುಧಾರಣೆಗಳು  
- **ರಚನಾತ್ಮಕ ಕಲಿಕೆ ಗುರಿಗಳು**: ನಿರ್ದಿಷ್ಟ ಗಮ್ಯಗಳನ್ನು ಹೊಂದಿರುವ ಸುಧಾರಿತ ಕಲಿಕೆ ಗುರಿಗಳು  
- **ಅಂತರ್-ಉಲ್ಲೇಖಗಳು**: ಸಂಬಂಧಿತ ಭದ್ರತಾ ಮತ್ತು ಮೂಲ ತತ್ವ ವಿಷಯಗಳ ನಡುವೆ ಲಿಂಕ್‌ಗಳನ್ನು ಸೇರಿಸಲಾಗಿದೆ  
- **ಪ್ರಸ್ತುತ ಮಾಹಿತಿ**: ಎಲ್ಲಾ ದಿನಾಂಕ ಉಲ್ಲೇಖಗಳನ್ನು ಮತ್ತು ವಿಶೇಷಣ ಲಿಂಕ್‌ಗಳನ್ನು ನವೀಕರಿಸಲಾಗಿದೆ  
- **ಅನುಷ್ಠಾನ ಮಾರ್ಗದರ್ಶನ**: ಎರಡು ವಿಭಾಗಗಳಲ್ಲಿಯೂ ನಿರ್ದಿಷ್ಟ ಮತ್ತು ಕಾರ್ಯಾಚರಣೀಯ ಅನುಷ್ಠಾನ ಮಾರ್ಗದರ್ಶಿಗಳೊಂದಿಗೆ ಹೆಚ್ಚಿಸಲಾಗಿದೆ
